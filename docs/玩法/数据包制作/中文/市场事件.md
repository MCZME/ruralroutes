# 市场事件生成规则

这里定义的不是某个周期里已经生成好的市场事件，而是市场事件生成规则。

每条规则会在贸易周期刷新时被抽取，并根据 `scopes` 展开为一个或多个实际生效的 `MarketEvent`。

它主要做三件事：

- 指定哪些物品或货物分组会受影响
- 指定规则生成的事件在哪些范围生效
- 指定传闻板该怎么描述这条事件

## 文件位置

```text
data/<namespace>/ruralroutes/market_event_rules/<rule>.json
```

`<namespace>` 可以自定义，不必使用 `ruralroutes`。

## 一份生成规则长什么样

```json
{
  "id": "ruralroutes:common/food_shortage",
  "name_key": "market.ruralroutes.common.food_shortage",
  "target_ref": "#ruralroutes:pool/food",
  "scopes": [
    {
      "type": "global",
      "selector": "all"
    }
  ],
  "delta": 0.15,
  "stock": {
    "sell": -0.35,
    "buy": 0.10
  },
  "weight": 100,
  "rumor_family": "shortage",
  "rumor_target_key": "rumor.target.food"
}
```

## 字段说明

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `id` | 是 | 规则 ID。建议写成 `namespace:分类/名字`。 |
| `name_key` | 是 | 规则名称的本地化 key，主要用于展示和调试。 |
| `target_ref` | 是 | 受影响目标。详见 [目标引用](目标引用.md)。 |
| `scopes` | 是 | 作用域规则。 |
| `delta` | 是 | 价格变化幅度。正数表示涨价，负数表示降价。 |
| `stock` | 否 | 库存修正。只改本期库存，不改长期规则。 |
| `weight` | 否 | 抽取权重，默认 `100`。 |
| `rumor_family` | 是 | 传闻语义类型。 |
| `rumor_target_key` | 否 | 传闻里显示的目标名称。 |

## 目标怎么写

`target_ref` 直接决定这条规则要影响什么。

常见写法有 3 种：

- `minecraft:bread`：精确物品
- `#ruralroutes:pool/food`：标签
- `@plains_workshop/basic_tools`：来源键

优先用标签，尤其是面向一类货物的事件。
如果只是某个特殊商品，直接写精确物品也可以。

## `scopes` 怎么写

`scopes` 是“这条规则会展开到哪里生效”。

### 单条作用域

```json
[
  {
    "type": "global",
    "selector": "all"
  }
]
```

### 字段说明

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `type` | 是 | `global`、`biome`、`theme` 之一。 |
| `selector` | 是 | `all` 或 `list`。 |
| `targets` | `selector=list` 时需要 | 目标列表。这里写群系或主题 ID，不是 [目标引用](目标引用.md) 里的那种目标写法。 |

### 三种作用域

| 值 | 说明 |
| :--- | :--- |
| `global` | 全局生效，直接生成 1 条全局事件。 |
| `biome` | 只影响指定群系；`all` 会展开到当前可用群系，`list` 只展开到列出的群系。 |
| `theme` | 只影响指定主题；`all` 会展开到当前可用主题，`list` 只展开到列出的主题。 |

### `selector=list` 示例

```json
[
  {
    "type": "biome",
    "selector": "list",
    "targets": [
      "minecraft:plains"
    ]
  }
]
```

这种写法表示：只让这条规则在列出的目标里生效，不会自动扩展到全部群系或主题。

## 库存怎么改

`stock` 是可选项，用来修正本期的库存基线。

```json
{
  "sell": -0.35,
  "buy": 0.10
}
```

### 字段说明

| 字段 | 说明 |
| :--- | :--- |
| `sell` | 修正出售现货量。 |
| `buy` | 修正收购容量。 |

`sell` 和 `buy` 都是百分比修正。
正数表示增加，负数表示减少。

## `rumor_family`

`rumor_family` 用来决定这条传闻属于哪类语义。

- `shortage`：短缺，表示货源紧
- `surplus`：过剩，表示货物多
- `demand`：需求，表示有人想要
- `release`：放货，表示供应放松或货物释放出来了

## `rumor_target_key`

传闻不是直接拿 `target_ref` 去显示，而是先经过一层传闻生成。

规则生成 `MarketEvent` 之后，系统会根据事件的 `target_ref` 和 `rumor_family` 生成传闻条目。显示时，传闻会先取目标名称，再套用对应的语义模板。

`rumor_target_key` 就是给这一步用的“玩家视角目标名称”。

如果不写，系统会根据 `target_ref` 自动兜底：

- 标签会转成标签型传闻键，比如 `#ruralroutes:pool/food` 会优先走标签名
- 物品会转成物品翻译键，比如 `minecraft:bread` 会走物品名

如果你想让传闻更自然、更像人话，就手动写这个字段。比如 `@plains_workshop/basic_tools` 这类来源键，系统本身不知道该怎么直接翻成玩家常用叫法，手动写成 `rumor.target.tool` 之类的键，传闻就会显示成更自然的“工具”“吃食”“木料”这些说法。

具体例子：

```json
{
  "rumor.shortage.biome.1": "听说%s一带缺%s。",
  "rumor.target.tool": "工具"
}
```

如果一条规则的 `rumor_family` 是 `shortage`，作用域是沙漠群系，并且 `rumor_target_key` 写成 `rumor.target.tool`，那么传闻生成时会把群系名称和目标名称填进模板。玩家最终看到的文本会类似：

```text
听说沙漠一带缺工具。
```

## 怎么使用

### 1. 先选目标

先决定这条规则是针对一类货物，还是某个特殊商品。

### 2. 再选作用域

- 全图通用，就用 `global`
- 某个群系特有，就用 `biome`
- 某个主题专属，就用 `theme`

### 3. 再定强度

`delta` 控制价格波动幅度，`stock` 控制库存联动。

一般建议不要写得太猛，市场波动更适合做阶段性窗口。

## 示例

### 全局食品短缺

```json
{
  "id": "ruralroutes:common/food_shortage",
  "name_key": "market.ruralroutes.common.food_shortage",
  "target_ref": "#ruralroutes:pool/food",
  "scopes": [
    {
      "type": "global",
      "selector": "all"
    }
  ],
  "delta": 0.15,
  "stock": {
    "sell": -0.35,
    "buy": 0.10
  },
  "weight": 100,
  "rumor_family": "shortage",
  "rumor_target_key": "rumor.target.food"
}
```

### 主题专属工具热销

```json
{
  "id": "ruralroutes:biome/plains/tool_demand",
  "name_key": "market.ruralroutes.biome.plains.tool_demand",
  "target_ref": "@plains_workshop/basic_tools",
  "scopes": [
    {
      "type": "biome",
      "selector": "list",
      "targets": [
        "minecraft:plains"
      ]
    }
  ],
  "delta": 0.22,
  "stock": {
    "sell": -0.12,
    "buy": 0.35
  },
  "weight": 40,
  "rumor_family": "demand",
  "rumor_target_key": "rumor.target.tool"
}
```

## 注意事项

- 一个规则可以按作用域生成一条或多条实际事件
- `GLOBAL` 通常生成 1 条事件，`BIOME` / `THEME` 可能按目标列表展开为多条事件
- 同一物品命中多个事件时会叠加
- 同一 `rule_id + target_ref + scope_type + scope_target` 的事件会去重
- `stock` 只改库存，不改价格本体
- `GLOBAL` 不需要 `scope_target`
- `weight` 只影响抽取概率，不影响强度
