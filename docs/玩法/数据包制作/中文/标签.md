# 标签

标签是 Minecraft 数据包中对物品进行分组的方式。

《乡野商路》会读取这些标签，用它们来组织价值表、主题买卖内容、市场事件目标和传闻文本。

## 文件位置

```text
data/<namespace>/tags/item/<tag>.json
```

这里的 `<namespace>` 可以换成你自己的命名空间，不必使用 `ruralroutes`。

NeoForge 官方文档：[Tags](https://docs.neoforged.net/docs/1.21.1/resources/server/tags)

## 文件结构

```json
{
  "replace": false,
  "values": [
    "minecraft:stone",
    "minecraft:cobblestone",
    "#minecraft:stairs",
    {
      "id": "othermod:optional_item",
      "required": false
    }
  ]
}
```

### 字段

| 字段 | 说明 |
| :--- | :--- |
| `replace` | 是否替换已有标签内容。通常写 `false`。 |
| `values` | 标签内容列表。 |

## `values` 怎么写

`values` 里可以写：

| 写法 | 示例 | 说明 |
| :--- | :--- | :--- |
| 物品 ID | `minecraft:stone` | 加入单个物品。 |
| 标签 ID | `#minecraft:planks` | 引用另一个标签。 |
| 模组物品 ID | `othermod:example_item` | 引用其他模组物品。 |
| 可选条目 | `{ "id": "othermod:example_item", "required": false }` | 目标不存在时不报错。 |

## 本模组的建议使用方法

标签在本模组里主要有三类作用：

- 给价值表提供批量定价目标。
- 给交易内容提供候选物品来源。
- 给市场事件提供影响目标。

建议按用途把标签分成两类。

### 系统分类标签

```text
#<namespace>:pool/<类别>
```

这类标签表达“这是什么类资源”，常用于价值表、主题价格修正、市场事件和传闻。

例子：

```text
#examplepack:pool/stone
#examplepack:pool/wood
#examplepack:pool/food
```

### 生成候选标签

```text
#<namespace>:candidate/biome/<群系>/<类别>
#<namespace>:candidate/theme/<主题>/<类别>
```

这类标签表达“某类村庄更容易出现哪些物品”，常用于主题交易内容。

例子：

```text
#examplepack:candidate/biome/plains/crop
#examplepack:candidate/theme/plains_workshop/tool_goods
```

## 举例用法

### 价值表

价值表可以直接用标签批量定价：

```json
{
  "replace": true,
  "values": {
    "#examplepack:pool/wood": { "value": 2 }
  }
}
```

### 交易内容

交易内容可以用标签作为候选来源：

```json
{
  "items": [
    "#examplepack:candidate/biome/plains/crop",
    "minecraft:bread"
  ],
  "pick": 2
}
```

### 市场事件

市场事件可以用标签作为影响目标，写法和其他目标引用一致，详见 [目标引用](目标引用.md)：

```json
{
  "target_ref": "#examplepack:pool/food"
}
```

## 示例

```json
{
  "replace": false,
  "values": [
    "minecraft:oak_log",
    "minecraft:spruce_log",
    "minecraft:acacia_log",
    "minecraft:charcoal"
  ]
}
```

这个标签可以被价值表、交易内容和市场事件一起引用。

## 注意事项

- 标签不一定都属于“物品池”，它只是 Minecraft 的分组机制。
- 命名空间可以自定义。
- 做完整覆盖时，记得确认 `replace` 是否应该为 `true`。
- 不要让标签互相循环引用，否则数据包加载会失败。
