# 主题模板

主题模板定义一个村庄的身份骨架。它负责说明：

- 这个村庄属于哪个群系。
- 这个村庄的基础库存大概怎么分布。
- 这个村庄要挂哪些交易内容。

主题模板不直接写完整买卖表。实际的买卖组合由交易内容提供。

## 文件位置

```text
data/<namespace>/ruralroutes/themes/<主题名>.json
```

`<namespace>` 可以自定义，不必使用 `ruralroutes`。

## 文件结构

```json
{
  "name": "examplepack:plains_granary",
  "biome": "minecraft:plains",
  "stock": {
    "default": { "min": 10, "max": 18 },
    "targets": {
      "#examplepack:candidate/biome/plains/crop": { "min": 20, "max": 30 },
      "@plains_granary/staples": {
        "sell": { "min": 18, "max": 28 },
        "buy": { "min": 8, "max": 12 }
      },
      "minecraft:golden_carrot": { "min": 1, "max": 3 }
    }
  },
  "price_modifiers": [
    {
      "target": "#examplepack:pool/crop",
      "sell": 0.82,
      "buy": 0.76
    },
    {
      "target": "@plains_granary/staples",
      "sell": 0.92,
      "buy": 1.08
    }
  ],
  "trade_profiles": [
    "examplepack:plains_granary_basic"
  ]
}
```

### 字段说明

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `name` | 是 | 主题 ID。建议写成 `namespace:主题名`。 |
| `biome` | 是 | 所属群系 ID。 |
| `stock` | 是 | 主题级库存设置。必须包含 `default`。 |
| `price_modifiers` | 否 | 主题级价格修正列表。 |
| `trade_profiles` | 否 | 需要合并进这个主题的交易内容列表。 |

## 库存

```json
{
  "default": { "min": 10, "max": 18 },
  "targets": {
    "#examplepack:candidate/biome/plains/crop": { "min": 20, "max": 30 },
    "@plains_granary/staples": {
      "sell": { "min": 18, "max": 28 },
      "buy": { "min": 8, "max": 12 }
    },
    "minecraft:golden_carrot": { "min": 1, "max": 3 }
  }
}
```

### 字段说明

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `default` | 是 | 默认库存范围。 |
| `targets` | 否 | 按目标键批量设置库存。详见 [目标引用](目标引用.md)。 |

`targets` 里的值可以直接写 `{ "min": 10, "max": 18 }`，也可以写成对象，分别控制出售和收购时的库存范围。  
`specific` 已移除，单个物品也直接写进 `targets`。

## 价格修正

```json
[
  {
    "target": "#examplepack:pool/crop",
    "sell": 0.82,
    "buy": 0.76
  }
]
```

### 字段说明

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `target` | 是 | 目标引用。详见 [目标引用](目标引用.md)。 |
| `sell` | 是 | 村庄卖给玩家时的修正系数。 |
| `buy` | 是 | 村庄从玩家收购时的修正系数。 |

## 交易内容

这是一个字符串数组，用来引用交易内容文件。

```json
"trade_profiles": ["examplepack:plains_granary_basic"]
```

### 作用

- 让主题和交易内容分离。
- 便于多个主题复用同一批交易内容。
- 便于整合包只改交易组合，不改主题身份。
- 按写入顺序合并，建议保持固定顺序。

## 怎么使用

### 1. 先定义身份

主题先回答这几个问题：

- 这个村庄属于哪个群系。
- 它偏向什么资源方向。
- 它的库存大概是什么量级。
- 它对哪些物品更便宜或更贵。

### 2. 再挂交易内容

主题本身不负责写完整买卖表，只负责引用交易内容。

### 3. 保持主题稳定

主题 ID 一旦开始被村庄、文案、进度和市场规则引用，最好不要频繁改名。

## 本模组的建议使用方法

- 主题层只做偏移，不要变成第二套商店。
- `stock.default` 先定基线，再用 `targets` 做局部调整。
- `stock` 只写主题骨架，不要把所有物品都塞进去。
- `price_modifiers` 优先用标签，精确物品只留给标志性商品。
- `trade_profiles` 负责组合，不负责分类。
