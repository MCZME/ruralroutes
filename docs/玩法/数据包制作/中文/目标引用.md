# 目标引用

目标引用用来告诉规则“要影响谁”。

在数据包里，下面这些字段使用的是同一套目标引用写法：

- `target_ref`
- `target`
- `stock.targets` 里的每一个键

注意：市场事件 `scopes` 里的 `targets` 是作用域目标列表，用来写群系或主题 ID，不属于这里说的目标引用。

## 三种常用写法

### 精确物品

```json
"minecraft:bread"
```

只影响这个物品。

适合少数特殊商品，例如烈焰棒、末影珍珠、蓝冰。

### 标签

```json
"#examplepack:pool/food"
```

影响标签里的全部物品。

适合一类货物，例如食物、木料、矿料、染色商品。

### 来源键

```json
"@plains_granary/staples"
```

影响交易内容里带有对应 `key` 的条目。

来源键来自交易内容里的 `key` 字段。例如：

```json
{
  "key": "plains_granary/staples",
  "items": [
    "minecraft:bread",
    "#examplepack:candidate/biome/plains/crop"
  ],
  "pick": 2
}
```

引用它时要在前面加 `@`：

```json
"@plains_granary/staples"
```

## 用在不同字段里

### `target_ref`

`target_ref` 是单个目标引用。

```json
{
  "target_ref": "#examplepack:pool/food"
}
```

### `target`

`target` 也是单个目标引用。

```json
{
  "target": "@plains_granary/staples",
  "sell": 0.92,
  "buy": 1.08
}
```

### `stock.targets`

`stock.targets` 是一组目标引用。这里不是把目标写在字段值里，而是把目标引用写成键。

```json
{
  "targets": {
    "#examplepack:pool/food": { "min": 20, "max": 30 },
    "@plains_granary/staples": {
      "sell": { "min": 18, "max": 28 },
      "buy": { "min": 8, "max": 12 }
    },
    "minecraft:golden_carrot": { "min": 1, "max": 3 }
  }
}
```

## 带组件的精确物品

如果必须区分同一种物品的不同组件，可以用对象写法。

```json
{
  "item": "minecraft:written_book",
  "components": {
    "minecraft:custom_name": "{\"text\":\"Village Ledger\"}"
  }
}
```

这种写法只适合单个精确物品。标签和来源键不要写 `components`。

在 `stock.targets` 这种以键作为目标的地方，不建议使用带组件对象。需要组件区分时，优先在交易内容里给条目设置稳定 `key`，再用 `@来源键` 引用。

## 怎么选择

- 面向一类货物时，用标签。
- 面向某个固定物品时，用精确物品。
- 面向一组交易候选或带组件条目时，用来源键。

## 注意事项

- 来源键引用时要写 `@`。
- 标签引用时要写 `#`。
- `target_ref`、`target` 和 `stock.targets` 的目标含义一致，只是所在字段结构不同。
- 修改来源键会影响所有引用它的库存、价格和市场规则。
