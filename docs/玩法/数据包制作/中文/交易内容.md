# 交易内容

交易内容负责定义一个主题真正会卖什么、收什么，以及少量特殊交换规则。

它和主题模板分开，是为了让“身份”与“买卖组合”解耦。

## 文件位置

```text
data/<namespace>/ruralroutes/trade_profiles/<profile名>.json
```

`<namespace>` 可以自定义，不必使用 `ruralroutes`。

## 文件结构

```json
{
  "name": "examplepack:plains_granary_basic",
  "sell_items": [
    "minecraft:oak_planks",
    {
      "key": "plains_granary/staples",
      "items": [
        "minecraft:bread",
        "#examplepack:candidate/biome/plains/crop"
      ],
      "pick": 2
    }
  ],
  "buy_items": [
    "minecraft:wheat",
    {
      "key": "plains_granary/procurement",
      "items": [
        "minecraft:raw_iron",
        "minecraft:charcoal"
      ],
      "pick": 1
    }
  ],
  "stock": {
    "targets": {
      "@plains_granary/staples": { "min": 18, "max": 32 },
      "#examplepack:candidate/biome/plains/crop": {
        "sell": { "min": 18, "max": 28 },
        "buy": { "min": 8, "max": 12 }
      }
    }
  },
  "trade_contracts": [
    {
      "type": "fixed",
      "inputs": [
        { "item": "minecraft:carrot", "count": 8 }
      ],
      "outputs": [
        { "item": "minecraft:golden_carrot", "count": 1 }
      ]
    }
  ]
}
```

### 字段说明

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `name` | 是 | 交易内容 ID。建议写成 `namespace:profile名`。 |
| `sell_items` | 否 | 村庄卖给玩家的候选条目。可以留空。 |
| `buy_items` | 否 | 村庄从玩家收购的候选条目。可以留空。 |
| `stock` | 否 | 这组交易内容的库存补丁，只能写 `targets`。目标写法详见 [目标引用](目标引用.md)。 |
| `trade_contracts` | 否 | 附加交换契约。通常只给少量特殊交换使用。 |

## 物品条目

`sell_items` 和 `buy_items` 都是 `ItemReference` 列表。每个条目都可以写成下面三种形态。

### 单项简写

```json
"minecraft:oak_planks"
```

表示一个精确物品或标签。

### 单项对象

```json
{
  "id": "minecraft:bread",
  "key": "plains_granary/bread",
  "components": {
    "minecraft:custom_name": "{\"text\":\"Village Bread\"}"
  }
}
```

适合带组件的精确物品。`key` 可以一起写，用来给这个条目一个稳定来源键。

### 候选组

```json
{
  "key": "plains_granary/staples",
  "items": [
    "minecraft:bread",
    {
      "id": "minecraft:written_book",
      "components": {
        "minecraft:written_book_content": {
          "title": "Plains Notes",
          "author": "Village Scribe",
          "pages": [
            "{\"text\":\"local trade note\"}"
          ]
        }
      }
    },
    "#examplepack:candidate/biome/plains/crop"
  ],
  "pick": 2
}
```

### 字段说明

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `id` | 单项时可用 | 单个物品或标签。 |
| `items` | 候选组时需要 | 候选列表，每项可以是物品、标签，或带组件的单项对象。 |
| `pick` | 否 | 从候选里随机抽取多少个；不写就全用。 |
| `key` | 否 | 稳定来源键，建议在候选组里写。 |
| `components` | 否 | 只用于单项精确物品。 |

`pick` 是在候选展开后再抽取。候选数不够时不会报错，会直接全用。

## 库存补丁

`stock` 只用来补充这组交易内容的库存，不负责定义默认基线。默认基线仍由主题模板提供。

```json
{
  "targets": {
    "@plains_granary/staples": { "min": 18, "max": 32 },
    "#examplepack:candidate/biome/plains/crop": {
      "sell": { "min": 18, "max": 28 },
      "buy": { "min": 8, "max": 12 }
    }
  }
}
```

### 字段说明

| 字段 | 说明 |
| :--- | :--- |
| `targets` | 按目标引用设置库存。详见 [目标引用](目标引用.md)。值可以是共享范围，也可以分别写 `sell` / `buy`。 |

`stock` 里不要写 `default`。如果你要给整个主题设默认库存，请写在主题模板里。

## 交换契约

`trade_contracts` 是可选项。普通买卖只写 `sell_items` 和 `buy_items` 就够了。

### `fixed`

固定交换规则。

```json
{
  "type": "fixed",
  "inputs": [
    { "item": "minecraft:carrot", "count": 8 }
  ],
  "outputs": [
    { "item": "minecraft:golden_carrot", "count": 1 }
  ]
}
```

字段说明：

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `type` | 是 | 固定写死为 `fixed`。 |
| `inputs` | 是 | 玩家需要投入的物品和数量。 |
| `outputs` | 是 | 玩家得到的物品和数量。 |

### `currency_basket_dynamic`

动态组合契约，用于把价格拆成一组可接受物品。

它只用于货币组合，不用于普通物品的通用拼装或其他类型的组合逻辑。

`accepted_currencies` 可以写货币标签，也可以写具体货币物品。
常用写法是：

- `#ruralroutes:currency`：全部货币物品
- `#ruralroutes:currency_base`：基础货币
- `ruralroutes:copper_coin`、`ruralroutes:iron_coin`、`ruralroutes:gold_coin`：单独指定某几种货币

```json
{
  "type": "currency_basket_dynamic",
  "side": "sell_to_player",
  "items": [
    "minecraft:bread"
  ],
  "accepted_currencies": [
    "#ruralroutes:currency_base",
    "ruralroutes:iron_coin",
    "ruralroutes:gold_coin"
  ],
  "composition": "largest_first"
}
```

字段说明：

| 字段 | 必需 | 说明 |
| :--- | :--- | :--- |
| `type` | 是 | 固定写死为 `currency_basket_dynamic`。 |
| `side` | 是 | `sell_to_player` 或 `buy_from_player`。 |
| `items` | 是 | 参与货币组合的物品列表。 |
| `accepted_currencies` | 是 | 可用于组合的货币列表。可以写货币标签，也可以写具体货币物品。 |
| `composition` | 是 | 组合策略。 |

### 组合策略

`composition` 现在有 3 种写法：

| 值 | 说明 | 适用场景 |
| :--- | :--- | :--- |
| `largest_first` | 先用面额最大的货币拆分，再补更小面额。 | 最常用的默认写法，适合大多数交易。 |
| `smallest_only` | 只用可用列表里最小的面额输出。 | 想让报价尽量保持单一小额货币时使用。 |
| `single` | 如果只给了一种可用货币，就只用这一种；如果给了多种，就退回到 `largest_first`。 | 只想在单一货币场景下强制保持简洁时使用。 |

如果你不确定怎么选，优先用 `largest_first`。

如果某个物品没有被任何 `currency_basket_dynamic` 命中，就会回退到默认货币篮，只使用基础货币继续计算。也就是说，价格会直接用基础货币表达，比如某个物品价格是 `110`，默认就是 `110` 铜币。

## 怎么使用

### 1. 先写候选，再写精确项

大多数情况下，先用标签和候选组组织，再用精确物品补特殊货。

### 2. 让卖货和收货保持语义对称

卖什么、收什么最好能对应主题身份，不要各写各的。

### 3. 契约只做特殊行为

固定契约和动态组合契约适合做少量关键交换，不要把所有交易都写成契约。

### 4. `key` 尽量固定

如果某个候选组会被库存、价格或其他规则引用，`key` 最好写稳定，不要随便改。

## 注意事项

- `sell_items` 和 `buy_items` 可以留空，但通常至少要有一侧。
- `stock` 只写 `targets`，不要写 `default`。
- `pick` 会随机抽样，想固定结果就不要写。
- 候选组里重复出现同一物品时，建议保持顺序稳定。
