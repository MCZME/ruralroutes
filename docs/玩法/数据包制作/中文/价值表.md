# 价值表

价值表是基础定价表。它只管“这个物品最基础值多少”，不管主题修正、库存、市场波动。

## 文件位置

```text
data/<namespace>/data_maps/item/value.json
```

这里的 `<namespace>` 可以换成你自己的命名空间，不必使用 `ruralroutes`。

## 文件结构

```json
{
  "replace": true,
  "values": {
    "#examplepack:pool/stone": { "value": 1 },
    "#examplepack:pool/crop": { "value": 2 },
    "minecraft:diamond": { "value": 40 }
  }
}
```

### 根字段

| 字段 | 说明 |
| :--- | :--- |
| `replace` | 建议显式写出。做一套完整价值表时，通常用 `true`。 |
| `values` | 价值条目。 |

## 单个条目

每个条目都很简单：

```json
{ "value": 40 }
```

### 字段

| 字段 | 说明 |
| :--- | :--- |
| `value` | 基础价值，整数。 |

## `values` 怎么写

键可以写成：

| 写法 | 示例 | 说明 |
| :--- | :--- | :--- |
| 物品 ID | `minecraft:iron_ingot` | 单独定价。 |
| 标签 ID | `#examplepack:pool/stone` | 给一整类物品定价。 |

## 默认值

未命中的物品会回退到模组配置里的默认值。这个值不是数据包字段，是兜底值。

## 例子

```json
{
  "replace": true,
  "values": {
    "#examplepack:pool/wood": { "value": 2 },
    "#examplepack:pool/food": { "value": 4 },
    "minecraft:golden_carrot": { "value": 40 }
  }
}
```

## 注意事项

- 价值表只给基础值。
- 命名空间可以自定义。
- 做完整方案时，记得显式写 `replace`。
