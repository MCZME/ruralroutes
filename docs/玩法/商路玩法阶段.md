本文件记录乡野商路的核心玩法阶段，用于把“跨村跑商是中期以后主循环”落成可检查的设计框架。

## 核心判断

中期以后，跨村跑商是主循环；原地采集只是启动、补洞和应急手段，不应成为长期最优解。

玩家的主要成长不来自固定地点生产，而来自：

1. 识别不同村庄的供需身份。
2. 发现足够多的村庄节点，理解不同区域和主题的资源身份。
3. 在多个村庄之间组织连续货物流，而不是只完成单次点对点转卖。
4. 利用库存、周期和市场事件调整商路。
5. 将利润和高价值路线货转化为终局关键材料。

## 四个阶段

| 阶段 | 核心目标 | 玩家主要行为 | 系统重点 |
| :--- | :--- | :--- | :--- |
| 起步期 | 认识贸易系统并获得第一笔周转资源 | 卖出附近资源，查看贸易站、展示柜和传闻板 | 入口清晰、交易可完成、补给不断裂 |
| 拓路发现期 | 发现足够多的村庄节点 | 使用商路图册、自然探索和贸易站确认记录更多村庄 | 节点扩张、风格识别、路线素材积累 |
| 商路构建期 | 把多个村庄组织成可重复的货物流 | 手动规划商路边，串联补给、产地、需求地和高价值节点 | 多节点路线、库存压力、事件调线 |
| 终局收束期 | 把跑商成果转化为通关关键材料 | 用金币、高价值路线货和收束合约获取末影珍珠、烈焰棒 | T5 消耗财富，不成为倒卖利润来源 |

## 阶段一：起步期

起步期的职责是让玩家确认“村庄贸易系统可用”，而不是立刻要求玩家理解完整路线网络。

玩家应能通过附近易得的大宗资源完成第一批交易，例如木材、作物、燃料、常见副产物、基础建材或村庄明确收购的本地低阶物品。

### 村庄寻找入口

起步期需要解决的不是“如何重新设计村庄查找算法”，而是用符合模组气质的方式，把原版已有的村庄定位能力提供给玩家。

阶段层面的结论是：玩家需要先合成商路图册。商路图册承担“找到第一个村庄”的入口职责；在后续阶段，它继续承担“选择下一个村庄”和“管理商路节点”的职责。

商路图册只负责村庄节点与路线规划，不提供当前交易、价格、库存、代表性卖品、市场传闻、供需摘要或利润判断。

这个阶段要成立，需要满足：

- 商路图册合成成本足够低，玩家在早期原版生存中可以自然完成。
- 商路图册的第一次入口定位免费，玩家不需要先完成交易才能找到第一个村庄。
- 贸易站入口容易理解。
- 展示柜能快速告诉玩家这个村庄大致卖什么。
- 传闻板能提示外部市场变化，但不要求玩家立刻精算。
- 第一批交易能自然产生少量周转货币、补给或下一段路线的试探货。

启动收购应优先选择玩家在该村庄附近能够稳定、大量获取的物品，例如木材、作物、燃料、常见副产物或基础建材。收购量可以少，但对象不能是少量难获取的东西，否则起步期会被错误地抬高门槛。

第一批交易不承担“购买第一个村庄线索”的职责。它的核心作用是让玩家在第一个村庄获得一点周转能力：可以买少量补给，也可以承担后续一次低成本普通定位的制图消耗。它不应让玩家连续购买多次指定风格定位，也不应让玩家长期依赖本地采集换钱。

起步期允许采集参与，但采集只负责启动资金和补洞。若玩家长期靠原地采集卖货获得最高收益，则说明路线收益不够明确。

## 阶段二：拓路发现期

拓路发现期的职责不是立刻建立商路，而是让玩家发现足够多的村庄节点。

玩家需要先知道“世界上有哪些村庄、它们大概在哪里、属于什么风格”，才有条件判断哪些村庄之间能形成商路。此阶段的重点是扩张商路图册中的节点图，而不是追求路线利润最大化。

玩家可以通过以下方式扩张节点：

- 使用商路图册进行低成本普通定位。
- 使用更高成本的指定风格定位补齐路线缺口。
- 在旅行中自然发现村庄。
- 与贸易站交互，把已发现村庄升级为更明确的商业节点。

这个阶段的检查重点：

- 玩家不再只知道第一个村庄。
- 玩家能在商路图册中看到多个村庄节点。
- 玩家开始识别不同村庄风格，而不是只追最近村庄。
- 玩家有足够的节点素材，为后续商路构建做准备。
- 此阶段可以出现单次买卖和试探性转卖，但它们还不构成真正的商路。

## 阶段三：商路构建期

商路构建期是跨村跑商真正成型的阶段。

这里的“商路”不是从一个村庄到另一个村庄完成一次转卖。单段运输只是一笔生意。真正的商路需要多个节点之间形成可重复的货物流：有产地、有需求地、有下一段去向，最好还能形成回程货、连续链条或循环。

典型商路包括：

- 平原补给 -> 积雪平原 -> 冰雪货 -> 沙漠 -> 染料 / 建材 -> 热带草原或平原。
- 平原工具 -> 针叶林 / 积雪平原 -> 木炭 / 骨块 / 皮货 -> 平原粮仓或沙漠染坊。
- 针叶林 / 热带草原木炭 -> 沙漠染坊 / 热带草原陶瓦 -> 装饰建材 -> 平原或建造节点。

这一阶段的核心不是额外追求“扩大单笔利润”，因为商路本身就应该具有利润。玩家的成长来自把多个村庄串成稳定路线，而不是只在两个村庄之间找到一次价差。

商路构建期的系统支撑：

- 商路图册允许玩家手动绘制商路边，记录自己的路线理解。
- 贸易站提供真实价格、库存和交易执行。
- 展示柜帮助玩家快速识别村庄代表货。
- 传闻板提供周期行情，让玩家调整已有商路或临时绕路。
- 库存耗尽、收购饱和和面额管理让玩家自然移动。

商路构建期应避免：

- 事件只改变数字，玩家路线不变。
- 高价值货库存过高，导致路线被单村解决。
- 合约变成稳定批发渠道，替代路线本身。
- 玩家只记住某个菜单，而不是记住“哪片区域、哪类主题、哪条路线”。
- 把单次 A 村买入、B 村卖出包装成完整商路。

## 阶段四：终局收束期

终局收束期负责把商路成果导向“理论上仅靠本模组资源链可击杀末影龙”的目标。

这个阶段不替代原版终局动作。玩家仍需要：

- 自己合成末影之眼。
- 寻找要塞。
- 激活末地传送门。
- 进入末地并击杀末影龙。

商路系统只负责提供关键材料来源，尤其是末影珍珠和烈焰棒。

终局材料应通过三类入口出现：

- 全局稀有货架：避免锁档，低库存、高价格。
- 稀有进口事件：制造绕路也值得的短期目标。
- 高价值路线收束合约：用金币和 T4 路线货换取少量关键材料。

终局收束期的底线：

- 不直接出售末影之眼。
- 不让单个村庄或单个周期解决完整开门材料。
- 不让末影珍珠和烈焰棒成为倒卖利润货。
- 不让后期玩法退化成逐村刷新货架。

## 后续检查口径

后续调整主题、价值、库存、事件或合约时，优先反查这些问题：

- 这个改动服务哪个阶段？
- 它是帮助玩家启动、建立路线、扩大收益，还是收束终局？
- 它会不会让采集重新变成长期最优解？
- 它会不会让单村菜单替代跨村路线？
- 它会不会让终局材料变成普通商品？
