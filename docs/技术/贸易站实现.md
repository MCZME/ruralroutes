# 贸易站实现

## 数据结构

### BlockEntity

`TradeStationBlockEntity` 仅存储校验数据，不存储核心业务数据：

| 字段 | 类型 | 说明 |
|---|---|---|
| `villageTheme` | `ResourceLocation` | 主题标识，用于校验 |
| `tradeNodeId` | `UUID` | 节点唯一标识 |

核心业务数据（库存、刷新时间戳）存储在区块的 `CommercialNodeData` 中。

### 数据同步机制

BlockEntity 实现标准数据同步：

- `getUpdateTag()` - 同步 `villageTheme` 到客户端
- `handleUpdateTag()` - 客户端接收更新
- `setVillageTheme()` - 修改后调用 `sendBlockUpdated()` 触发同步

### TradeSlot 设计

`TradeSlot` 是不可交互的展示槽位：

| 字段 | 说明 |
|------|------|
| `itemId` | 物品ID（ResourceLocation） |
| `displayStack` | 物品类型模板 |
| `baseStock` | 原始库存 |
| `pendingCount` | 暂存数量（用于实时扣减显示） |
| `price` | 价格 |
| `isBuy` | true=购买（村庄卖给玩家），false=出售（玩家卖给村庄） |

计算属性：`stockCount = baseStock - pendingCount`

特点：
- `mayPickup()` 返回 false — 不可拿起
- `mayPlace()` 返回 false — 不可放入
- 支持叠加逻辑和实时库存扣减显示

### PendingTradeSlot

暂存区专用槽位，继承 `TradeSlot`：

| 额外字段 | 说明 |
|----------|------|
| `sourceSlotIndex` | 来源槽位索引（-1 表示无） |
| `sourceIsBuy` | 来源是否为出售区 |

用于追踪交易来源，取消时恢复来源槽位的 `pendingCount`。

### 槽位管理

Menu 中槽位独立管理：

```java
List<TradeSlot> sellSlots;           // 出售槽位（村庄卖给玩家）
List<TradeSlot> buySlots;            // 收购槽位（玩家卖给村庄）
List<PendingTradeSlot> pendingSlots; // 暂存区槽位（多物品）
```

## 交易暂存区

### 设计理念

暂存区 = 购物车/订单，一笔交易可以包含多个物品：

- 一笔交易可以**同时包含买入和卖出**物品
- 每个物品独立记录交易类型（买入/卖出）
- 结算时由 `TradeEngine` 统一处理，货币自动补齐

### 交易请求构建

确认时调用 `buildTradeRequest()` 构建 `TradeRequest`：

| 交易类型 | takeItems（玩家获得） | giveItems（玩家付出） |
|---------|---------------------|---------------------|
| 买入物品 | 物品 | 等值货币 |
| 卖出物品 | 等值货币 | 物品 |

`TradeEngine` 要求 `giveItems价值 == takeItems价值`，每次操作都保证此条件。

### 交互逻辑

| 操作 | 行为 |
|------|------|
| 点击出售槽位 | 左侧添加物品卡片，右侧累加货币 |
| 点击收购槽位 | 右侧添加物品卡片，左侧累加货币 |
| 点击左侧物品卡片 | 移除该物品（恢复库存） |
| 点击右侧物品卡片 | 移除该物品（恢复库存） |
| 点击确认 | 构建 TradeRequest，交给 TradeEngine 执行 |

### 显示布局

```
┌─────────────────────────────────────────┐
│ 玩家想要的          玩家支付的           │
│ ┌────┐┌────┐┌────┐  ┌────┐┌────┐┌────┐ │
│ │面包││种子││50铜│  │小麦││铁矿││30铜│ │
│ └────┘└────┘└────┘  └────┘└────┘└────┘ │
│              [确认]                      │
└─────────────────────────────────────────┘
```

**左侧"玩家想要的"**：买入的物品 + 卖出获得的货币总额
**右侧"玩家支付的"**：卖出的物品 + 买入支付的货币总额

**货币合并显示**：
- 左侧货币 = 所有卖出物品的价值总和（显示为一个货币卡片）
- 右侧货币 = 所有买入物品的价值总和（显示为一个货币卡片）

**点击交互**：
- 左右两侧的物品卡片都可点击取消
- 货币卡片不可点击，只是显示总额

## 初始化流程

贸易站方块采用**两阶段初始化**：

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：使用配置工具预置 `themeName`
确定数据：`themeName`

### 阶段二：首次交互（运行时）

执行者：玩家
时机：右键核心方块时
操作：检查并创建区块数据
确定数据：库存数据

## 首次交互流程

```
玩家右键贸易站
    │
    ▼
检查 BlockEntity.themeName 是否有效
    │
    ├─ 无效 → 无响应
    │
    ▼ 有效
检查区块是否已有 CommercialNodeData
    │
    ├─ 有 → 校验 tradeNodeId，打开 GUI
    │
    ▼ 无
检查旁边是否有村庄
    │
    ├─ 无 → 无响应
    │
    ▼ 有
创建 CommercialNodeData：
  - tradeNodeId = UUID.randomUUID()
  - themeName = BlockEntity.themeName
  - sellItems = 从 ThemeTemplate.sellItems 展开
  - buyItems = 从 ThemeTemplate.buyItems 展开
  - stocks = 从 ThemeTemplate 初始化（支持叠加）
  - refreshTimestamp = 当前时间
同步写入 BlockEntity.tradeNodeId（贸易站、展示柜、传闻板）
打开 GUI
```

## 节点同步机制

首次交互时，贸易站会将 `tradeNodeId` 同步到附近的展示柜和传闻板：

- 同步范围：贸易站周围 16 格半径
- 同步对象：`DisplayCaseBlockEntity`、`RumorBoardBlockEntity`
- 同步时机：创建 `CommercialNodeData` 后立即同步

这确保同一建筑内的三个方块共享同一个商业节点。展示柜和传闻板不需要 `villageTheme`，仅通过 `tradeNodeId` 与贸易站关联。

## 数据同步流程

### 打开 GUI 时同步

```
服务端 openMenu()
    │
    ▼
createMenu() 创建 TradeStationMenu
    │
    ▼
Menu 构造时从 CommercialNodeData 初始化槽位
    │
    ▼
openMenu() 返回后发送 TradeSlotSyncPayload
    │
    ▼
客户端收到后更新/重建槽位
    │
    ▼
Screen 渲染
```

### 同步时机

| 场景 | 同步方式 | 网络包 |
|------|----------|--------|
| 打开 GUI | 全量同步所有槽位数据 | TradeSlotSyncPayload |
| 添加到暂存区 | 增量同步 | PendingTradeSyncPayload |
| 移除暂存区物品 | 增量同步 | PendingTradeSyncPayload |
| 清空暂存区 | 增量同步 | PendingTradeSyncPayload |
| 贸易周期刷新 | 全量同步（待实现） | TradeSlotSyncPayload |

### 网络包设计

#### TradeSlotSyncPayload（服务端→客户端）

全量同步所有槽位：

| 字段 | 说明 |
|------|------|
| `containerId` | 菜单 ID |
| `sellSlotCount` | 出售槽位数量 |
| `buySlotCount` | 收购槽位数量 |
| `sellSlots` | 出售槽位数据列表 |
| `buySlots` | 收购槽位数据列表 |

#### PendingTradeSyncPayload（服务端→客户端）

增量同步暂存区：

| 字段 | 说明 |
|------|------|
| `containerId` | 菜单 ID |
| `isBuyTrade` | 保留字段（兼容性） |
| `pendingSlots` | 暂存槽位数据列表（包含每个槽位的 isBuy 标志） |
| `pendingCountMap` | 来源槽位 pendingCount 更新 |

#### TradeRequestPayload（客户端→服务端）

交易请求：

| 字段 | 说明 |
|------|------|
| `containerId` | 菜单 ID |
| `requestType` | 请求类型：ADD_BUY=0, ADD_SELL=1, REMOVE_ENTRY=2, CLEAR=3, REMOVE_BUY=4, REMOVE_SELL=5 |
| `slotIndex` | 槽位索引 |
