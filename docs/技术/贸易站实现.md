# 贸易站实现

## 数据结构

### BlockEntity

`TradeStationBlockEntity` 仅存储校验数据，不存储核心业务数据：

| 字段 | 类型 | 说明 |
|---|---|---|
| `villageTheme` | `ResourceLocation` | 主题标识，用于校验 |
| `tradeNodeId` | `UUID` | 节点唯一标识 |

核心业务数据（库存、刷新时间戳）存储在区块的 `CommercialNodeData` 中。

### 数据同步机制

BlockEntity 实现标准数据同步：

- `getUpdateTag()` - 同步 `villageTheme` 到客户端
- `handleUpdateTag()` - 客户端接收更新
- `setVillageTheme()` - 修改后调用 `sendBlockUpdated()` 触发同步

## 初始化流程

贸易站方块采用**两阶段初始化**：

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：使用配置工具预置 `themeName`
确定数据：`themeName`

### 阶段二：首次交互（运行时）

执行者：玩家
时机：右键核心方块时
操作：检查并创建区块数据
确定数据：库存数据

## 首次交互流程

```
玩家右键贸易站
    │
    ▼
检查 BlockEntity.themeName 是否有效
    │
    ├─ 无效 → 无响应
    │
    ▼ 有效
检查区块是否已有 CommercialNodeData
    │
    ├─ 有 → 校验 tradeNodeId，打开 GUI
    │
    ▼ 无
检查旁边是否有村庄
    │
    ├─ 无 → 无响应
    │
    ▼ 有
创建 CommercialNodeData：
  - tradeNodeId = UUID.randomUUID()
  - themeName = BlockEntity.themeName
  - stocks = 从 ThemeTemplate 初始化
  - specialties = 从 ThemeTemplate 生成特产列表
  - refreshTimestamp = 当前时间
同步写入 BlockEntity.tradeNodeId（贸易站、展示柜、传闻板）
打开 GUI
```

## 节点同步机制

首次交互时，贸易站会将 `tradeNodeId` 同步到附近的展示柜和传闻板：

- 同步范围：贸易站周围 16 格半径
- 同步对象：`DisplayCaseBlockEntity`、`RumorBoardBlockEntity`
- 同步时机：创建 `CommercialNodeData` 后立即同步

这确保同一建筑内的三个方块共享同一个商业节点。展示柜和传闻板不需要 `villageTheme`，仅通过 `tradeNodeId` 与贸易站关联。
