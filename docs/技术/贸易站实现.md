# 贸易站实现

## 数据结构

### BlockEntity

`TradeStationBlockEntity` 存储以下数据：

| 字段 | 类型 | 说明 |
|---|---|---|
| `villageTheme` | `ResourceLocation` | 主题标识 |
| `villageId` | `UUID` | 村庄 ID |
| `stock` | `Map<ResourceLocation, Integer>` | 各物品库存 |

### 数据同步机制

BlockEntity 实现标准数据同步：

- `getUpdateTag()` - 同步 `villageTheme` 到客户端
- `handleUpdateTag()` - 客户端接收更新
- `setVillageTheme()` - 修改后调用 `sendBlockUpdated()` 触发同步

## 初始化流程

贸易站方块采用**两阶段初始化**：

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：使用配置工具预置 `themeName`
确定数据：`themeName`

### 阶段二：首次交互（运行时）

执行者：玩家
时机：右键核心方块时
操作：检查并创建区块数据
确定数据：库存数据

## 首次交互流程

```
玩家右键贸易站
    │
    ▼
检查 NBT 中 themeName 是否有效
    │
    ├─ 无效 → 无响应（方块不工作）
    │
    ▼ 有效
检查所在区块是否已有商业节点数据
    │
    ├─ 已有 → 打开 GUI
    │
    ▼ 没有
判断旁边是否有村庄
    │
    ├─ 有 → 创建区块数据，写入 themeName，按主题 stock 规则初始化库存
    │
    └─ 无 → 待定（见待办）
```
