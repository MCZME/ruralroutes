# 贸易站实现

## 数据结构

### BlockEntity

`TradeStationBlockEntity` 仅存储校验数据，不存储核心业务数据：

| 字段 | 类型 | 说明 |
|---|---|---|
| `villageTheme` | `ResourceLocation` | 主题标识，用于校验 |
| `tradeNodeId` | `UUID` | 节点唯一标识 |

核心业务数据（库存、刷新时间戳）存储在区块的 `CommercialNodeData` 中。

### 数据同步机制

BlockEntity 实现标准数据同步：

- `getUpdateTag()` - 同步 `villageTheme` 到客户端
- `handleUpdateTag()` - 客户端接收更新
- `setVillageTheme()` - 修改后调用 `sendBlockUpdated()` 触发同步

### TradeSlot 设计

`TradeSlot` 是不可交互的展示槽位：

| 字段 | 说明 |
|------|------|
| `displayStack` | 物品类型模板（始终 count=1） |
| `stockCount` | 当前库存（独立变量，不修改 ItemStack） |
| `price` | 价格（预留接口） |

特点：
- `mayPickup()` 返回 false — 不可拿起
- `mayPlace()` 返回 false — 不可放入
- 数量由 `stockCount` 独立管理，支持叠加逻辑

### 槽位管理

Menu 中槽位独立管理，支持动态数量：

```java
List<TradeSlot> sellSlots;  // 出售槽位（村庄卖给玩家）
List<TradeSlot> buySlots;   // 收购槽位（玩家卖给村庄）
```

### 布局参数

| 参数 | 值 |
|------|-----|
| GUI 尺寸 | 320×240px |
| 槽位大小 | 18×18px |
| 槽位间距 | 2px |
| 布局方式 | 双排蛇形（奇数上排，偶数下排） |

## 初始化流程

贸易站方块采用**两阶段初始化**：

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：使用配置工具预置 `themeName`
确定数据：`themeName`

### 阶段二：首次交互（运行时）

执行者：玩家
时机：右键核心方块时
操作：检查并创建区块数据
确定数据：库存数据

## 首次交互流程

```
玩家右键贸易站
    │
    ▼
检查 BlockEntity.themeName 是否有效
    │
    ├─ 无效 → 无响应
    │
    ▼ 有效
检查区块是否已有 CommercialNodeData
    │
    ├─ 有 → 校验 tradeNodeId，打开 GUI
    │
    ▼ 无
检查旁边是否有村庄
    │
    ├─ 无 → 无响应
    │
    ▼ 有
创建 CommercialNodeData：
  - tradeNodeId = UUID.randomUUID()
  - themeName = BlockEntity.themeName
  - sellItems = 从 ThemeTemplate.sellItems 展开
  - buyItems = 从 ThemeTemplate.buyItems 展开
  - stocks = 从 ThemeTemplate 初始化（支持叠加）
  - refreshTimestamp = 当前时间
同步写入 BlockEntity.tradeNodeId（贸易站、展示柜、传闻板）
打开 GUI
```

## 节点同步机制

首次交互时，贸易站会将 `tradeNodeId` 同步到附近的展示柜和传闻板：

- 同步范围：贸易站周围 16 格半径
- 同步对象：`DisplayCaseBlockEntity`、`RumorBoardBlockEntity`
- 同步时机：创建 `CommercialNodeData` 后立即同步

这确保同一建筑内的三个方块共享同一个商业节点。展示柜和传闻板不需要 `villageTheme`，仅通过 `tradeNodeId` 与贸易站关联。

## 数据同步流程

### 打开 GUI 时同步

```
服务端 openMenu()
    │
    ▼
createMenu() 创建 TradeStationMenu
    │
    ▼
Menu 构造时从 CommercialNodeData 初始化槽位
    │
    ▼
openMenu() 返回后发送 TradeSlotSyncPayload
    │
    ▼
客户端收到后更新/重建槽位
    │
    ▼
Screen 渲染
```

### 同步时机

| 场景 | 同步方式 |
|------|----------|
| 打开 GUI | 全量同步所有槽位数据 |
| 交易发生 | 增量同步变化的槽位（待实现） |
| 贸易周期刷新 | 全量同步（待实现） |

### 网络包设计

`TradeSlotSyncPayload` 包含：

| 字段 | 说明 |
|------|------|
| `containerId` | 菜单 ID |
| `sellSlotCount` | 出售槽位数量 |
| `buySlotCount` | 收购槽位数量 |
| `sellSlots` | 出售槽位数据列表 |
| `buySlots` | 收购槽位数据列表 |

每个 `SlotData` 包含：`slotIndex`、`displayStack`、`stockCount`、`price`
