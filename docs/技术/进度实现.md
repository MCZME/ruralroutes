# 进度实现

## 概览

模组进度通过 NeoForge datagen 生成，不手写 advancement JSON。

当前实现由三部分组成：

- datagen 生成进度树与多 criterion 旅行线
- 自定义 trigger 承接模组交互事件
- 轻量玩家状态记录承接交易次数与特产累计
- 主题旅行线从内置主题定义中自动收集 criterion，不再手动维护主题 ID 名单

## 文件入口

| 文件 | 作用 |
| :--- | :--- |
| `RRAdvancementProvider` | 生成整棵 advancement 树 |
| `RRAdvancementKeys` | 统一管理进度 ID |
| `OpenTradeStationTrigger` | 模组自定义 trigger |
| `RRPlayerProgressState` | 玩家累计进度状态 |
| `RRCriteriaTriggers` | trigger 注册入口 |

生成结果输出到：

- `src/generated/resources/data/ruralroutes/advancement/`

## Trigger 分层

### 原版 trigger

以下节点直接复用原版条件：

- `进入村庄`：`minecraft:location` + `#minecraft:village`
- `获得铜币 / 获得铁币 / 获得金币`：`inventory_changed`

### 自定义 trigger

`OpenTradeStationTrigger` 当前承接以下事件：

- `open`
- `first_trade`
- `fixed_trade`
- `coin_exchange`
- `open_rumor_board`
- `open_display_case`
- `buy_specialty`
- `collector`
- `discover_village_style`
- `discover_village_theme`
- `trade_10_times`
- `trade_100_times`
- `big_spender`

其中旅行线使用带参数的条件匹配：

- `discover_village_style` + `village_style`
- `discover_village_theme` + `village_theme`

## 事件挂点

### 贸易站

`TradeStationBlock` 负责：

- 首次成功打开贸易站时触发 `open`
- 读取当前 `themeName`
- 根据主题触发一次 `discover_village_style`
- 根据主题触发一次 `discover_village_theme`

### 交易菜单

`TradeStationMenu` 负责：

- 成功完成常规交易时触发 `first_trade`
- 成功完成固定合约交易时触发 `fixed_trade`
- 成功购买当前村庄特产时触发 `buy_specialty`
- 成功完成货币兑换时触发 `coin_exchange`
- 更新交易次数与特产累计
- 在阈值满足时触发 `collector / trade_10_times / trade_100_times / big_spender`

### 展示柜与传闻板

- `DisplayCaseBlock` 负责首次打开展示柜事件
- `RumorBoardBlock` 负责首次打开传闻板事件

## 玩家累计状态

`RRPlayerProgressState` 当前只保存两类需要跨事件累计的数据：

- `successfulTradeCount`
- `purchasedSpecialties`

用途如下：

- `熟客 / 老主顾`：统计成功常规交易次数
- `收藏家`：统计购买过的不同特产

旅行线不依赖 attachment 去重，而是完全交给 advancement 自己的 criterion 完成状态处理。

## 旅行线的累计显示

`走南闯北` 与 `识遍乡野` 采用与原版 `adventuring_time` 相同的思路：

- 为每个村庄风格或主题单独生成一个 criterion
- `requirements` 中每组只放一个 criterion
- 客户端据此显示 `x/5` 与 `x/N`

其中：

- `5` 个 `VillageStyle`
- 所有由 `ThemeDataProvider` 定义的内置主题 ID

其中主题 ID 通过 `ThemeDataProvider.collectBuiltinThemeIds()` 自动发现，因此新增或删除内置主题时，旅行线 criterion 会自动跟随变化。

### 对外部 JSON 主题的容错边界

- 外部 JSON 主题只要能被 `ThemeManager` 正常加载，就可以参与实际玩法
- 旅行线不会把外部主题自动纳入 `识遍乡野`
- 若外部主题使用了无法映射到当前 `VillageStyle` 的 biome，`走南闯北` 会跳过该次风格统计，只记日志，不会误算为 `plains`

需要人工同步的只剩：

- 对应语言文件中的主题文案
- 若设计上调整旅行线命名或阶段含义，更新系统文档与进度文案

## 当前实现范围

当前已落地的树包含：

- root
- 主线：贸易站、第一笔生意、以物易物、货币兑换
- 信息支线：传闻板、展示柜、购买特产、收藏家
- 货币支线：铜币、铁币、金币、一掷千金
- 旅行线：走南闯北、识遍乡野
- 挑战线：熟客、老主顾

## 验证方式

每次改动后至少运行：

- `./gradlew.bat compileJava`
- `./gradlew.bat runData`

前者用于确认触发器、挂点与状态结构可编译，后者用于确认 advancement JSON 与语言文件正常生成。
