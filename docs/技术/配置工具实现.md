# 配置工具实现

## 用途

配置工具是模组开发者使用的工具，用于查看核心方块信息和配置贸易站主题。

## 支持的方块

| 方块 | BlockEntity | 功能 |
|------|-------------|------|
| 贸易站 | `TradeStationBlockEntity` | 可编辑主题 + 自动复制节点ID |
| 展示柜 | `DisplayCaseBlockEntity` | 可粘贴节点ID |
| 传闻板 | `RumorBoardBlockEntity` | 可粘贴节点ID |

**注意**：展示柜和传闻板不需要主题，仅通过 `tradeNodeId` 与贸易站关联。

## 使用场景

### 贸易站（可编辑）

开发者在制作贸易站结构模板时：

1. 放置贸易站方块
2. 使用配置工具右键方块
3. 在 GUI 中选择主题
4. 点击应用按钮
5. 保存结构模板（.nbt）

**自动复制**：右键贸易站时，自动将其节点ID复制到配置工具的剪贴板。

### 展示柜/传闻板（可粘贴节点ID）

开发者手动关联节点ID：

1. 右键贸易站复制其节点ID到配置工具
2. 右键展示柜或传闻板打开 GUI
3. GUI 显示"粘贴节点ID"按钮（如有复制的节点ID）
4. 点击粘贴按钮设置节点ID

**用途**：修复断开关联、手动配置、结构模板制作、测试调试。

## 实现结构

### 物品注册

- 物品类：`ConfigToolItem`
- 注册名：`config_tool`
- 位于创造模式标签页

### GUI 结构

| 类 | 职责 |
|---|---|
| `ConfigToolMenu` | 菜单逻辑，区分方块类型，提供 `canEditTheme()` 方法 |
| `ConfigToolScreen` | 根据方块类型显示不同 UI |

### 数据流

```
配置工具右键 → 打开 GUI
       ↓
判断方块类型
       ├─ 贸易站 → 显示主题列表 + 应用/取消按钮
       └─ 展示柜/传闻板 → 仅显示节点ID + 关闭按钮
       ↓
（仅贸易站）选择主题 → 应用
       ↓
ConfigToolApplyPayload → 服务端
       ↓
TradeStationBlockEntity.setVillageTheme(theme)
       ↓
sendBlockUpdated() → 同步到客户端
       ↓
保存结构模板 → themeName 写入 .nbt
```

## 关键实现细节

### Menu 方块类型判断

```java
public enum BlockType {
    TRADE_STATION,
    DISPLAY_CASE,
    RUMOR_BOARD,
    UNKNOWN
}

public boolean canEditTheme() {
    return getBlockType() == BlockType.TRADE_STATION;
}
```

### 网络包仅处理贸易站

`ConfigToolApplyPayload` 服务端仅处理 `TradeStationBlockEntity`：

```java
if (level.getBlockEntity(payload.blockPos) instanceof TradeStationBlockEntity station) {
    station.setVillageTheme(payload.themeName);
}
```
