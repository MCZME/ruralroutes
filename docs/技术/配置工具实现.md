# 配置工具实现

## 用途

配置工具是模组开发者使用的工具，用于在 .nbt 结构模板中预置贸易站的 `themeName`。

## 使用场景

开发者在制作贸易站结构模板时：

1. 放置贸易站方块
2. 使用配置工具右键方块
3. 在 GUI 中选择主题
4. 点击应用按钮
5. 保存结构模板（.nbt）

## 实现结构

### 物品注册

- 物品类：`ConfigToolItem`
- 注册名：`config_tool`
- 位于创造模式标签页

### GUI 结构

| 类 | 职责 |
|---|---|
| `ConfigToolMenu` | 菜单逻辑，存储 blockPos、可选主题列表 |
| `ConfigToolScreen` | 客户端渲染，显示主题按钮列表 |

### 数据流

```
配置工具右键 → 打开 GUI → 选择主题 → 应用
       ↓
BlockEntity.setVillageTheme(theme)
       ↓
sendBlockUpdated() → 同步到客户端
       ↓
保存结构模板 → themeName 写入 .nbt
```

## 关键实现细节

### BlockEntity 数据同步

`TradeStationBlockEntity` 实现标准数据同步机制：

- `getUpdateTag()` - 将 `villageTheme` 写入更新包
- `handleUpdateTag()` - 客户端接收并更新
- `setVillageTheme()` - 修改后调用 `sendBlockUpdated()` 触发同步

### Menu 实现

- 只传递 `blockPos`（通过 `FriendlyByteBuf`）
- `getCurrentTheme()` 直接从客户端 BlockEntity 读取（已同步）
- 服务端执行 `applyTheme()` 写入数据
