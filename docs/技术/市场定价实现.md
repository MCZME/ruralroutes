# 市场定价实现

## 背景

贸易站价格系统需要组合基础价值、主题修正和市场因子，同时保证 GUI 展示价格与服务端执行价格使用同一套规则。

本轮重构后，定价输入已经不再只是“一个裸 `ItemStack`”，而是：

- 展示栈本身
- 该物品在候选中的 `sourceKey`
- 若存在组件，则通过 `TradeItemKey` 参与匹配

## 核心原则

- **服务端权威计算**
- **单一价格来源**
- **不信任客户端**
- **库存修正与价格修正分离**
- **支持组件感知与来源键匹配**

## 定价公式

出售价：

```text
sellPrice = round(baseValue * sellModifier * marketFactor)
```

收购价：

```text
buyPrice = round(baseValue * buyModifier * marketFactor)
price = max(1, calculatedPrice)
```

`TradePricingService` 只负责输出整数价格，不负责决定最终使用哪些货币物品。实体货币报价由货币篮逻辑后续生成。

## TradePricingService

当前对外接口：

```java
public static TradePrice calculate(
    ServerLevel level,
    CommercialNodeData nodeData,
    ItemStack stack,
    TradeSide side
);

public static TradePrice calculate(
    ServerLevel level,
    CommercialNodeData nodeData,
    ItemStack stack,
    TradeSide side,
    Optional<String> sourceKey
);
```

### 计算流程

1. `ValueTableManager.queryBaseValue(stack)` 获取基础价值
2. `resolveThemeModifier(...)` 获取主题修正
3. `resolveMarketAdjustment(...)` 获取市场修正
4. 组合得出最终价格

## 主题修正匹配

`TradePricingService` 会把展示栈转换成 `TradeItemKey`，再调用：

```java
ThemePriceModifierResolver.resolve(template, itemKey, sourceKey)
```

支持四种目标：

- 精确组件物品
- `@source_key`
- 裸物品 ID
- 标签

匹配优先级：

1. 带组件精确匹配
2. `sourceKey`
3. 物品 ID
4. 标签

这意味着同一个 `minecraft:book`：

- 可以因自定义名称不同而得到不同主题修正
- 也可以因来自不同候选组而得到不同主题修正

## 市场修正匹配

市场价格调整同样通过 `TradeItemKey + sourceKey` 参与匹配：

```java
MarketStateResolver.resolvePriceAdjustment(
    marketState,
    context,
    TradeItemKey.from(stack),
    sourceKey
)
```

`MarketStateResolver` 现在会同时处理：

- 精确组件目标
- `@source_key`
- 物品 ID
- 标签

价格链只读取市场事件的 `delta`，不会把 `stock` 字段带回报价。

## MarketContext

当前市场上下文只保留事件作用域匹配所需字段：

```java
public record MarketContext(
    Optional<ResourceLocation> themeId,
    Optional<ResourceLocation> biomeId
)
```

来源：

- `themeId` 直接来自 `CommercialNodeData.themeName`
- `biomeId` 通过 `ThemeManager.getTheme(themeName)` 返回的 `ResolvedTheme.biome()` 获得

## GUI 与服务端执行的一致性

贸易站菜单初始化槽位时，会把每个 `NodeTradeEntry` 的：

- `displayStack`
- `sourceKey`

一起传入 `TradePricingService`。

这保证：

- GUI 卡片展示价格
- 契约匹配时的动态货币篮价格
- 服务端执行时的权威价格

都使用同一套输入语义。

## 与库存链的边界

需要明确区分两条链：

- **价格链**：`baseValue * themeModifier * marketFactor`
- **库存链**：节点创建/周期刷新时的库存初始化，以及交易执行后的实时消耗

即使市场事件同时命中价格与库存：

- 价格链只读 `delta`
- 库存链只读 `stock`
