# 市场定价实现

## 背景

贸易站价格系统需要组合基础价值、主题修正和市场因子。核心问题是确保 GUI 显示价格与服务端执行价格一致。

## 核心原则

- **服务端权威计算**：价格必须由服务端计算，客户端不自行计算
- **单一价格来源**：GUI 展示和服务端执行使用同一个定价服务
- **不信任客户端**：交易确认时服务端重新计算权威价格

## 定价公式

出售价（村庄卖给玩家）：

```text
sellPrice = round(baseValue * sellModifier * marketFactor)
```

收购价（玩家卖给村庄）：

```text
buyPrice = round(baseValue * buyModifier * marketFactor)
price = max(1, calculatedPrice)
```

`TradePricingService` 只负责输出当前市场下的整数价格，不负责决定最终使用哪些货币物品。实体货币报价由动态货币篮契约在定价之后生成。

## 核心类设计

### TradeSide

交易方向枚举，用于选择主题修正系数：

```java
public enum TradeSide {
    SELL_TO_PLAYER,    // 村庄出售给玩家，使用 sellModifier
    BUY_FROM_PLAYER    // 村庄从玩家收购，使用 buyModifier
}
```

### TradePrice

价格计算结果记录：

```java
public record TradePrice(
    int baseValue,                       // 基础价值
    float themeModifier,                 // 主题修正系数
    MarketPriceAdjustment marketAdjustment,  // 市场调整
    int finalPrice                       // 最终价格
)
```

### TradePricingService

统一定价服务入口：

```java
public final class TradePricingService {
    public static TradePrice calculate(
        ServerLevel level,
        CommercialNodeData nodeData,
        ItemStack stack,
        TradeSide side
    );
    
    public static int calculateFinalPrice(
        ServerLevel level,
        CommercialNodeData nodeData,
        ItemStack stack,
        TradeSide side
    );
}
```

计算流程：
1. 获取基础价值 `ValueTableManager.queryBaseValue(stack)`
2. 获取主题修正 `ThemePriceModifierResolver.resolve(template, stack)`
3. 获取市场调整 `MarketStateResolver.resolvePriceAdjustment(state, context, stack)`
4. 计算最终价格 `round(baseValue * themeModifier * marketFactor)`
5. 应用下限 `max(1, finalPrice)`

动态货币篮交易会在第 5 步之后，将 `finalPrice` 交给 `CurrencyBasketComposer`，生成明确的铜板、铁币、金币物品报价。

### ThemePriceModifierResolver

主题价格修正缓存解析器：

```java
public final class ThemePriceModifierResolver {
    public static ThemeTemplate.PriceModifier resolve(
        ThemeTemplate template,
        ItemStack stack
    );
    
    public static void invalidate();
}
```

解析规则：
1. 精确物品修正优先（如 `minecraft:bread`）
2. 标签修正其次（如 `#ruralroutes:pool/crop`）
3. 多标签命中时使用第一个匹配项
4. 无匹配时返回默认修正 (sell=1.0, buy=1.0)

缓存失效：
- 主题 reload 后清空
- 标签 reload 后清空（`TagsUpdatedEvent`）

## 交易执行流程

确认交易时，服务端直接从 `pendingSlots` 读取数据：

```
客户端 TradeRequestPayload(CONFIRM)
    ↓
服务端 TradeStationMenu.executeTrade()
    ↓
TradeContractExecutor.executeMixedTrade(level, nodeData, player, pendingSlots, cycleIndex, blockPos)
    ↓
验证周期一致性
    ↓
遍历 pendingSlots，重新计算每个物品的权威价格
    ↓
验证价值匹配、库存
    ↓
原子执行转移
```

**关键设计**：不需要发送订单数据到服务端，服务端已有 `pendingSlots`。

## 周期变化处理

- 打开 GUI 时记录当前周期索引
- 确认交易时验证周期是否一致
- 跨周期时返回 `CYCLE_CHANGED` 失败，刷新 GUI
