# 传闻板实现

## 数据结构

### BlockEntity

`RumorBoardBlockEntity` 仅存储节点ID，不存储主题：

| 字段 | 类型 | 说明 |
|---|---|---|
| `tradeNodeId` | `UUID` | 节点唯一标识，由贸易站同步 |

情报数据存储在区块的 `CommercialNodeData` 中（第二阶段实现）。

**注意**：传闻板不需要 `villageTheme`，仅通过 `tradeNodeId` 与贸易站关联。

### 数据同步机制

BlockEntity 实现标准数据同步：

- `getUpdateTag()` - 同步 `villageTheme` 到客户端
- `handleUpdateTag()` - 客户端接收更新
- `setVillageTheme()` - 修改后调用 `sendBlockUpdated()` 触发同步

## 初始化流程

传闻板不需要预置主题，仅通过贸易站自动同步节点ID：

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：放置传闻板方块即可，无需配置
确定数据：无

### 阶段二：首次交互（运行时）

执行者：贸易站（自动同步）
时机：玩家首次激活贸易站时
操作：贸易站创建区块数据后同步 `tradeNodeId` 到附近传闻板
确定数据：`tradeNodeId`

## 交互流程

```
玩家右键传闻板
    │
    ▼
检查区块是否已有 CommercialNodeData
    │
    ├─ 无 → 提示"请先激活贸易站"
    │
    ▼ 有
校验 tradeNodeId（如已同步）
    │
    ├─ 不匹配 → 提示"数据不匹配"
    │
    ├─ 未同步（null） → 允许通过，等待贸易站同步
    │
    ▼ 匹配或未同步
显示占位信息"商路消息将在下一个周期到来"
```

## 第一阶段功能

- 传闻板方块实体数据结构完成
- 校验交互流程完成
- 显示占位信息
- GUI 界面和情报系统留待第二阶段实现

## 第二阶段规划

- 实现情报生成逻辑（基于市场波动）
