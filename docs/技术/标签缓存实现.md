## 背景

本模组在主题模板、价值表、商业节点生成、市场事件规则中广泛使用标签引用，例如：

```text
#ruralroutes:pool/crop
#minecraft:axes
```

NeoForge 标签来自数据包，支持 reload。市场系统第二阶段继续使用 `targetRef` 保存标签引用，因此需要明确标签查询、缓存和失效策略。

## NeoForge 相关约束

根据 NeoForge 1.21.1 标签文档：

- 标签通过数据包定义和加载。
- 标签内容可能在资源重载后变化。
- 频繁获取标签内容时，应缓存标签 entries。
- 标签缓存应在 `TagsUpdatedEvent` 发生时失效。

因此，任何缓存后的"标签 -> 物品集合"结果都不能跨 reload 假定稳定。

## 使用边界

### 可以直接查询的场景

单次物品命中检查可以直接使用：

```java
ItemStack.is(TagKey<Item>)
```

例如市场价格解析时判断某个 `ItemStack` 是否命中 `#ruralroutes:pool/crop`，不需要先展开整个标签。

### 需要缓存的场景

以下场景应使用统一缓存：

- 同一 tick 或同一 GUI 多次检查同一个标签。
- 需要从标签列出所有物品。
- 调试界面、传闻预览、配置工具需要展示标签内容。
- 市场规则校验需要批量检查标签是否为空。
- 后续主题、库存、市场系统出现重复的标签展开逻辑。

## 设计原则

- 市场事件规则加载时只校验 `targetRef` 字符串格式。
- 市场事件生成时不展开标签为具体物品。
- `MarketEvent.targetRef` 保留原始 `#namespace:path` 引用。
- 价格解析阶段再判断物品是否命中标签。
- 所有"标签 -> 物品集合"缓存必须集中管理。
- 缓存必须监听 `TagsUpdatedEvent` 并清空。
- 不允许多个系统各自维护无法统一失效的标签缓存。

## 实现方案

### TagLookupCache 工具类

**文件**: `core/util/TagLookupCache.java`

```java
public final class TagLookupCache {
    /** 缓存：标签引用字符串 -> 物品集合 */
    private static final Map<String, Set<Item>> TAG_CACHE = new ConcurrentHashMap<>();

    /**
     * 判断物品栈是否匹配引用
     * @param stack 物品栈
     * @param itemRef 引用字符串，如 "#minecraft:axes" 或 "minecraft:bread"
     */
    public static boolean matchesItem(ItemStack stack, String itemRef);

    /**
     * 获取引用下的所有物品
     * @param ref 引用字符串，如 "#minecraft:axes" 或 "minecraft:bread"
     * @return 不可变的物品集合
     */
    public static Set<Item> getItems(String ref);

    /**
     * 解析引用为 ResourceLocation（去除 # 前缀）
     */
    public static ResourceLocation parseRef(String ref);

    /**
     * 判断引用是否为标签引用
     */
    public static boolean isTagRef(String ref);

    /**
     * 清空所有缓存（在 TagsUpdatedEvent 时调用）
     */
    public static void invalidate();
}
```

**设计要点**：

- `matchesItem()` 对于标签引用使用 `ItemStack.is(TagKey)` 检查，无需展开
- `getItems()` 展开标签时缓存结果，返回不可变集合
- 使用 `ConcurrentHashMap` 保证线程安全
- 空标签或无效引用记录警告日志，返回空集合

### 事件监听

**文件**: `RuralRoutes.java`

```java
NeoForge.EVENT_BUS.addListener(this::onTagsUpdated);

private void onTagsUpdated(TagsUpdatedEvent event) {
    TagLookupCache.invalidate();
}
```

## 对各系统的影响

### CommercialNodeManager

- `generateSellItems()` 和 `generateBuyItems()` 使用 `TagLookupCache.getItems()` 展开标签
- `addItemsToStocks()` 使用统一工具展开标签
- 消除了原有的重复标签展开代码

### MarketEventRuleCatalog

- 只校验 `targetRef` 字符串格式。
- 不在规则加载时展开标签。
- 不因为标签当前为空就永久拒绝规则。

### MarketStateGenerator

- 只把规则展开为 `MarketEvent`。
- 不把标签目标拆成多个物品事件。
- 不缓存标签内容。

### MarketStateResolver

- 使用 `TagLookupCache.matchesItem()` 判断命中。
- 返回的 `matchedEvents` 仍然保留原始 `targetRef`。

## 设计决策

| 问题 | 决策 |
|------|------|
| 工具类命名 | `TagLookupCache`（通用命名，可用于所有标签查询场景） |
| 空标签处理 | 记录警告日志，返回空集合（不阻止规则加载） |
| 代码重构 | 统一改造 `CommercialNodeManager` 中的标签展开代码 |
| 调试命令 | 暂不添加，后续如有需要再考虑 |
