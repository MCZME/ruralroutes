# 展示柜实现

## 设计概要

展示柜展示村庄特产，一个展示柜只能展示一个物品。

**核心规则：**
- 一个周期内展示物品不变
- 展示柜初始化在贸易站初始化之后
- 同步范围沿用贸易站的 16 格半径

## 数据结构

### CommercialNodeData 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `specialties` | `List<NodeTradeEntry>` | 特产列表（主题特产 + 随机特产） |

特产同时加入 `sellItems` 和 `stocks`。

### DisplayCaseBlockEntity 数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `tradeNodeId` | `UUID` | 节点唯一标识（已有） |
| `tradeStationPos` | `BlockPos` | 贸易站位置（已有） |
| `displayItem` | `ItemStack` | 展示的物品（新增） |

## 特产生成逻辑

### 生成时机

创建 `CommercialNodeData` 时一次性生成。

### 生成流程

```
创建节点数据
    │
    ├─ 主题特产：从 ResolvedTheme.themeSpecialtyItems 获取
    │
    ├─ 随机特产：从标签 #ruralroutes:pool/specialty 随机抽取
    │           抽取数量：random(1, 3)
    │
    ▼
合并为 specialties 列表
同时加入 sellItems 和 stocks
```

### 随机特产抽取规则

- 来源标签：`#ruralroutes:pool/specialty`
- 抽取数量：`1 + random.nextInt(3)`（1-3 种）
- 不排除已在主题特产中的物品
- 允许抽到少量全局稀有特产（如末影珍珠、烈焰棒）；这类物品依靠抽取权重与库存控制稀缺感

## 展示柜分配逻辑

### 分配时机

贸易站首次交互时，`syncNodeIdToNearbyBlocks` 中执行。

### 分配流程

```
遍历 16 格半径内的展示柜
    │
    ├─ 前 N 个展示柜（N = specialties.size）
    │   → 依次分配 specialties[i]
    │   → 直接使用 NodeTradeEntry.displayStackOrDefault()
    │
    ├─ 多余展示柜
    │   → 从 stocks 中随机选一个有库存的物品
    │   → 从 TradeItemKey 还原展示栈
    │
    ▼
设置 displayItem，同步到客户端
```

### 分配示例

```
村庄特产：[小麦, 胡萝卜]（N = 2）
展示柜数量：4

展示柜1 → 小麦（特产）
展示柜2 → 胡萝卜（特产）
展示柜3 → 钻石（库存随机）
展示柜4 → 铁锭（库存随机）
```

## 周期刷新

贸易周期刷新时：
1. 重新生成 `specialties`
2. 重新分配展示柜 `displayItem`

## 初始化流程

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：放置展示柜方块即可，无需配置

### 阶段二：首次交互（运行时）

执行者：贸易站（自动同步）
时机：玩家首次激活贸易站时
操作：
1. 贸易站创建区块数据，生成 specialties
2. 遍历附近展示柜，分配 displayItem
3. 同步 `tradeNodeId` 和 `tradeStationPos` 到展示柜

## 交互流程

```
玩家右键展示柜
    │
    ▼
检查区块是否有 CommercialNodeData
    │
    ├─ 无 → 提示"请先激活贸易站"
    │
    ▼ 有
校验 tradeNodeId
    │
    ├─ 不匹配 → 提示"数据不匹配"
    │
    ▼ 匹配
打开 GUI（纯展示）
```

## GUI 设计

**定位：** 纯展示，无交易功能。

**展示内容：**
- 物品图标
- 物品名称
- 物品 tooltip（原版风格）

**布局：** 单物品展示，待细化。

## 数据同步

继承 `TradeNodeBlockEntity`，新增 `displayItem` 同步：

- `getUpdateTag()` - 同步 `tradeNodeId`、`tradeStationPos`、`displayItem`
- `handleUpdateTag()` - 客户端接收更新
- `setDisplayItem()` - 修改后调用 `sendBlockUpdated()` 触发同步
