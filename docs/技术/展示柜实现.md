# 展示柜实现

## 设计概要

展示柜展示村庄当前出售商品中的随机物品，一个展示柜只能展示一个物品。

**核心规则：**
- 一个周期内展示物品不变
- 展示柜初始化在贸易站初始化之后
- 同步范围沿用贸易站的 16 格半径

## 数据结构

### CommercialNodeData 读取来源

展示柜当前直接读取：

- `sellItems`
- `tradeNodeId`
- `tradeStationPos`

### DisplayCaseBlockEntity 数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `tradeNodeId` | `UUID` | 节点唯一标识（已有） |
| `tradeStationPos` | `BlockPos` | 贸易站位置（已有） |
| `displayItem` | `ItemStack` | 展示的物品（新增） |

## 展示商品来源

### 生成时机

创建或刷新 `CommercialNodeData` 后，在展示柜分配阶段从当期 `sellItems` 中随机选择。

### 选择流程

```
创建节点数据
    │
    ▼
读取当期 sellItems
    │
    ▼
对每个展示柜独立随机抽取一个展示项
```

### 当前规则

- 只从当期 `sellItems` 中抽取
- 不单独读取主题特产字段
- 不单独读取全局 `#ruralroutes:pool/specialty`
- 多个展示柜之间允许抽到重复商品

## 展示柜分配逻辑

### 分配时机

贸易站首次交互和周期刷新后，`syncNodeIdToNearbyBlocks` / `refreshDisplayCases` 中执行。

### 分配流程

```
遍历 16 格半径内的展示柜
    │
    ▼
从 sellItems 中随机选一个 NodeTradeEntry
    │
    ▼
使用 NodeTradeEntry.displayStackOrDefault() 作为 displayItem
    │
    ▼
同步到客户端
```

### 分配示例

```
村庄出售清单：[小麦, 胡萝卜, 钻石, 铁锭]
展示柜数量：4

展示柜1 → 小麦
展示柜2 → 胡萝卜
展示柜3 → 钻石
展示柜4 → 小麦
```

## 周期刷新

贸易周期刷新时：
1. 重新生成当期 `sellItems`
2. 重新分配展示柜 `displayItem`

## 初始化流程

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：放置展示柜方块即可，无需配置

### 阶段二：首次交互（运行时）

执行者：贸易站（自动同步）
时机：玩家首次激活贸易站时
操作：
1. 贸易站创建区块数据，生成当期 sell/buy 清单
2. 遍历附近展示柜，分配 displayItem
3. 同步 `tradeNodeId` 和 `tradeStationPos` 到展示柜

## 交互流程

```
玩家右键展示柜
    │
    ▼
检查区块是否有 CommercialNodeData
    │
    ├─ 无 → 提示"请先激活贸易站"
    │
    ▼ 有
校验 tradeNodeId
    │
    ├─ 不匹配 → 提示"数据不匹配"
    │
    ▼ 匹配
打开 GUI（纯展示）
```

## GUI 设计

**定位：** 纯展示，无交易功能。

**展示内容：**
- 物品图标
- 物品名称
- 物品 tooltip（原版风格）

**布局：** 单物品展示，待细化。

## 数据同步

继承 `TradeNodeBlockEntity`，新增 `displayItem` 同步：

- `getUpdateTag()` - 同步 `tradeNodeId`、`tradeStationPos`、`displayItem`
- `handleUpdateTag()` - 客户端接收更新
- `setDisplayItem()` - 修改后调用 `sendBlockUpdated()` 触发同步
