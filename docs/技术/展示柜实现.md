# 展示柜实现

## 数据结构

### BlockEntity

`DisplayCaseBlockEntity` 仅存储节点ID，不存储主题：

| 字段 | 类型 | 说明 |
|---|---|---|
| `tradeNodeId` | `UUID` | 节点唯一标识，由贸易站同步 |

特产数据存储在区块的 `CommercialNodeData.specialties` 中。

**注意**：展示柜不需要 `villageTheme`，仅通过 `tradeNodeId` 与贸易站关联。

### 数据同步机制

BlockEntity 继承 `TradeNodeBlockEntity`，实现标准数据同步：

- `getUpdateTag()` - 同步 `tradeNodeId` 和 `tradeStationPos` 到客户端
- `handleUpdateTag()` - 客户端接收更新
- `setTradeNodeId()` - 修改后调用 `sendBlockUpdated()` 触发同步

## 初始化流程

展示柜不需要预置主题，仅通过贸易站自动同步节点ID：

### 阶段一：结构模板（开发时）

执行者：模组开发者
时机：制作 .nbt 结构模板时
操作：放置展示柜方块即可，无需配置
确定数据：无

### 阶段二：首次交互（运行时）

执行者：贸易站（自动同步）
时机：玩家首次激活贸易站时
操作：贸易站创建区块数据后同步 `tradeNodeId` 到附近展示柜
确定数据：`tradeNodeId`

## 交互流程

```
玩家右键展示柜
    │
    ▼
检查区块是否已有 CommercialNodeData
    │
    ├─ 无 → 提示"请先激活贸易站"
    │
    ▼ 有
校验 tradeNodeId（如已同步）
    │
    ├─ 不匹配 → 提示"数据不匹配"
    │
    ├─ 未同步（null） → 允许通过，等待贸易站同步
    │
    ▼ 匹配或未同步
读取 CommercialNodeData.specialties
显示特产列表（第一阶段为占位显示）
```

## 特产生成逻辑

特产列表在创建 `CommercialNodeData` 时一次性生成：
