基于自定义 JSON 多文件加载实现主题模板数据的注册与查询。

## 实现形式

**自定义 JSON 多文件加载**，与 DataMaps 无关。文件和内容完全自控，数据结构贴合业务需求。

**文件位置**：
```
data/<namespace>/ruralroutes/themes/<主题名>.json
```

## 文件格式

```json
{
  "name": "plains_granary",
  "biome": "minecraft:plains",
  "sell_items": [
    { "type": "tag", "id": "#ruralroutes:pool/crop" },
    { "type": "item", "id": "minecraft:bread" }
  ],
  "buy_items": [
    { "type": "tag", "id": "#ruralroutes:pool/mineral" },
    { "type": "item", "id": "minecraft:iron_ingot" }
  ],
  "theme_specialties": [
    "minecraft:golden_carrot"
  ],
  "stock": {
    "default": { "min": 8, "max": 16 },
    "specific": {
      "#ruralroutes:pool/crop": { "min": 32, "max": 64 },
      "minecraft:golden_carrot": { "min": 1, "max": 3 }
    }
  },
  "price_modifiers": {
    "#ruralroutes:pool/crop": { "sell": 0.8, "buy": 1.2 },
    "#ruralroutes:pool/mineral": { "sell": 1.2, "buy": 0.8 }
  }
}
```

### 字段说明

| 字段 | 必需 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `name` | 是 | 字符串 | 主题标识，需与文件名一致 |
| `biome` | 是 | 字符串 | 所属群系 ID |
| `sell_items` | 是 | 数组 | 村庄出售给玩家的物品。`type` 为 `"tag"` 或 `"item"` |
| `buy_items` | 是 | 数组 | 村庄从玩家收购的物品，格式同上 |
| `theme_specialties` | 否 | 字符串数组 | 主题特产，1-2 种物品 ID |
| `stock` | 否 | 对象 | 库存范围配置 |
| `stock.default` | 否 | `{"min", "max"}` | 默认库存范围。不填回退到模组全局配置。 |
| `stock.specific` | 否 | 对象 | 特定物品/标签的库存范围。键为 `item_id` 或 `#tag_id`。 |
| `price_modifiers` | 否 | 对象 | 价格修正系数。键为 `#tag_id` 或 `item_id`。值为 `{"sell", "buy"}`。 |

## 核心规则

**1. sell / buy 不强制使用标签**

每项支持 `"type": "tag"` 或 `"type": "item"`，可混用。

**2. 库存范围随机**

村庄生成时，在 `[min, max]` 内取随机整数作为该物品的库存上限。

匹配优先级：精确 `item` ID > 标签（第一个匹配） > `default` > 模组全局配置。

**3. 价格修正不校验 buy > sell**

技术上不做约束，允许出现 `buy >= sell`。这是玩法设计的自由。

**4. 多池修正——加性叠加**

物品属于多个池且都匹配到 `price_modifiers` 时，修正系数求和：

> 最终 sell 系数 = 1 + Σ(各命中池的 sell - 1)  
> 最终 buy 系数 = 1 + Σ(各命中池的 buy - 1)

命中判定同时包含标签匹配和精确 ID 匹配。精确 ID 可叠加在标签之上。

**5. 物品展开**

`sell_items` / `buy_items` / `stock.specific` / `price_modifiers` 中的标签键在运行时动态展开为具体物品列表。由物品池模块通过原版标签 API 提供。

## 加载与查询

**加载时机**：服务器启动时，扫描所有数据包主题文件夹。

**索引结构**：
- `Map<ResourceLocation, ThemeTemplate>`：主题名 → 模板对象。
- `Map<ResourceLocation, List<ThemeTemplate>>`：群系 ID → 该群系下的主题列表。村庄生成时调用。

**响应 `/reload`**：重新加载所有文件，重建索引。已生成的村庄数据不受影响。

**查询接口**：
- `getTheme(String name)`：按名称获取完整模板。
- `getThemesForBiome(ResourceLocation biome)`：获取指定群系可用主题列表，供村庄生成随机选取。

## 与相关模块的联动

- **物品池**：提供标签展开能力，`sell_items` 等字段依赖物品池动态解析。
- **价值表**：基础价值由价值表模块独立查询，主题模板仅提供修正系数。
- **村庄生成**：选中主题后，按模板创建买卖清单、库存和特产。
- **贸易站 GUI**：读取模板生成的买卖清单和实时库存，显示给玩家。
