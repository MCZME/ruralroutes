# 主题模板实现

基于自定义 JSON 多文件加载实现主题模板与交易配置片段的注册、合并与查询。

当前实现已经从“单文件主题承载全部交易定义”重构为两层数据：

- `ThemeTemplate`：只描述主题级字段，如 `biome`、库存默认值、价格修正、引用哪些交易 profile
- `TradeProfile`：承载买卖候选、主题特产、契约和按目标声明的库存覆盖
- `ResolvedTheme`：服务端 reload 时将两者合并后的运行时对象，供节点生成、定价和 GUI 直接消费

## 文件位置

```text
data/<namespace>/ruralroutes/themes/<主题名>.json
data/<namespace>/ruralroutes/trade_profiles/<profile名>.json
```

`ThemeManager` 会同时扫描两个目录，并在加载阶段合并为最终的 `ResolvedTheme`。

## ThemeTemplate 文件格式

示例：

```json
{
  "name": "ruralroutes:plains_granary",
  "biome": "minecraft:plains",
  "stock": {
    "default": { "min": 10, "max": 18 },
    "specific": {
      "plains_granary/staples": { "min": 22, "max": 34 },
      "minecraft:golden_carrot": { "min": 1, "max": 3 }
    }
  },
  "price_modifiers": [
    {
      "target": "#ruralroutes:candidate/biome/plains/crop",
      "sell": 0.82,
      "buy": 0.76
    },
    {
      "target": "minecraft:golden_carrot",
      "sell": 1.12,
      "buy": 0.84
    }
  ],
  "trade_profiles": [
    "ruralroutes:plains_granary"
  ]
}
```

### 字段说明

| 字段 | 必需 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `name` | 是 | 字符串 | 主题标识 |
| `biome` | 是 | 字符串 | 所属群系 ID |
| `stock` | 否 | 对象 | 主题级库存默认值与覆盖规则 |
| `price_modifiers` | 否 | 数组或旧版对象 | 主题级价格修正列表，运行时按顺序匹配 |
| `trade_profiles` | 否 | 字符串数组 | 需要合并进该主题的交易 profile 列表 |

## TradeProfile 文件格式

示例：

```json
{
  "name": "ruralroutes:plains_granary",
  "sell_items": [
    "minecraft:oak_planks",
    {
      "key": "plains_granary/staples",
      "items": [
        "minecraft:bread",
        "#ruralroutes:candidate/biome/plains/crop"
      ],
      "pick": 3
    },
    "#ruralroutes:currency"
  ],
  "buy_items": [
    "minecraft:wheat",
    "minecraft:iron_ingot",
    "minecraft:coal",
    {
      "key": "plains_granary/procurement",
      "items": [
        "minecraft:raw_iron",
        "minecraft:charcoal",
        "minecraft:stone_hoe"
      ],
      "pick": 2
    },
    "#ruralroutes:currency"
  ],
  "theme_specialties": [
    "minecraft:golden_carrot"
  ],
  "trade_contracts": [
    {
      "type": "fixed",
      "inputs": [
        { "item": "minecraft:carrot", "count": 8 },
        { "item": "minecraft:bread", "count": 2 },
        { "item": "minecraft:wheat", "count": 6 }
      ],
      "outputs": [
        { "item": "minecraft:golden_carrot", "count": 1 }
      ]
    }
  ]
}
```

### 字段说明

| 字段 | 必需 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `name` | 是 | 字符串 | profile 标识 |
| `sell_items` | 否 | 数组 | 村庄出售给玩家的候选物品 |
| `buy_items` | 否 | 数组 | 村庄从玩家收购的候选物品 |
| `theme_specialties` | 否 | 数组 | 主题特产定义 |
| `stock` | 否 | 对象 | profile 级库存覆盖。允许声明 `targets/specific`，不允许声明 `default` |
| `trade_contracts` | 否 | 数组 | profile 附带的契约定义 |

## 物品引用格式

`sell_items`、`buy_items` 和 `theme_specialties` 统一使用 `ItemReference`：

| 格式 | 示例 | 含义 |
|------|------|------|
| `#<标签ID>` | `#ruralroutes:pool/crop` | 标签引用，运行时展开为标签内所有物品 |
| `<物品ID>` | `minecraft:bread` | 精确物品引用 |
| `{ "id": "#<标签ID>", "pick": N }` | `{ "id": "#ruralroutes:pool/crop", "pick": 3 }` | 单引用展开后随机抽取 `N` 个不同物品 |
| `{ "items": ["<物品ID>", "#<标签ID>", ...], "pick": N }` | `{ "items": ["minecraft:bread", "#ruralroutes:candidate/biome/plains/crop"], "pick": 2 }` | 混合候选组，先合并候选再抽样 |
| `{ "key": "<来源键>", "items": [...], "pick": N }` | `{ "key": "plains_granary/staples", "items": [...], "pick": 3 }` | 为候选组声明稳定来源键，供库存和规则匹配 |
| `{ "id": "minecraft:book", "components": { ... } }` | 自定义命名书本 | 精确到组件的物品引用 |

### 组件化物品

当前实现允许在单物品引用中附带 `components`，运行时会：

- 在节点生成时构造带组件的展示栈
- 用 `TradeItemKey` 记录“基础物品 + 组件签名”
- 在主题价格修正、市场事件匹配、库存索引和混合交易执行中保持该身份

混合交易执行器现在会按完整 `TradeItemKey` 合并、盘点和扣除，不会再把带组件物品压成裸 `Item`。

## 库存配置

库存配置统一使用 `StockConfig`：

```json
{
  "default": { "min": 8, "max": 16 },
  "targets": {
    "#ruralroutes:candidate/biome/plains/crop": { "min": 20, "max": 30 },
    "plains_granary/staples": {
      "sell": { "min": 22, "max": 34 },
      "buy": { "min": 8, "max": 12 }
    }
  },
  "specific": {
    "minecraft:golden_carrot": { "min": 1, "max": 3 }
  }
}
```

### 规则

- `default` 只允许出现在 `ThemeTemplate.stock`
- `TradeProfile.stock` 只能提供 `targets` 或 `specific`
- `targets` 支持共享范围，或显式区分 `sell` / `buy`
- `specific` 保留旧式精确覆盖写法，兼容 `item_id`、`#tag`、来源键

运行时解析优先顺序：

1. 先按 `targets` 解析统一目标
2. 若未命中，再按 `specific` 做兼容解析
3. 若仍未命中，回退到 `default`

## 价格修正

`price_modifiers` 使用 `PriceModifier` 列表，目标统一由 `TradeTargetRef` 表达：

```json
[
  {
    "target": "#ruralroutes:candidate/biome/plains/crop",
    "sell": 0.82,
    "buy": 0.76
  },
  {
    "target": "@plains_granary/staples",
    "sell": 0.90,
    "buy": 0.84
  },
  {
    "target": {
      "item": "minecraft:book",
      "components": {
        "minecraft:custom_name": "{\"text\":\"Catalog\"}"
      }
    },
    "sell": 1.10,
    "buy": 0.95
  }
]
```

支持四类目标：

- 精确物品 ID
- 标签
- `@source_key`
- 带组件的精确物品

匹配优先级：

1. 带组件精确物品
2. `source_key`
3. 裸物品 ID
4. 标签

同一层级内按列表声明顺序决定优先级。

## TradeProfile 合并规则

`ThemeManager` 会把 `ThemeTemplate.trade_profiles` 按声明顺序展开并合并：

- `sell_items` 追加拼接
- `buy_items` 追加拼接
- `theme_specialties` 追加拼接
- `trade_contracts` 追加拼接
- `stock.targets/specific` 后声明覆盖先声明
- `price_modifiers` 目前只取主题级字段，不从 profile 继承

合并结果写入 `ResolvedTheme`，之后节点系统、定价系统和 GUI 都只消费 `ResolvedTheme`。

## 运行时对象

### ThemeTemplate

原始主题数据，只保留主题级字段：

- `name`
- `biome`
- `stock`
- `priceModifiers`
- `tradeProfiles`

### TradeProfile

原始交易片段，承载可复用的交易内容：

- `sellItems`
- `buyItems`
- `themeSpecialties`
- `stock`
- `tradeContracts`

### ResolvedTheme

服务端 reload 后的运行时消费对象：

- `sellItems` / `buyItems`
- `themeSpecialtyItems`
- `stock`
- `priceModifiers`
- `tradeContracts`
- `tradeProfiles`

## 加载与查询

加载时机：服务器启动和 `/reload` 时。

索引结构：

- `Map<ResourceLocation, ThemeTemplate>`：原始主题
- `Map<ResourceLocation, TradeProfile>`：原始 profile
- `Map<ResourceLocation, ResolvedTheme>`：运行时已解析主题
- `Map<ResourceLocation, List<ResolvedTheme>>`：按 biome 分组后的运行时主题

查询接口：

- `ThemeManager.getTheme(name)`：返回 `ResolvedTheme`
- `ThemeManager.getThemesForBiome(biome)`：返回该群系下所有已解析主题
- `ThemeManager.getAllRawThemes()` / `getAllProfiles()`：调试与校验使用

## 与相关模块的联动

- **物品池**：提供标签展开能力
- **节点生成**：消费 `ResolvedTheme`，抽样生成 `NodeTradeEntry` 和 `NodeStockEntry`
- **市场定价**：通过 `TradeItemKey + sourceKey` 匹配主题修正
- **市场状态**：共享 `TradeTargetRef` 匹配骨架
- **贸易站 GUI**：直接使用节点数据中的展示栈和库存键，不再从原始主题文件重建物品身份
- **交易执行**：混合交易按完整 `TradeItemKey` 合并和消耗
