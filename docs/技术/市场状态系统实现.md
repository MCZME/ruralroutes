## 概述

市场状态系统是第二阶段的核心模块，包含两个子模块：
- **模块1**：市场状态模型与查询接口
- **模块2**：市场状态生成引擎

## 核心类

### 数据模型（`core/market/`）

| 类 | 说明 |
|---|-----|
| `MarketScopeType` | 作用域枚举：GLOBAL/BIOME/THEME |
| `MarketEvent` | 市场事件 record |
| `MarketState` | 市场状态快照 record |
| `MarketContext` | 查询上下文 record |
| `MarketPriceAdjustment` | 价格调整结果 record |
| `MarketEventScopeRule` | 作用域展开规则 record |
| `MarketEventRule` | 市场事件规则 record |

### 核心组件

| 类 | 说明 |
|---|-----|
| `MarketStateResolver` | 静态解析器，计算价格调整 |
| `MarketEventRuleCatalog` | 规则目录，资源重载监听器 |
| `MarketStateGenerator` | 状态生成器 |

## 数据流

```
数据包 JSON → MarketEventRuleCatalog → MarketEventRule
                                           ↓
                              MarketStateGenerator
                                           ↓
                                  MarketState
                                           ↓
                              CycleManager（持有和持久化）
                                           ↓
                              MarketStateResolver（查询）
                                           ↓
                              MarketPriceAdjustment
```

## 周期更新接入

市场状态在以下时机更新：

1. **节点刷新时**（`CommercialNodeManager.checkAndRefreshCycle()`）
   - 调用 `cycleManager.getOrInitMarketState()`
   - 惰性刷新，确保市场状态存在

2. **周期边界时**（`RuralRoutes.onServerTick()`）
   - 监听 `ServerTickEvent.Post`
   - 检测周期边界，预先生成市场状态

## 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `market.enabled` | 是否启用市场系统 | true |
| `market.rule_pick_count` | 每周期抽取规则数量 | 3 |
| `market.max_delta` | 最大涨跌幅度 | 0.50 |

## 管理命令

| 命令 | 说明 |
|------|------|
| `/ruralroutes refresh` | 强制刷新贸易周期 |
| `/ruralroutes market refresh` | 强制刷新市场状态 |
| `/ruralroutes market info` | 显示当前市场信息 |

## 规则文件格式

路径：`data/<namespace>/ruralroutes/market_event_rules/*.json`

```json
{
  "id": "ruralroutes:test_event",
  "name_key": "market.ruralroutes.test_event",
  "target_ref": "minecraft:bread",
  "scopes": [
    {"type": "global", "selector": "all"}
  ],
  "delta": -0.15,
  "weight": 80
}
```
