# 调试工具实现

## 用途

调试工具用于开发阶段检查和配置核心方块，当前包含两类独立物品：

| 物品 | 注册名 | 用途 |
|---|---|---|
| 配置工具 | `config_tool` | 配置贸易站主题、复制节点信息、调整展示柜/传闻板外观 |
| 节点数据查看器 | `node_data_viewer` | 查看核心方块对应的商业节点数据快照 |

两者互不替代：配置工具负责修改，节点数据查看器负责只读调试。

## 配置工具

### 支持范围

| 方块 | 功能 |
|---|---|
| `TradeStationBlockEntity` | 选择并应用主题，复制当前节点信息 |
| `DisplayCaseBlockEntity` | 选择外观，粘贴节点信息 |
| `RumorBoardBlockEntity` | 选择外观，粘贴节点信息 |

### 交互方式

- 使用 `ConfigToolItem.onItemUseFirst` 拦截右键。
- 服务端通过 `player.openMenu()` 打开 `ConfigToolMenu`。
- 客户端通过 `ConfigToolScreen` 渲染配置界面。
- 变更通过 `ConfigToolApplyPayload` 回传服务端执行。

### 数据职责

- 主题配置仅对贸易站生效。
- 外观配置仅对带 `STYLE` 属性的核心方块生效。
- 复制/粘贴节点信息依赖 `RRDataComponents` 中的 `copied_node_id` 与 `copied_station_pos`。

## 节点数据查看器

### 支持范围

节点数据查看器仅支持当前 3 个核心方块：

| 方块 | 读取规则 |
|---|---|
| `TradeStationBlockEntity` | 直接读取点击位置所在区块的 `CommercialNodeData` |
| `DisplayCaseBlockEntity` | 先读取 `tradeStationPos`，再读取对应贸易站的节点数据 |
| `RumorBoardBlockEntity` | 先读取 `tradeStationPos`，再读取对应贸易站的节点数据 |

对不支持的方块返回 `PASS`，不影响原交互。

### 打开链路

节点数据查看器不使用 `Menu`，只走服务端发包后客户端直接打开 `Screen`：

```text
NodeDataViewerItem.onItemUseFirst()
    ↓
服务端解析目标方块与节点来源
    ↓
OpenNodeDataViewerPayload
    ↓
客户端 setScreen(new NodeDataViewerScreen(...))
```

### 状态模型

Payload 固定携带目标方块类型与查看状态：

| 字段 | 说明 |
|---|---|
| `TargetBlockType` | `TRADE_STATION` / `DISPLAY_CASE` / `RUMOR_BOARD` |
| `ViewStatus` | `HAS_DATA` / `MISSING_STATION` / `MISSING_NODE_DATA` |
| `CommercialNodeData` | 可选节点快照，仅 `HAS_DATA` 时存在 |

`CommercialNodeData` 使用现有 `CODEC` 编码到 NBT，再随 payload 下发到客户端。

### 空状态规则

查看器不会触发节点初始化，也不会写回世界数据：

- `MISSING_STATION`：展示柜或传闻板还没有关联贸易站
- `MISSING_NODE_DATA`：目标还没有商业节点数据

这两种情况都会打开空状态界面，方便排查链路问题。

### UI 显示内容

`NodeDataViewerScreen` 为只读调试界面，分为 5 个区块：

1. 概要：`tradeNodeId`、`themeName`、`refreshTimestamp`
2. `sellItems`
3. `buyItems`
4. `specialties`
5. `stocks`

显示规则：

- `sellItems`、`buyItems`、`specialties` 按原始顺序展示
- `stocks` 按物品 id 排序，保证调试结果稳定
- 每行显示物品图标、译名与注册表 id
- 库存行额外显示 `current/max`
