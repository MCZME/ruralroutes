# 贸易周期实现

记录贸易周期的技术实现细节。

## 设计概要

采用**惰性刷新**设计：
- 全局周期核心维护周期索引计算
- 玩家交互时检查节点是否需要刷新
- 避免主动轮询所有节点

## 核心类

### CycleManager (SavedData)

位置：`core/cycle/CycleManager.java`

```java
public class CycleManager extends SavedData {
    // 缓存的当前周期索引（初始值 -1）
    private long currentCycle;
    
    // 当前周期是否需要刷新（初始值 true）
    private boolean needsRefresh;
    
    // 当前周期的市场状态
    private MarketState marketState;
    
    // 获取有效时间（根据配置的时间模式）
    public static long getEffectiveTime(Level level);
    
    // 获取实例
    public static CycleManager get(ServerLevel level);
    
    // 计算周期索引（使用有效时间）
    public long getCycleIndex(ServerLevel level);
    
    // 计算时间戳对应的周期索引
    public long getCycleIndex(long timestamp);
    
    // 获取缓存的周期索引
    public long getCurrentCycle();
    
    // 更新缓存周期索引，返回 true 表示周期推进
    public boolean updateCurrentCycle(ServerLevel level);
    
    // 判断是否需要刷新（周期推进或需要刷新）
    public boolean needsRefresh(long nodeTimestamp);
    
    // 获取或初始化市场状态（使用缓存周期）
    public MarketState getOrInitMarketState();
    
    // 检查/标记/请求刷新标志
    public boolean isNeedsRefresh();
    public void markRefreshed();
    public void requestRefresh();
}
```

存储位置：`data/ruralroutes_cycle.dat`

存储内容：周期缓存、刷新标志、市场状态。

## 刷新逻辑

### 时间源

根据配置的时间模式获取有效时间：

```java
public static long getEffectiveTime(Level level) {
    return switch (Config.CYCLE_TIME_MODE.get()) {
        case SERVER_TIME -> level.getGameTime();  // 服务器累计 ticks
        case GAME_TIME   -> level.getDayTime();   // 游戏内时间
    };
}
```

### 周期索引计算

```
周期索引 = getEffectiveTime(level) / (cycleDays * 24000)
```

所有节点按同一全局周期边界刷新。

### 判断流程

```
玩家交互贸易站
    ↓
获取节点的 refreshTimestamp
    ↓
计算节点周期索引 vs 当前周期索引
检查 needsRefresh 标志
    ↓
如果 当前周期 > 节点周期 或 needsRefresh == true
    ↓
刷新节点，标记 needsRefresh = false
    ↓
按主题候选重建当期买卖清单
    ↓
恢复库存到基准值（全量恢复）
更新 refreshTimestamp 为当前有效时间
```

### 触发点

`TradeStationBlock.useWithoutItem()` 中，数据校验通过后、打开菜单前调用：

```java
if (level instanceof ServerLevel serverLevel) {
    nodeData = CommercialNodeManager.checkAndRefreshCycle(serverLevel, pos, nodeData);

```

### 传闻板适配

[`RumorBoardScreen.java`](./src/main/java/github/mczme/ruralroutes/client/gui/screen/RumorBoardScreen.java) 中的 `collectClientData()` 方法继续使用 `CycleManager.getEffectiveTime(level)` 计算剩余时间显示，但不再自行生成 `MarketState`。当前周期的市场状态由服务端通过 [`OpenRumorBoardPayload.java`](./src/main/java/github/mczme/ruralroutes/network/packet/OpenRumorBoardPayload.java) 下发，`RumorBoardScreen` 仅以 `marketState.cycleIndex()` 作为当前展示周期，并基于这份服务端权威数据生成情报文本。

## 恢复规则

第一阶段实现 `full`（全量恢复）：
- 从主题模板重新抽样当期 sell/buy 清单
- 从主题模板重新初始化库存
- 出售物品：库存恢复到上限
- 收购物品：库存清零

第二阶段将实现 `category`（分类处理）和 `incremental`（增量恢复）。
