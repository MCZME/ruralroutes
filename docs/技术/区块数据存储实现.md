# 区块数据存储实现

## 数据结构

每个区块存储一个商业节点数据：

```java
public class CommercialNodeData {
    private ResourceLocation themeName;      // 主题标识
    private Map<ResourceLocation, StockEntry> stocks;  // 各物品库存
    private long refreshTimestamp;           // 上次刷新时间戳
}

public class StockEntry {
    private int current;    // 当前库存
    private int max;        // 库存上限
}
```

## 存储方案

使用 NeoForge 的 `ChunkCapability` 或 `Attachment` 系统。

## 数据生命周期

1. **创建**：玩家首次交互时，由贸易站方块触发
2. **持久化**：随区块保存/加载
3. **刷新**：根据贸易周期更新库存
