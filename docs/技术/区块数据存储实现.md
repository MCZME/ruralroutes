# 区块数据存储实现

## 数据结构

每个区块存储一个 `CommercialNodeData`，包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `tradeNodeId` | UUID | 节点唯一标识 |
| `themeName` | ResourceLocation | 主题标识 |
| `sellItems` | List<ResourceLocation> | 当前周期的出售物品列表（村庄卖给玩家） |
| `buyItems` | List<ResourceLocation> | 当前周期的收购物品列表（玩家卖给村庄） |
| `stocks` | Map<ResourceLocation, StockEntry> | 各物品库存 |
| `refreshTimestamp` | long | 上次刷新时间戳 |

`StockEntry` 包含 `current`（当前库存）和 `max`（库存上限）。

`sellItems` 和 `buyItems` 存储该村庄当前周期的买卖清单，在创建节点和周期刷新时由主题候选展开并抽样生成。

## 存储方案

使用 NeoForge 的 **Attachment 系统**存储区块数据。

- 注册 `AttachmentType<CommercialNodeData>`
- 数据随区块自动持久化
- 通过 `chunk.getData()` / `chunk.setData()` 访问

## 数据存储位置对比

| 数据 | 存储位置 | 用途 |
|------|----------|------|
| `tradeNodeId` | BlockEntity + 区块数据 | 唯一标识，双重存储保证一致性 |
| `themeName` | BlockEntity + 区块数据 | BlockEntity 用于校验，区块数据用于运行 |
| `sellItems` | 仅区块数据 | 展示柜读取显示 |
| `buyItems` | 仅区块数据 | 收购物品列表 |
| `stocks` | 仅区块数据 | 核心业务数据 |
| `refreshTimestamp` | 仅区块数据 | 刷新逻辑 |

## BlockEntity 角色

贸易站 BlockEntity 保留校验数据：
- `villageTheme` - 用于校验区块数据是否匹配
- `tradeNodeId` - 节点唯一标识

展示柜和传闻板 BlockEntity 仅保留节点ID：
- `tradeNodeId` - 节点唯一标识（由贸易站同步）

**注意**：展示柜和传闻板不需要 `villageTheme`，仅通过 `tradeNodeId` 与贸易站关联。

**校验逻辑**：
1. 首次交互时（通过贸易站触发），检查贸易站的 `villageTheme` 是否有效
2. 创建区块数据时，将 `tradeNodeId` 同步写入贸易站及附近展示柜、传闻板的 BlockEntity
3. 后续交互时，校验 BlockEntity 的 `tradeNodeId` 与区块数据是否一致

## 数据生命周期

### 创建

玩家首次交互时触发：

```
玩家右键贸易站
    │
    ▼
检查 BlockEntity.themeName 是否有效
    │
    ├─ 无效 → 无响应
    │
    ▼ 有效
检查区块是否有 CommercialNodeData
    │
    ├─ 有 → 校验 tradeNodeId，打开 GUI
    │
    ▼ 无
检查旁边是否有村庄
    │
    ├─ 无 → 无响应
    │
    ▼ 有
创建 CommercialNodeData：
  - tradeNodeId = UUID.randomUUID()
  - themeName = BlockEntity.themeName
  - sellItems = 从 ThemeTemplate.sellItems 展开候选并抽样
  - buyItems = 从 ThemeTemplate.buyItems 展开候选并抽样
  - stocks = 仅为本次入选物品初始化
  - refreshTimestamp = 当前时间
同步写入 BlockEntity.tradeNodeId（贸易站、展示柜、传闻板）
```

### 持久化

- 区块数据随区块保存/加载
- BlockEntity 数据随方块实体保存/加载

### 刷新

贸易周期管理器在全局刷新时：

1. 遍历所有已加载区块
2. 检查是否有 `CommercialNodeData`
3. 按主题候选重建当前周期清单
4. 恢复库存至基准值（从 `ThemeTemplate.stock` 获取）
5. 更新 `refreshTimestamp`
