# 交易引擎实现

## 核心思想

底层只做一件事：**价值匹配的物品交易**。不区分购买、出售、铸币。

上层业务（购买、出售、铸币）负责构造交易请求，引擎只负责验证和执行。

---

## 当前实现范围

- 使用物品基础价值进行判断（从 DataMap 获取）
- 暂不考虑主题修正系数
- 暂不考虑货币特殊处理

---

## 交易约束

一次交易满足以下四个条件才能执行：

| 条件 | 说明 |
|------|------|
| **价值匹配** | 付出组总价值 = 获得组总价值 |
| **玩家库存** | 玩家背包有足够数量的付出组物品 |
| **村庄库存** | 村庄的物品库存能覆盖获得组 |
| **背包空间** | 玩家背包有空位容纳获得组物品 |

### TradeRequest

```java
public record TradeRequest(
    List<ItemStack> giveItems,   // 玩家付出的物品
    List<ItemStack> takeItems    // 玩家获得的物品
) {
    public boolean isValid() {
        // 验证请求非空且无空 ItemStack
    }
}
```

### TradeResult

```java
public record TradeResult(
    boolean success,
    FailReason failReason,       // 失败原因枚举
    List<ItemStack> shortfall,   // 差额列表（UI 提示用）
    int totalValueExchanged      // 实际交易的总价值
) {
    public enum FailReason {
        VALUE_MISMATCH,          // 价值不匹配
        PLAYER_INSUFFICIENT,     // 玩家库存不足
        VILLAGE_INSUFFICIENT,    // 村庄库存不足
        PLAYER_NO_SPACE,         // 玩家背包空间不足
        INVALID_REQUEST          // 无效请求
    }
}
```

失败原因使用本地化翻译键，如 `trade.fail.value_mismatch`。

## 执行流程

```
1. 获取锁（基于 ChunkPos）
   - 使用 TradeLockManager 的 Striped Lock
   - 同一区块串行执行，不同区块并行

2. 验证请求有效性
   - 非空且无空 ItemStack

3. 验证价值匹配
   - 计算付出组总价值（基础价值 × 数量）
   - 计算获得组总价值
   - 不匹配 → 返回失败

4. 验证玩家库存
   - 遍历付出组，检查玩家背包是否有足够数量
   - 不足 → 返回失败及差额

5. 验证村庄库存
   - 遍历获得组，检查村庄物品库存
   - 不足 → 返回失败及差额

6. 验证玩家背包空间
   - 计算获得组需要的空位数量
   - 扣除付出组释放的空位
   - 不足 → 返回失败

7. 原子执行
   - 玩家背包 -= 付出组，+= 获得组
   - 村庄库存 -= 获得组，+= 付出组
   - 通过 CommercialNodeManager.updateNodeData() 更新区块数据

8. 释放锁
```

任一验证失败 → 返回失败及具体原因。

## 价值计算（当前简化版）

直接使用 `ValueTableManager.queryBaseValue(ItemStack)` 获取基础价值：

```java
private int calculateTotalValue(List<ItemStack> items) {
    int total = 0;
    for (ItemStack stack : items) {
        total += ValueTableManager.queryBaseValue(stack) * stack.getCount();
    }
    return total;
}
```

## 并发处理

同一区块的交易请求排队执行，保证数据一致性。

**TradeLockManager 实现**：
- 1024 个 ReentrantLock 复用（Striped Lock）
- 基于 ChunkPos 哈希分配锁
- 同一区块内的交易请求串行执行
- 不同区块可并行
- 5 秒超时防止死锁

```java
public class TradeLockManager {
    private static final int STRIPES = 1024;
    private final ReentrantLock[] locks;
    
    public boolean acquire(ChunkPos chunkPos);
    public void release(ChunkPos chunkPos);
}
```

## 上层业务封装

上层业务负责将用户操作转换为交易请求：

### 购买（玩家从村庄买入）

```java
TradeRequest request = new TradeRequest(
    List.of(货币物品),      // 玩家付出货币
    List.of(购买的物品)     // 玩家获得物品
);
```

### 出售（玩家向村庄卖出）

```java
TradeRequest request = new TradeRequest(
    List.of(出售的物品),    // 玩家付出物品
    List.of(货币物品)       // 玩家获得货币
);
```

### 铸币（面额转换）

```java
TradeRequest request = new TradeRequest(
    List.of(兑出的货币),    // 玩家付出低面额/高面额
    List.of(兑入的货币)     // 玩家获得高面额/低面额
);
```
