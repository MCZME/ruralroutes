# 交易引擎实现

## 核心思想

交易引擎负责**执行贸易站交易**，使用服务端权威价格计算。

核心原则：
- **服务端权威**：价格由服务端在执行时重新计算
- **不信任客户端**：客户端不发送价格数据
- **直接读取暂存区**：服务端已有 pendingSlots 数据
- **契约执行模式**：通过 `TradeContractExecutor` 统一执行契约，生成 `TradePaymentPlan` 表示具体物品转移

## 核心执行方法

```java
// TradeContractExecutor 单例模式
public static final TradeContractExecutor INSTANCE = new TradeContractExecutor();

/** 执行单一契约（含库存更新） */
public TradeResult executeContract(
    ServerLevel level,
    CommercialNodeData nodeData,
    ServerPlayer player,
    TradeContract contract,
    BlockPos blockPos
);

/** 执行混合交易（贸易站） */
public TradeResult executeMixedTrade(
    ServerLevel level,
    CommercialNodeData nodeData,
    ServerPlayer player,
    List<PendingTradeSlot> pendingSlots,
    long currentCycleIndex,
    BlockPos blockPos
);
```

## 执行流程

### 单一契约执行（executeContract）

```
1. 获取锁（基于 ChunkPos）
   - 使用 TradeLockManager 的 Striped Lock
   - 同一区块串行执行，不同区块并行

2. 调用契约的 execute() 方法
   - 验证玩家输入（validateInput）
   - 计算价格（动态契约）或使用固定价格（固定契约）
   - 生成 TradePaymentPlan
   - 校验玩家和村庄库存
   - 执行物品转移

3. 更新村庄库存（若成功）
   - consumed 物品加入村庄库存
   - outputs 物品从村庄库存扣除

4. 释放锁
```

### 混合交易执行（executeMixedTrade）

```
1. 获取锁（基于 ChunkPos）
2. 验证周期一致性
   - 对比 currentCycleIndex 与实际周期索引
   - 跨周期 → 返回 CYCLE_CHANGED
3. 服务端权威价格计算
   - 遍历 pendingSlots
   - 对每个物品调用 TradePricingService.calculateFinalPrice()
   - 分别计算 sellValueTotal 和 buyValueTotal
4. 构建 TradePaymentPlan
   - 使用 CurrencyBasketComposer 生成具体货币物品
   - 汇总玩家输入/输出物品
5. 校验玩家和村庄库存
6. 原子执行
   - 玩家背包转移
   - 村庄库存更新
   - 通过 CommercialNodeManager.updateNodeData() 保存
7. 释放锁
```

## TradeResult

统一交易结果，用于混合交易和单一契约执行：

```java
public record TradeResult(
    Reason reason,
    List<ItemStack> outputs,         // 玩家获得的物品
    List<ItemStack> consumed,        // 玩家消耗的物品
    List<ItemStack> shortfall,       // 失败时的不足物品
    int totalValueExchanged          // 交易总价值（净货币差额）
) {
    public enum Reason {
        SUCCESS,
        PLAYER_INSUFFICIENT,
        VILLAGE_INSUFFICIENT,
        INVALID_INPUT,
        INVALID_REQUEST,
        CYCLE_CHANGED
    }
}
```

## TradePaymentPlan

支付计划，表示具体的输入/输出物品集合：

```java
public record TradePaymentPlan(
    List<ItemStack> playerInputs,    // 玩家需要支付的物品
    List<ItemStack> playerOutputs,   // 玩家获得的物品
    List<ItemStack> villageInputs,   // 村庄需要支付的物品
    List<ItemStack> villageOutputs   // 村庄获得的物品
) {}
```

## 与其他模块的关系

| 模块 | 依赖方式 |
|------|----------|
| **定价服务** | `TradePricingService.calculateFinalPrice()` |
| **价值表** | `ValueTableManager.queryBaseValue()` 提供基础价值和货币面额 |
| **标签缓存** | `TagLookupCache.matchesItem()` 判断货币物品 |
| **市场状态** | `MarketStateResolver.resolvePriceAdjustment()` |
| **主题修正** | `ThemePriceModifierResolver.resolve()` |
| **区块数据** | `CommercialNodeData` 存储村庄库存 |
| **节点管理** | `CommercialNodeManager.getNodeData()` / `updateNodeData()` |
