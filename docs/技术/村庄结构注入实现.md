# 村庄结构注入实现

实现贸易站建筑在村庄中的**必定生成**与**唯一生成**。

## 设计目标

| 目标 | 说明 |
|------|------|
| 必定生成 | 每个平原村庄至少有一个贸易站 |
| 唯一生成 | 每个村庄最多只有一个贸易站 |
| 兼容性 | 与其他修改村庄的模组（如 Waystones）共存 |

---

## 技术方案

### 核心原理

通过 Mixin 拦截 `JigsawPlacement.Placer.tryPlacingChildren` 方法，在首次访问 `houses` 模板池时替换为贸易站专用池。

### 村庄生成流程

```
town_centers (起始池)
    ↓ Jigsaw 方块连接
streets (街道)
    ↓ Jigsaw 方块连接
houses (房屋池) ← 我们在这里拦截
    ↓ 递归扩展
terminators (终止)
```

### 执行逻辑

```
村庄生成 →
  第一次访问 houses →
    JigsawPlacementPlacerMixin 拦截 →
    检查路径是否以 "/houses" 结尾 →
    替换为 trade_stations 池 →
    生成贸易站，标记 hasTradeStation = true
  后续访问 houses →
    hasTradeStation = true，不替换 →
    返回原始 houses 池 →
    正常房屋生成
```

## 关键代码

### JigsawPlacementPlacerMixin.java

```java
@Mixin(targets = "net.minecraft.world.level.levelgen.structure.pools.JigsawPlacement$Placer")
public class JigsawPlacementPlacerMixin {

    @Unique
    private boolean ruralroutes$hasTradeStation = false;

    @ModifyArg(
        method = "tryPlacingChildren(...)",
        at = @At(value = "INVOKE", target = "...Registry.getHolder...")
    )
    private ResourceKey<StructureTemplatePool> ruralroutes$forceTradeStationPool(
        ResourceKey<StructureTemplatePool> resourceKey
    ) {
        // 已生成贸易站，不替换
        if (ruralroutes$hasTradeStation) {
            return resourceKey;
        }

        // 检查是否是 houses 池
        String poolPath = resourceKey.location().getPath();
        if (poolPath.endsWith("/houses")) {
            // 替换为贸易站池
            ResourceKey<StructureTemplatePool> tradeStationPoolKey = ...;
            if (pools.getHolder(tradeStationPoolKey).isPresent()) {
                ruralroutes$hasTradeStation = true;
                return tradeStationPoolKey;
            }
        }

        return resourceKey;
    }
}
```

## 兼容性分析

### 与 Waystones 模组共存

两个模组使用相同的技术方案，但互不冲突：

| 情况 | 结果 |
|------|------|
| Waystones 先执行 | 第一次 houses → Waystone，第二次 houses → 贸易站 |
| 我们先执行 | 第一次 houses → 贸易站，第二次 houses → Waystone |
| 两者都标记 | 各自只在首次触发时生成，后续保持原始 houses |

**原因**：两个模组都只替换**首次** houses 访问，标记后返回原始池。因此各自建筑都会生成，只是顺序取决于 Mixin 优先级。

### 兼容性保证

- 使用 `@ModifyArg` 而非 `@Overwrite`，不覆盖整个方法
- 只在首次触发后标记，不影响后续生成
- 检查池是否存在才替换，避免破坏不支持的村庄类型
