# 乡野商路 (Rural Routes)

以村庄为节点，通过交换获取生存、发展直至通关所需资源。

## 核心原则

- **不替代**原版流程
- **不削弱**原版体验
- **不强制**玩家使用

玩家按需选择，与原版流程完全兼容。

## 最终目标

理论上仅靠本模组资源链可击杀末影龙。

末影龙通关是检验标准，不是目的。

玩家自由选择使用与否。

## 定位

完全聚焦**资源端**：村庄、交换、资源分布。

独立于原版村民交易体系。

与「浪漫的载具模组」协同——载具提供旅途体验，本模组为旅行提供停靠意义和资源循环节点。

## 设计层次

模组设计分为两个层次，上层依赖下层：

### 系统层（底层骨架）

定义玩家与村庄之间资源流动的**基本规则**。回答：资源从哪来、怎么流动、有什么约束。

系统层是**玩法中立的**——只提供可能性，不规定玩家必须怎么玩。

### 玩法层（上层血肉）

定义玩家用系统做什么、为什么这么做、有什么反馈。回答：具体行为模式和目标。

---

## 玩法层核心体验

**核心体验定义**

> 「以旅行为生的流浪者」——移动本身就是资源获取的方式。停留太久，库存耗尽、收购饱和、需求不匹配，玩家自然会想移动。去哪、回不回头，都是玩家自己的权衡。

**与原版生存的本质区别**

| | 原版生存 | 乡野商路 |
| :--- | :--- | :--- |
| 资源获取逻辑 | 停下，生产 | 移动，交换 |
| 时间关系 | 等待生长/刷新 | 不等，去下个地方 |
| 空间关系 | 家是固定基地 | 家是移动的（房车） |
| 与村民关系 | 刷列表的工具人 | 真正的交换对象 |
| 核心节奏 | 建农场→等待→收获 | 旅行→发现→交换→继续旅行 |

**玩法层三大方向**

1. **让偶遇有发现感** — 建筑风格+展示柜让村庄有辨识度
2. **让交换有微量策略** — 背包压力+信息不对称，自然留存
3. **让旅程有记忆点** — 关键获取、供需匹配、地图轨迹

**传闻板驱动**

传闻板提供模糊价格情报，让玩家知道「外面有什么变化」，产生移动探索的动力。
