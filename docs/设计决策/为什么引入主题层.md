## 决策

在「群系」与「村庄」之间插入一层手动设计的「主题」。

架构变为：**群系 → 主题 → 村庄**

## 原因

### 问题

早期方案直接用群系物品池随机生成村庄清单：

- 随机性太大，村庄缺乏特点
- 同一群系的村庄买卖清单差异不显著
- 玩家无法通过村庄外观预期能解决什么需求

### 解决

主题是我们主动设计的模板，定义具体村庄类型。

好处：
- **可预期** - 玩家看到主题名称就能知道买卖倾向
- **有辨识度** - 同一群系不同主题的村庄本质不同
- **设计可控** - 我们主动定义每种主题的买卖子集，而非完全随机

## 示例

沙漠群系可包含 3-5 个主题：

| 主题 | 出售倾向 | 收购倾向 |
| :--- | :--- | :--- |
| 砂岩开采村 | 沙子、砂岩变种（大量建筑方块） | 食物、工具 |
| 绿洲农场 | 仙人掌绿、少量农作物 | 矿物 |
| 染料工坊 | 染料、染色陶瓦 | 黏土、骨粉 |
