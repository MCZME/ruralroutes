# 为什么用自建货币

## 决策

模组使用自定义统一货币，而非复用绿宝石。

## 形式确认

- **实体物品** - 占用背包空间，死亡可掉落
- **唯一获取途径** - 仅通过本模组交换系统获得，不出现在原版村民交易、战利品箱、自然生成等任何外部机制中
- **通用性** - 所有村庄统一使用该货币

## 原因

1. **隔离原版经济** - 不干扰原版村民交易体系
2. **独立价值体系** - 可针对模组平衡性单独调整
3. **避免混淆** - 玩家能清晰区分两套交易系统

## 设计影响

- **背包成本** - 携带大量货币占用背包空间，跑商时需考虑钱包厚度
- **死亡风险** - 和所有实体物品一样，死于途中会丢失货币，增加隐性风险管理
- **展示性** - 可放物品展示框，可与其他模组的银行、钱包机制兼容

## 衍生设计

- **货币生成逻辑** - 已确定：向村庄出售物品获得，详见 [货币系统](../系统/货币系统.md)
- **命名与材质** - 已确定：铜板（价值1）、铁币（价值10）、金币（价值100），详见 [货币系统](../系统/货币系统.md)
- **转化机制** - 已确定：贸易站 GUI 铸币标签页，依赖村庄货币库存

## 替代方案

- 复用绿宝石 - 会与原版村民交易产生干扰
- 纯以物易物 - 复杂度高，缺少通用价值尺度
