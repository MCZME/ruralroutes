# 为什么用贸易站+展示柜+传闻板

---
创建日期: 2026-04-29
最后更新: 2026-05-16
版本: v3.1
状态: 已确定
关联系统: 交换机制, 村庄资源身份, 市场波动
---

## 决策

村庄交换通过三个核心方块实现：

- **贸易站**：承载常规买卖与货币交换
- **展示柜**：陈列特产
- **传闻板**：显示当前刷新周期的模糊价格情报

## 原因

### 分离功能，各司其职

| 方块 | 功能 |
| :--- | :--- |
| 贸易站 | 处理所有村庄通用的买卖，提供货币交换功能 |
| 展示柜 | 突出每个村庄的独特性（特产名片） |
| 传闻板 | 提供全局市场信息，驱动玩家移动 |

### 视觉辨识

展示柜作为「村庄名片」，传闻板显示简略图标（是否有新消息、消息种类）。

### 界面简洁

贸易站界面分为出售区、收购区、交易暂存区和货币交换区，同一界面内完成所有操作。
展示柜用独立小界面，只展示特产物品。
传闻板右键打开查看详细情报。

## 具体设计

### 贸易站

| 项目 | 决策 |
| :--- | :--- |
| GUI 架构 | BlockEntity → Menu → Screen（标准三层模式） |
| 界面布局 | 垂直分区：出售区（上）+ 收购区（中）+ 交易暂存区与货币交换区（下，左右并列） |
| 出售区 | 绿色标题，2 行卡片横向滚动，点击卡片添加到暂存区 |
| 收购区 | 红色标题，2 行卡片横向滚动，点击卡片添加到暂存区 |
| 交易暂存区 | 左侧 70% 宽度，显示玩家想要的和玩家支付的物品，确认按钮 |
| 货币交换区 | 右侧紧凑面板，底部钱包筹码 + 点击后向上弹出的“使用该货币可执行的兑换”选项 |
| GUI 尺寸 | 384×256px |
| 卡片尺寸 | 65×26px（两行布局） |

### 展示柜

| 项目 | 决策 |
| :--- | :--- |
| GUI 架构 | BlockEntity → 纯 Screen（无 Menu） |
| 界面布局 | 单物品展示，中央物品槽 + 物品名称 |
| GUI 尺寸 | 176×120px |
| 交互 | 纯展示，无交易功能；鼠标悬停显示原版物品 tooltip |

### 传闻板

| 项目 | 决策 |
| :--- | :--- |
| GUI 架构 | BlockEntity → 纯 Screen（无 Menu） |
| 界面布局 | 便签风格，杂乱分布；底部显示周期剩余时间 |
| GUI 尺寸 | 280×200px |
| 数据来源 | 服务端通过 `OpenRumorBoardPayload` 下发 `MarketState`，客户端生成展示文本 |
| 交互 | 只读，无交易操作 |

### 方块信息呈现

| 项目 | 决策 |
| :--- | :--- |
| 命名与材质 | 统一名称，材质随群系换色 |
