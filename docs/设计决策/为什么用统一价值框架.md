# 为什么用统一价值框架

## 决策

模组内所有可交换物品使用统一的内部基础价值表，货币作为一般等价物。

## 核心规则

### 价值表

所有可交换物品有一个内部基础价值（数值由设计时定义）。货币本身也在价值表上，作为一般等价物。

### 支付方式

玩家购买村庄物品时：
- 用货币支付
- 用村庄收购清单上的物品按价值折算支付
- 混合支付

### 出售回报

玩家向村庄出售物品时，村庄根据物品价值给予回报：
- 默认为货币
- 某些情况下可以是村庄的产出物（由具体设计决定）

### 边界

玩家只能使用村庄明确"接受"的物品进行支付或出售，村庄不接受清单外的物品。

## 设计影响

### 与系统层的关系

- **村庄资源身份** - 群系主题决定村庄的收购/出售倾向，特产为清单增加变量
- **库存与刷新** - 每种物品的库存限制该物品的累计交换量，刷新周期恢复

### 对玩法层的支撑

- **跑商** - 基于不同村庄对同一物品的价值差异自然产生套利空间，无需系统层额外特殊处理
- **特殊以物易物** - 后续设计时为特定物品设置固定交换规则，可覆盖统一价值换算

## 原因

1. **简化实现** - 统一的价值换算逻辑，避免每种物品单独定义交换关系
2. **玩家直觉** - 有一个明确的"值多少钱"概念，降低理解成本
3. **玩法可扩展** - 价值差异可以自然驱动跑商，无需额外机制
