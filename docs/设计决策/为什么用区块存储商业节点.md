# 为什么用区块存储商业节点

## 问题

贸易站的核心方块需要存储商业数据（主题、库存、刷新时间）。存储位置有哪些选择？

## 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| BlockEntity | 最直观，与方块绑定 | 方块被移动/破坏时数据丢失或需额外处理 |
| 村庄实例 | 与村庄概念绑定 | 村庄范围不固定，一个村庄可能有多个贸易站 |
| 区块 | 自然空间划分，与生成位置绑定 | 跨区块边界的村庄需额外处理 |

## 决策

选择**区块**作为存储容器。

**Why:**

核心方块是**自然生成、不可移动、分散分布**的。这些特性使得区块成为最自然的存储边界：

- **不可移动** → 无需处理移动时的数据迁移
- **分散分布** → 一个区块只会有一个商业节点，不会产生冲突
- **自然生成** → 生成时即确定位置，与区块生命周期绑定

**How to apply:**

- 数据以区块为单位存储，使用 `ChunkStorage` 或 `Attachment`
- 每个区块最多一个商业节点
- 方块仅作为交互入口，不存储核心数据
