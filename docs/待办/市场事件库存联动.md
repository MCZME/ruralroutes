# 市场事件库存联动

---
创建日期: 2026-05-15
最后更新: 2026-05-15
版本: v1.0
状态: 讨论
关联系统: 市场波动, 村庄库存规则, 贸易周期, 交换机制
---

## 问题

当前市场事件只通过价格计算链路影响交易结果：

- `MarketState` 生成当期市场事件
- `MarketStateResolver` 解析价格调整
- `TradePricingService` 将市场因子注入报价

但库存生成与周期刷新仍只读取主题模板的库存基准值，不受市场事件影响。

这会带来一个表达落差：

- 传闻板会说“缺货”“放货”“出得多”
- 实际村庄库存数量却可能没有对应变化

因此，市场事件的文本语义和玩家实际看到的库存状态还没有真正联动起来。

## 目标

让市场事件不仅影响价格，还能影响当期库存生成数量，使以下体验更一致：

- **缺货 / 紧俏**：不仅更贵，也更少见、更容易卖空
- **放货 / 出得多**：不仅更便宜，也应该有更多现货
- **热销 / 高价收**：除了价格变化，还可以体现在村庄收购容量或消化速度上

这样传闻板提供的消息，能同时在“价格”和“货量”两个维度被玩家感知。

## 待定内容

- 市场事件影响的是 `current`、`max`，还是两者都影响
- 库存联动发生在什么时机：世界初始生成、周期刷新、清单重建，还是全部都要影响
- 出售库存与收购容量是否需要分开处理，还是统一用一套库存修正规则
- `DEMAND / SHORTAGE / SURPLUS / RELEASE` 四类传闻，是否直接映射为库存增减方向
- 库存修正是否继续沿用现有 `delta`，还是为规则新增独立的库存字段
- 多条市场事件同时命中时，库存修正如何叠加、如何设置上下限
- 玩家卖给村庄后形成的动态库存，是否也应受本周期市场事件影响
- 与主题模板库存基准的关系应如何定义：先取主题基准，再乘市场修正，还是先按市场筛选后再生成
- 现有“库存不影响价格，只影响能不能换”的系统表述，是否需要细化为“库存不反向参与报价，但可被市场状态影响”

## 设计提醒

- 不要让“价格高”和“库存少”永远强绑定，避免所有正向事件都只剩一种体验
- 需要区分“玩家看到的卖货量”和“村庄愿意收货的容量”，两者不一定使用相同规则
- 若库存联动加入 `MarketEventRule`，应确保数据包作者能直观理解，不要把价格语义和库存语义混成一个难以维护的字段
- 传闻文本仍应保持模糊表达，不要求直接暴露具体库存数值
