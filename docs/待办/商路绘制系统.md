本文件记录商路图册中的商路绘制设计。

它从 [商路图册系统](商路图册系统.md) 中拆出，专门讨论玩家如何把已知村庄节点组织成自己的跑商路线。

## 设计定位

商路不是两点之间的一条线，而是玩家对一段可重复旅程的命名、分段、记忆和意图。

一句话定义：

> 商路是玩家创建的、可选择显示的路线图层；它由停靠点实例和路段组成，用来表达玩家自己的贸易路线理解。

商路绘制不提供经济判断，也不替玩家规划路线。它只记录玩家自己的路线语言。

## 边界

商路绘制明确不做以下事情：

- 不自动推荐路线。
- 不计算收益。
- 不寻找最短路径。
- 不绘制真实道路。
- 不改变世界地形、村庄数据、交易价格、库存或市场事件。
- 不限制玩家实际移动方向。

商路绘制回答的是：

- 我把哪些村庄组织成一条商路？
- 这条商路里有哪些停靠点？
- 每个停靠点在这条商路里扮演什么角色？
- 哪些路段是我习惯连接的？
- 哪几条商路现在需要显示在图上？

商路绘制不回答：

- 哪条商路最赚钱？
- 哪个村庄应该成为下一站？
- 哪条路径距离最短？
- 这两个村庄之间有没有真实道路？

## 核心对象

商路绘制包含三个核心对象：商路、停靠点实例、路段。

| 对象 | 含义 |
| :--- | :--- |
| 商路 | 玩家命名的一条路线图层，可以包含分支、闭环和重复节点 |
| 停靠点实例 | 某个图册节点在某条商路中的一次出现 |
| 路段 | 连接两个停靠点实例的关系 |

关键原则：

**图册节点是世界认知，停靠点实例是商路语义。**

同一个图册节点可以在同一条商路中出现多次。每一次出现都可以拥有独立的角色、备注和顺序含义。

## 商路规则

- 一条商路最少包含一条路段，也就是至少两个停靠点实例。
- 商路允许分支，因此它不是严格线性路径，而是一个局部路线网络。
- 商路允许闭环。
- 商路允许同一个图册节点在同一条商路中出现多次。
- 商路默认不显示，只有玩家选中后才叠加到节点图上。
- 玩家可以多选商路，同时显示多条路线图层。
- 未选中的商路不参与节点图显示，避免默认视图过度拥挤。

## 停靠点实例

停靠点实例表示“某个图册节点在某条商路中的一次使用”。

它不是图册节点本身。

这样设计是为了支持：

- 同一个村庄在同一条商路中多次出现。
- 同一个村庄在不同商路中承担不同意义。
- 同一个村庄在同一条商路的不同位置拥有不同角色。

示例：

```text
平原粮仓 -> 雪地驿站 -> 平原粮仓
```

这里两个“平原粮仓”可以是同一个图册节点，但它们是两个停靠点实例。第一次可以是“进货”，第二次可以是“回程补给”。

### 停靠点字段

停靠点实例可以考虑包含以下字段：

| 字段 | 作用 |
| :--- | :--- |
| 停靠点 ID | 区分同一节点在同一商路中的多次出现 |
| 引用节点 ID | 指向商路图册中的村庄节点或线索节点 |
| 自定义角色 | 玩家定义该停靠点在本商路中的意义 |
| 备注 | 玩家记录更具体的停靠意图 |
| 排序或布局信息 | 辅助商路编辑和列表展示 |

停靠点角色允许玩家自定义。系统不内置“进货”“出货”“补给”等固定语义，也不根据角色触发经济计算。

## 路段

路段连接两个停靠点实例。

路段表达的是玩家对路线关系的记录，不是世界中的真实道路。

### 路段规则

- 路段只能连接同一条商路内的停靠点实例。
- 同一对图册节点之间可以在不同商路中出现多条不同路段。
- 同一对停靠点实例之间是否允许重复路段，后续再定。
- 路段可以有方向，也可以标记为双向。
- 路段方向表达玩家习惯的跑法或货物流向，不限制玩家实际行动。

### 路段字段

路段可以考虑包含以下字段：

| 字段 | 作用 |
| :--- | :--- |
| 路段 ID | 区分商路内不同连接 |
| 起点停靠点 ID | 指向商路内一个停靠点实例 |
| 终点停靠点 ID | 指向商路内一个停靠点实例 |
| 方向模式 | 单向或双向 |
| 备注 | 玩家记录这段路的用途或注意事项 |

## 商路字段

商路可以考虑包含以下字段：

| 字段 | 作用 |
| :--- | :--- |
| 商路 ID | 玩家数据中的唯一标识 |
| 名称 | 玩家自定义商路名 |
| 颜色 | 多选显示时区分商路 |
| 状态 | 固定枚举，用于区分草图、常跑、暂停等整理状态 |
| 停靠点实例列表 | 这条商路包含的停靠点 |
| 路段列表 | 这条商路内部的连接关系 |
| 备注 | 玩家对整条商路的说明 |

当前阶段商路状态使用固定枚举，不允许玩家自定义状态名称。

初始状态枚举：

- 草图：还在规划或试跑中的商路。
- 常跑：玩家认为已经稳定、经常使用的商路。
- 暂停：暂时不跑，但仍希望保留在商路列表中的商路。

状态只表达玩家自己的整理习惯，不代表系统对路线价值的判断。

当前阶段不添加归档功能，因此不提供归档状态、默认隐藏规则或找回入口。需要降低列表噪音时，先使用“暂停”状态承载。

## 显示规则

商路是节点图上的可选图层。

默认节点图显示：

- 村庄节点。
- 线索节点。
- 玩家当前位置。
- 当前目标。

默认节点图不显示：

- 未选中的商路。
- 未选中商路的路段。
- 未选中商路的停靠点角色。

选中一条商路时：

- 高亮该商路包含的停靠点实例。
- 绘制该商路包含的路段。
- 显示停靠点角色或备注摘要。
- 使用商路颜色区分路线图层。

多选商路时：

- 同时绘制多条商路。
- 用颜色、线型或图层顺序区分。
- 重叠路段使用小幅平行偏移线显示。

## 重复停靠点与重叠路段

重复停靠点和重叠路段遵循同一个视觉原则：

**地图上的真实节点位置不动，商路语义通过轻微偏移、编号和选中态展开。**

### 重复停靠点

同一个图册节点在一条商路中出现多次时，不在地图上生成多个大型节点。

显示规则：

- 图册节点本体始终保留在真实 `x/z` 坐标位置。
- 商路被选中时，在节点周围显示小号停靠点实例徽标。
- 停靠点实例徽标围绕节点轻微环形偏移。
- 徽标可以显示该停靠点在商路中的序号、短角色或其他紧凑标记。
- 鼠标悬停徽标时显示该停靠点实例的自定义角色和备注。
- 点击徽标时，右侧详情面板显示停靠点实例，而不是图册节点本身。
- 如果同一节点在当前商路中只出现一次，可以只显示商路高亮外圈，不强制显示编号徽标。

示例：

```text
平原粮仓 -> 雪地驿站 -> 平原粮仓
```

两个“平原粮仓”引用同一个图册节点。节点本体只显示一个，但节点周围可以展开两个停靠点实例徽标，用于表达第 1 站和第 3 站的不同角色。

### 重叠路段

多选商路时，如果多条商路包含相同端点的路段，不能完全画在同一条中心线上。

显示规则：

- 单条商路选中时，路段优先画在两个节点或停靠点实例锚点的中心连线上。
- 多条商路选中且路段端点重叠时，按商路颜色画成几条小幅平行偏移线。
- 偏移距离应保持很小，让玩家理解它们属于同一段节点关系，而不是不同真实道路。
- 方向箭头放在线段中部，并跟随偏移线。
- `A -> B` 与 `B -> A` 视为同一组重叠线，但箭头方向相反。
- 双向路段可以使用双箭头，或使用无箭头线加端点标记。

### 实例锚点

数据上，路段连接的是停靠点实例。

视觉上，路段应尽量连接停靠点实例徽标或实例锚点；当实例未展开时，路段可以退回连接图册节点中心。

交互规则：

- 选中或悬停某条商路时，展开该商路相关的停靠点实例和对应路段。
- 多选商路时，默认保持紧凑显示，只对当前悬停或正在编辑的商路展开完整实例细节。
- 不把重复实例拉离真实节点太远，避免破坏真实坐标感。
- 不让所有重复实例常驻展开，避免节点图噪音过高。

## 商路区域

商路创建入口放在商路图册主界面左侧栏下半区域。

左侧栏上半保留村庄节点列表，下半作为商路区域。商路区域负责承载商路列表、创建入口和多选显示控制。

这样划分的理由：

- 左侧栏统一承载“玩家已掌握对象”的列表。
- 村庄节点和商路图层都是玩家图册认知，但层级不同，适合上下分区。
- 商路默认不显示，需要一个明确的选择区域来控制哪些商路叠加到节点图上。
- 右侧详情面板保留给当前选中对象，不承担商路总入口职责。

商路区域应至少支持：

- 创建新商路。
- 选择单条商路显示。
- 多选商路同时显示。
- 进入商路编辑。
- 查看商路名称、颜色和状态。

### 第一版实现口径

第一版实现采用轻量编辑流，而不是完整绘图编辑器：

- 左侧上半是村庄节点列表，左侧下半是商路区域。
- 创建商路时，玩家先选中一个节点并在商路区域标记为起点，再选中另一个节点创建商路。
- 新建商路默认不显示，需要玩家在商路区域显式切换显示状态。
- 商路列表支持多条商路同时切换显示。
- 选中商路后，右侧详情面板进入商路编辑模式。
- 商路编辑模式支持改名、切换固定状态、选择停靠点、编辑停靠点角色和备注、删除非必要停靠点、切换路段方向。
- 在选中商路后，玩家可以把当前选中的节点继续追加为停靠点；同一节点可以被重复追加。
- 追加停靠点会从当前选中的停靠点连出一条新路段，因此首版已经允许分支，但还没有专门的分支编辑器。
- 中央节点图只绘制被显式显示的商路。
- 选中商路或存在重复停靠点时，节点周围显示停靠点实例徽标。
- 多条显示商路存在重叠端点路段时，使用小幅平行偏移线区分。

## 不确定性

商路允许使用线索节点和失效节点。

- 线索节点加入商路，表示玩家计划或猜测中的停靠点。
- 线索节点被贸易站确认后，对应停靠点继续引用同一个图册节点。
- 当前线索被取消后，线索节点仍保留，因此它仍可作为商路中的未确认停靠点。
- 失效节点加入商路，表示玩家保留这段旧路线记忆，但需要现场重新确认。

## 多人边界

商路属于玩家个人图册认知。

默认规则：

- 每个玩家拥有自己的商路、停靠点实例、路段、角色和备注。
- 服务器不会因为某个玩家创建商路，就自动同步给其他玩家。
- 图册物品丢失、交易或重新合成，不应导致玩家个人商路丢失。

后续可以考虑主动分享机制，例如抄录商路、队伍商路或共享商队图册。但这些属于扩展功能。

## 待细化问题

- 分支编辑的具体交互。
- 路段方向和双向状态的视觉表现。
