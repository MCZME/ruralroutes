## 问题

当前周期基于服务器 `gameTime` 计算：

```java
周期索引 = gameTime / (cycleDays * 24000)
```

`gameTime` 是服务器累计 ticks，玩家离线期间也在增长。这导致：
- 玩家离线几天后再上线，周期可能已推进多个周期
- 周期刷新与玩家实际经历的昼夜循环脱节

## 目标

提供基于玩家实际经历的昼夜循环的周期计算方式，让玩家离线时周期不推进。

## 待实现内容

### 方案 A：节点级时间戳

每个商业节点独立记录玩家上次交互时的 `gameTime`，周期推进基于节点自身的时间戳计算。

**优点**：
- 简单直接，玩家离线不影响节点周期
- 各节点可独立推进

**缺点**：
- 同一村庄的多个方块（贸易站、展示柜、传闻板）需要同步状态
- 玩家可能通过在不同节点间切换"卡"周期

### 方案 B：玩家级时间追踪

记录每个玩家上次交互时的游戏时间，周期推进基于玩家个人时间计算。

**优点**：
- 每个玩家有独立的周期进度
- 更符合"个人旅程"的体验

**缺点**：
- 需要额外存储玩家数据
- 多人环境下同一村庄不同玩家可能看到不同市场状态

### 方案 C：混合模式

提供配置选项，让服务器管理员选择周期计算方式：

| 配置值 | 说明 |
|--------|------|
| `server_time` | 当前模式，基于服务器 gameTime |
| `node_time` | 基于节点上次交互时间 |
| `player_time` | 基于玩家上次交互时间 |

## 待决策
