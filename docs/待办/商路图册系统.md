# 商路图册系统

```
创建日期: 2026-05-27
最后更新: 2026-05-29
版本: v1.20
状态: 讨论
关联系统: 村庄资源身份, 交换机制, 市场波动, 模组进度
```

本文件记录商路图册系统的待定设计。它从 [商路玩法阶段](../玩法/商路玩法阶段.md) 中拆出，作为后续细化村庄节点、图册定位和 GUI 交互的集中入口。

## 设计定位

商路图册是玩家长期持有的商路信息中心。

它不是单一的村庄查找工具，也不是经济情报工具。它负责把村庄作为节点显示在一幅图上，帮助玩家记录已知村庄、选择目标村庄，并承载玩家后期手动规划的商路图层。

一句话定义：

> 商路图册记录“我知道哪些村庄、它们在哪里、我打算怎样跑商”，但不替玩家判断哪里最赚钱。

## 基础设定

| 项目 | 设定 |
| :--- | :--- |
| 名称 | 商路图册 |
| 形式 | 物品 |
| 获取方式 | 玩家合成 |
| 使用方式 | 右键打开 GUI |
| 核心视图 | 村庄节点图 |
| 核心对象 | 村庄节点、商路图层入口 |
| 定位原则 | 第一个村庄免费，后续主动寻找需要支付成本 |

## 信息边界

商路图册只提供村庄节点与路线规划信息，不提供经济判断。

| 信息类型 | 是否由商路图册提供 | 说明 |
| :--- | :--- | :--- |
| 村庄节点位置 | 是 | 显示已知或已定位的村庄节点 |
| 玩家当前位置 | 是 | 在节点图上显示玩家当前位置，用于判断自己与已知节点的相对关系 |
| 村庄风格 | 是 | 用于区分平原、沙漠、热带草原、针叶林、积雪平原等路线方向 |
| 村庄主题 | 已记录后可显示 | 线索节点不提前揭示具体主题 |
| 当前目标 | 是 | 玩家可选择一个节点作为当前目的地 |
| 商路图层 | 是 | 由 [商路绘制系统](商路绘制系统.md) 定义，用于表达玩家自己的跑商计划 |
| 玩家标记 | 可以 | 可用于收藏节点、自定义短名、标记目标状态 |
| 当前交易、价格、库存 | 否 | 由贸易站负责 |
| 代表性卖品 | 否 | 由展示柜负责 |
| 当前市场传闻 | 否 | 由传闻板负责 |
| 供需摘要或利润判断 | 否 | 避免图册替代玩家判断和村庄交互 |

因此，商路图册回答的是：

- 我知道哪些村庄？
- 它们大概在哪里？
- 它们属于哪种村庄风格？
- 我现在要去哪个节点？
- 我还想找哪种风格的村庄？
- 我自己规划了哪些商路？

它不回答：

- 哪个村庄现在最赚钱？
- 哪个村庄当前缺什么？
- 哪个村庄库存是否还够？
- 哪个物品当前价格最好？

## 核心对象

商路图册中有两类核心对象：节点和商路图层入口。

| 对象 | 含义 | 生成方式 |
| :--- | :--- | :--- |
| 节点 | 图册中的村庄相关记录，分为结构线索和商业节点记录 | 结构定位、贸易站交互等方式产生 |
| 商路图层入口 | 玩家自定义商路的查看与编辑入口 | 玩家手动创建、编辑、选择显示 |

核心原则：

**商业节点事实属于世界，图册认知和商路图层属于玩家。**

## 合成配方

商路图册应是早期可获得、但不是出生点白送的长期工具。

推荐首版配方：

```text
纸    纸    纸
皮革  指南针  皮革
纸    书    纸
```

产出：`商路图册 x1`

设计理由：

- 纸体现地图与记录。
- 书体现长期图册，而不是一次性定位道具。
- 指南针提供定位合理性，并要求玩家完成基础铁与红石进度。
- 皮革体现封面与装订，让图册比普通地图更有重量。
- 不需要村庄资源、货币、绿宝石或贸易站材料，避免玩家在找到第一个商业节点前被配方卡住。
- 不使用空地图作为核心材料，避免入口成本被空地图自身的纸和指南针成本抬得过高。

## 村庄节点

节点表示一个玩家已知的村庄相关记录。它可能只是结构线索，也可能是已确认的商业节点记录。

### 节点来源

节点可以通过以下方式进入图册：

- 首次打开商路图册时，玩家先选择一种村庄风格，再免费获取对应风格的入口村庄线索，用于解决模组入口问题。
- 玩家主动使用图册请求附近村庄线索。
- 玩家与贸易站交互后，图册确认该村庄的商业节点信息。
- 玩家在合成图册前与贸易站交互，也会先记录到玩家数据中；之后打开图册时直接显示。
- 单纯进入村庄结构范围或 POI 意义上的村庄，首版不自动创建图册节点。

### 结构线索与商业节点

Minecraft 的村庄结构和村庄不是同一个概念。

- 村庄结构是世界生成结构，例如 `minecraft:village_plains`、`minecraft:village_desert` 等。
- 村庄是运行时由村民、床、工作方块等 POI 条件形成的概念。
- 商路图册的主动定位搜索的是村庄结构，只能产生“结构线索”。
- 结构线索不代表当地一定存在当前有效的 POI 村庄，也不代表已经存在贸易站、主题或商业数据。
- 商路图册确认商业节点的权威入口是贸易站交互。

因此，图册定位得到的是线索节点；玩家与贸易站交互后，才创建或合并为已记录节点。

### 村庄寻找成本

商路图册的主动寻找遵循：

**第一个村庄免费，后续扩张付费。**

第一个免费线索用于解决模组入口问题，不应要求玩家先拥有货币或贸易资源。后续主动寻找村庄则需要支付制图成本，让“扩张节点图”成为跑商收益的一种消耗，而不是无限免费刷新地图信息。

| 寻找方式 | 触发条件 | 成本 | 结果 | 设计目的 |
| :--- | :--- | :--- | :--- | :--- |
| 首次入口定位 | 玩家尚未领取过入口线索，玩家数据中没有任何图册节点，并选择一种村庄风格 | 免费 | 对所选风格执行一次结构搜索，并添加线索节点 | 保证玩家能可靠进入模组主循环，同时让玩家决定第一条路线方向 |
| 风格定位 | 玩家选择平原、沙漠、热带草原、针叶林、积雪平原等风格 | 付费 | 查询或搜索对应村庄结构，尝试添加该风格线索节点 | 允许玩家有方向地补路线缺口，但不提供利润答案 |
| 贸易站确认 | 玩家与贸易站交互 | 免费 | 创建、合并或升级为已记录节点，并允许显示更明确的主题信息 | 鼓励玩家真正到场确认村庄身份 |

结构搜索口径：

- Minecraft 的结构搜索以“最近结构”为基础，图册不设计成全世界任意枚举未知村庄。
- 村庄按结构拆分为五类：`minecraft:village_plains`、`minecraft:village_desert`、`minecraft:village_savanna`、`minecraft:village_taiga`、`minecraft:village_snowy`。
- 结构搜索在主进程中执行，主动定位必须控制性能成本。
- 一次玩家操作最多触发一次结构搜索，不同时查询五种村庄结构。
- 首次入口定位不自动推断风格，也不使用默认风格；必须由玩家在五种村庄风格中选择一种。
- 后续主动定位不存在“普通定位”；玩家每次都必须明确选择一个村庄风格。
- 风格定位只查询所选风格对应的村庄结构。
- 搜索失败或遇到重复候选时，不自动切换到其他风格继续搜索，避免一次操作隐含多次结构搜索。
- 因为搜索基于玩家当前位置，图册扩张应鼓励玩家旅行到新区域后再次定位，而不是站在原地连续购买远处节点。
- 是否使用 `#minecraft:village` 这类结构标签作为单次“任意村庄”搜索，需要先做性能验证；在未验证前，设计上不依赖它。

搜索流程：

1. 玩家选择一个村庄风格。
2. 系统检查玩家是否正在进行定位；若已有定位未完成，则阻止本次操作。
3. 系统检查玩家是否已有当前未取消的未确认线索；若有，则阻止本次定位。
4. 系统按玩家当前拥有的图册节点数量计算本次成本，并在搜索期间锁定或预扣这笔成本。
5. 系统查询 level 层级已定位村庄结构表，寻找同风格、玩家尚未拥有的最近缓存候选。
6. 若缓存命中，直接给玩家添加线索，不调用结构搜索。
7. 若缓存未命中，调用可跳过已定位结构表的结构搜索。
8. 若搜索命中新候选，将其写入 level 表，再写入玩家图册认知。
9. 若成功新增线索节点，本次成本正式消耗，并解除定位锁。
10. 若搜索失败、重复候选、结果无法新增为玩家线索或过程异常，退还已锁定或预扣成本，给出提示，并解除定位锁。

结构定位缓存：

- 在 level 层级维护一个已定位村庄结构表，记录服务器已经查找过的村庄结构候选。
- 表项至少包含：唯一 ID、维度、村庄结构类型、结构位置。
- 玩家请求某一风格定位时，系统先查询该 level 表，从同风格缓存中寻找玩家尚未拥有的最近线索。
- 如果缓存中能找到可用候选，直接给玩家添加线索，不再调用原版结构搜索。
- 如果缓存中没有可用候选，再调用结构搜索。
- 结构搜索返回的新候选需要写入 level 表，再写入玩家图册认知。
- level 表是性能缓存和重复规避依据，不等同于已确认商业节点；只有贸易站交互才能确认商业节点。
- level 表不需要自动过期。村庄结构位置是世界生成事实，只要玩家不从文件层面删除对应区块，缓存记录就可以长期保留。

结构搜索调整：

- 原版结构搜索通常只返回最近结构；如果最近结构已经在 level 表中，重复调用会一直返回同一个结果。
- 商路图册需要封装或调整结构搜索方法，使其可以跳过 level 表中已有的同类型村庄结构。
- 跳过规则只作用于 level 层级的“已定位结构缓存”；玩家是否已经知道某个节点由玩家数据决定。
- 玩家个人图册用于过滤缓存候选，避免把同一个线索重复发给同一名玩家。
- 调整后的搜索仍然必须保持一次玩家操作最多一次结构搜索。
- 搜索半径直接沿用原版结构搜索半径，不额外定义图册专属搜索半径。
- 如果跳过缓存后仍找不到新候选，应返回失败反馈，并退还成本。

成本口径：

- 后续主动定位优先消耗自建货币，作为图册自身的制图消耗，不进入任何村庄库存。
- 风格定位采用递增成本：玩家拥有的图册节点数量越多，下一次定位成本越高。
- 递增成本用于让大规模扩张节点图逐渐变成跑商收益的消耗，而不是站在原地连续刷新线索。
- 搜索期间可以先锁定或预扣成本；只有成功新增线索节点后才正式消耗。
- 如果搜索范围内没有合法目标、重复候选、已有未确认线索或其他原因导致未新增线索，必须退还成本并给出提示。
- 首次免费资格也应在成功新增入口线索后才视为已使用；如果所选风格搜索失败，不消耗这次免费资格。
- 如果玩家在获得图册前已经通过贸易站交互拥有已记录节点，首次入口定位不再出现，后续主动定位按付费规则执行。
- 贸易站确认永远免费，避免玩家因为没有付费而不敢探索；单纯自然路过不产生图册节点。
- 首次免费资格记录在玩家数据中，不绑定到单个图册物品，避免玩家反复合成图册刷免费线索。

操作冷却与反馈：

- 每名玩家同一时间只能有一个定位操作。
- 定位搜索完成前，玩家不能再次发起定位。
- 搜索成功时，新增线索并消耗成本。
- 搜索失败、无可用候选、重复候选或已有未确认线索时，退还成本并显示提示。

未确认线索限制：

- 图册同一时间只锁定一个“当前未确认线索”，它决定玩家是否可以再次发起主动定位。
- 玩家可以取消当前未确认线索，再重新支付成本寻找下一个目标。
- 取消只解除这次定位限制，不删除原有线索节点。
- 取消后的线索节点仍保留在图册中，玩家之后仍可前往对应村庄并通过贸易站确认它。
- 玩家再次主动定位后，新得到的线索成为新的当前未确认线索。
- 玩家与贸易站交互产生的已记录节点不受该限制，可以直接加入图册。
- 如果定位结果已经在图册中存在，不新增重复节点，也不重复收费。

### 节点状态

节点状态用于区分玩家对该村庄的了解程度。

| 状态 | 含义 |
| :--- | :--- |
| 线索节点 | 来自结构搜索，表示某处存在对应风格的村庄结构线索，尚未确认商业节点 |
| 已记录节点 | 玩家与贸易站交互过，已确认这是一个可用于跑商的商业节点 |
| 失效节点 | 曾经记录为商业节点，但玩家再次交互贸易站时发现附近已不满足村庄条件，当前不可作为有效商业节点 |

### 节点规则

- 图册定位只新增线索节点，不自动新增边。
- 线索节点只显示必要的路线信息，不提前揭示主题、交易、库存或行情。
- 玩家抵达结构附近不会自动改变节点状态。
- 玩家与贸易站交互后，节点可以创建、合并或升级为已记录节点。
- 当图册中同时存在多个同风格线索节点时，贸易站确认应优先匹配同维度、同风格且距离当前贸易站最近的线索，避免不同村庄被错误合并。
- 图册不会主动扫描或后台更新已记录节点是否仍然有效。
- 已记录节点只有在玩家再次交互对应贸易站时才做有效性检查。
- 如果贸易站交互时发现旁边不再存在有效村庄，节点标记为失效节点。
- 如果失效节点对应贸易站再次满足村庄条件并能正常确认商业节点，节点可以恢复为已记录节点。
- 如果定位到的村庄已经存在于图册中，不新增重复节点。
- 首次入口定位免费；后续主动定位需要支付成本。
- 付费只约束“主动寻找新线索”，不约束查看、设为目标或贸易站确认。
- 主动定位只承诺“从玩家当前位置寻找最近候选”，不承诺每次都发现全新的村庄。

## 商路图层

商路图册承载玩家自定义商路，但商路定义、停靠点实例、路段、分支和多选显示规则不在本文展开。

详细设计见 [商路绘制系统](商路绘制系统.md)。

本文只保留图册侧边界：

- 商路图层属于玩家个人图册认知。
- 商路图层只在玩家打开显示后叠加到节点图上。
- 商路图层不改变世界地形、村庄数据、价格、库存或市场事件。
- 图册节点提供商路绘制的基础对象，商路绘制系统负责解释这些节点如何组成路线图层。

## GUI 布局与交互

商路图册 GUI 应首先是“地图式工具”，而不是完整地形地图或经济分析面板。

首版目标：

- 让玩家知道自己已掌握哪些村庄节点。
- 让玩家选择当前目标村庄。
- 让玩家用一次明确操作请求新的村庄线索。
- 让玩家在后期把自己的跑商路线画出来。

### 首次打开流程

玩家第一次打开商路图册时，不直接进入完整节点图，也不自动搜索村庄。

首次界面只做一件事：让玩家选择第一个入口村庄的风格。

如果玩家在合成商路图册前已经与贸易站交互并拥有已记录节点，首次打开图册时直接进入主界面，不再触发免费入口定位流程；后续主动定位按付费规则执行。

可选项：

- 平原村庄
- 沙漠村庄
- 热带草原村庄
- 针叶林村庄
- 积雪平原村庄

每个选项应使用村庄风格图标和一句极短描述，例如“平原：稳定起步”。玩家确认后，图册对所选风格执行一次免费结构搜索。

首次搜索结果：

- 成功：新增一个线索节点，并进入主界面。
- 失败：返回风格选择界面，不消耗首次免费资格。
- 重复候选：如果因数据同步等原因发现该入口节点已存在，不重复新增，也不消耗额外成本。

### 主界面布局

主界面采用三栏结构。左栏再分为上下两块：上半显示村庄节点列表，下半显示商路区域。

| 区域 | 内容 | 作用 |
| :--- | :--- | :--- |
| 左侧上半：村庄列表 | 已知节点列表，显示风格、状态、短名、收藏和当前目标标记 | 快速浏览与选择节点 |
| 左侧下半：商路区域 | 商路列表、创建入口和多选显示控制 | 管理玩家自己的商路图层 |
| 中央节点图 | 按真实世界坐标等比例显示村庄节点、玩家当前位置和显示中商路图层 | 表达玩家已知的节点与路线认知 |
| 右侧详情面板 | 当前图中选中对象的详情与操作 | 解释节点、边、商路与当前目标或编辑中商路的关系 |

中央节点图不是完整地形地图，但节点位置应使用真实世界 `x/z` 坐标等比例映射。玩家可以拖动画布、缩放视图、点击节点或显示中商路的边，并在图上看到自己的当前位置。

### 左侧栏

左侧栏分为村庄列表和商路区域。

村庄列表占左侧栏上半区域，用于快速管理已知节点。

显示信息：

- 村庄风格图标。
- 节点状态：线索、已记录、失效。
- 玩家自定义短名；未命名时使用风格加大致坐标或方向。
- 当前目标标记。
- 收藏标记。

列表可以按状态、风格或收藏过滤，但首版不需要复杂排序。

商路区域占左侧栏下半区域，用于管理玩家自己的商路图层。

显示信息：

- 商路名称。
- 商路颜色或状态标记。
- 是否当前显示。
- 是否显示。

可用操作：

- 创建商路。
- 选择或取消选择商路图层。
- 进入商路编辑。
- 通过固定状态区分草图、常跑和暂停。

当前阶段不提供商路归档功能。需要让某条商路不显示时，通过商路区域取消显示；需要降低列表噪音时，先使用“暂停”状态。

### 中央节点图

节点图是商路图册的核心视图。

坐标映射规则：

- 首版采用真实比例地图。
- 村庄或结构线索的世界 `x` 坐标映射为节点图横向坐标。
- 村庄或结构线索的世界 `z` 坐标映射为节点图纵向坐标。
- 玩家当前位置使用同样的 `x/z` 坐标映射规则，并应与节点一起参与当前视图的缩放参考范围。
- 不使用半抽象布局，也不允许玩家手动拖动节点改变其地图位置。
- 玩家拖动的是画布视角，不是节点本身。
- 支持缩放、拖动画布、居中到当前目标或选中节点。
- 不绘制真实地形、区块、道路或群系底图，只绘制节点、玩家当前位置、显示中商路图层和必要的方向/距离辅助信息。

节点显示规则：

| 节点状态 | 显示方式 | 信息 |
| :--- | :--- | :--- |
| 线索节点 | 半透明或虚线边框 | 只显示风格、方向或大致位置 |
| 已记录节点 | 实色节点加贸易站标记 | 显示风格、短名、收藏、目标状态，可显示主题名 |
| 失效节点 | 灰色或警示标记 | 显示最后记录的风格、短名、目标状态，并提示需要现场重新确认 |

节点图还应显示玩家当前位置标记，便于玩家判断自己和已知节点的相对位置。

节点图不显示当前交易、价格、库存、传闻或利润判断。

### 全局交互状态

商路图册主界面需要区分三类状态，避免把“列表显示”和“地图选中”混在一起。

| 状态 | 含义 | 数量规则 | 主要入口 |
| :--- | :--- | :--- | :--- |
| 显示中商路 | 商路列表中被打开显示的路线图层 | 可多条 | 左侧商路区域 |
| 图中选中对象 | 当前地图焦点对象，可以是节点、边或商路 | 最多一个 | 中央节点图 |
| 编辑中商路 | 当前正在创建或编辑的商路 | 最多一条 | 左侧商路区域的创建或编辑入口 |

显示中商路只表示“这条商路的图层可见”，不等于它在地图中被选中。图中选中对象是另一套状态，同一时间只能有一个。

#### 普通浏览模式

普通浏览模式用于查看节点图、选择目标和检查已显示商路。

交互规则：

- 单击节点：选中该节点。
- 单击边：选中该边。
- 单击图中空白处：取消当前图中选中对象。
- 再次单击已选中的同一节点或边：保持选中，不取消。
- 双击显示中商路的边：选中该边所属商路。
- 双击只属于一条显示中商路的节点：选中该节点所属商路。
- 如果节点属于多条显示中商路，双击节点本体不直接选中商路，玩家需要双击对应停靠点标记。
- 双击停靠点标记：选中该标记对应的商路。

中央节点图只对显示中商路提供边、停靠点标记和商路双击命中。未显示商路不参与地图交互。

#### 商路编辑模式

玩家从左侧商路区域点击创建或编辑入口后，中央节点图进入商路编辑模式。

编辑模式只作用于编辑中商路：

- 新建商路时，地图先等待玩家选择起点节点，再选择终点节点创建第一条边。
- 编辑已有商路时，地图只强化编辑中商路的可编辑节点、停靠点标记和边。
- 单击节点用于创建、追加或选择编辑目标，具体行为由当前编辑步骤决定。
- 单击边选中编辑中商路的一条边，用于查看或调整路段。
- 单击停靠点标记选中编辑中商路中的一次停靠点实例。
- 单击空白处清空当前编辑选择，但不退出编辑模式。
- 编辑模式中暂不使用双击选中商路，避免和创建、追加、编辑动作冲突。

### 右侧详情面板

右侧详情面板不应拆成彼此独立的“节点页”和“商路页”。它是统一的上下文检查器，用来解释当前图中选中对象，以及该对象和当前目标、显示中商路、编辑中商路的关系。

右侧详情面板根据当前图中选中对象显示：

- 选中节点：显示节点状态、风格、位置、目标状态，以及该节点是否参与显示中商路或编辑中商路。
- 选中边：显示边所属商路、两端停靠点、方向模式，以及是否属于编辑中商路。
- 选中商路：显示商路名称、状态、是否显示、停靠点数量和路段数量。
- 未选中对象：显示空状态或当前编辑提示。

可用操作应跟随上下文出现，而不是固定展示两套页面式操作。

### 节点上下文字段

当图中选中对象是节点时，右侧详情面板可显示：

- 村庄风格。
- 节点状态。
- 大致方向、距离或坐标。
- 玩家短名和备注。
- 是否收藏。
- 是否当前目标。
- 已记录节点的主题名。
- 失效节点的状态提示。

可用操作：

- 设为目标。
- 取消目标。
- 收藏 / 取消收藏。
- 改名或编辑备注。
- 取消当前未确认线索，但保留对应线索节点。
- 发起连接路线。

当图中选中对象是边、商路或停靠点标记时，右侧详情面板显示规则见 [商路绘制系统](商路绘制系统.md)。

### 定位交互

主界面提供一个“定位”入口，用于请求新的村庄线索。

定位入口只包含一种操作：

| 操作 | 成本 | 搜索方式 | 说明 |
| :--- | :--- | :--- | :--- |
| 风格定位 | 递增成本 | 玩家选择一种风格，先查 level 缓存，必要时只查该风格结构 | 用于扩张节点图和补路线缺口 |

确认定位前必须显示本次成本。定位完成前禁用新的定位操作。只有成功新增线索节点后才正式消耗成本；失败、重复候选、已有当前未取消线索阻止定位时，退还成本并给出明确提示。玩家也可以主动取消当前未确认线索；取消后保留该线索节点，并重新允许下一次定位。

### 商路编辑入口

图册主界面的商路创建、选择显示和编辑入口放在左侧下半的商路区域。

具体编辑流程见 [商路绘制系统](商路绘制系统.md)。

## 数据存储与多人

商路图册采用混合存储：

**世界存事实，玩家存认知和规划，物品只做入口。**

### 数据归属

| 数据 | 存储位置 | 说明 |
| :--- | :--- | :--- |
| 商业节点数据 | 区块数据或现有村庄商业节点数据 | 这是服务器权威的商业节点数据，随对应区块保存 |
| 已定位村庄结构缓存 | Level 数据 | 记录服务器已经通过图册定位查找过的村庄结构，用于复用结果和跳过重复结构 |
| 结构线索的维度、结构类型、坐标 | 玩家数据 | 这是玩家通过图册定位获得的线索，不等同于已确认商业节点 |
| 玩家已知节点列表 | 玩家数据 | 表示“这个玩家知道哪些村庄” |
| 节点状态 | 玩家数据 | 线索、已记录、失效是玩家个人图册状态 |
| 当前未确认线索引用 | 玩家数据 | 表示当前占用主动定位资格的线索节点；取消时只清除此引用，不删除节点 |
| 当前目标、收藏、短名、备注 | 玩家数据 | 属于个人路线管理 |
| 商路图层数据 | 玩家数据 | 商路、停靠点实例和路段属于玩家规划，不是世界事实 |
| 首次免费入口线索是否已使用 | 玩家数据 | 防止通过反复合成图册刷新免费定位 |
| 图册物品组件 | 轻量入口信息 | 不存完整节点图，可只存样式、绑定标记或打开图册所需的轻量信息 |

图册物品不承担完整数据存储。它更像打开玩家个人商路日志的工具，而不是一本会随物品复制、丢弃和交易的完整地图数据库。

### 多人共享

多人服务器中，商路图册默认不自动共享节点信息。

默认规则：

- 每个玩家拥有自己的已知节点、节点状态、当前目标、收藏、备注和商路图层。
- 区块数据可以保存已确认商业节点数据，但“区块上存在这个商业节点数据”不等于“所有玩家都知道这个节点”。
- 玩家与贸易站交互时，只更新该玩家自己的图册认知；此时才检查节点是否有效、失效或恢复。
- 玩家丢失、交易或重新合成图册，不应丢失个人已知节点和路线规划。
- 服务器不会因为某个玩家发现村庄，就自动点亮其他玩家的图册。

后续可考虑主动分享机制，例如面对面抄录节点、复制路线、队伍图册或共享商队图册。但这些属于扩展功能，不进入默认规则。

## 阶段职责

### 起步期

玩家通常先合成商路图册。首次使用时，图册先让玩家选择一种村庄风格，再免费获取对应风格的入口村庄线索，让玩家可靠进入第一个村庄。

如果玩家在合成图册前已经遇到贸易站并完成交互，村庄贸易系统正常工作，图册数据会先记录在玩家身上；之后打开图册时直接显示这些已记录节点，不再触发首次免费入口定位。

起步期的设计重点：

- 合成成本足够低，玩家在早期原版生存中可以自然完成。
- 第一个村庄只作为模组入口，不提供交易优势。
- 第一次定位不收费，避免玩家在进入贸易系统前被货币或资源门槛卡住。
- 第一次定位不自动搜索，玩家需要主动选择平原、沙漠、热带草原、针叶林或积雪平原村庄。
- 线索节点不提前显示主题、交易、库存或行情。

### 拓路发现期

玩家开始通过图册记录多个村庄节点，并从“随机找村庄”转向“扩张自己的村庄节点图”。

拓路发现期的设计重点：

- 支持查看已知村庄节点。
- 支持付费按指定村庄风格寻找目标。
- 支持选择一个节点作为当前目的地。
- 此阶段以节点发现和风格识别为主，不要求玩家必须创建商路图层。

### 商路构建期

图册不提供经济情报，但帮助玩家把已发现的多个村庄节点组织成商路。玩家根据贸易站、展示柜和传闻板获得经济信息后，再用图册选择目的地或管理自己的商路图层。

商路构建期的设计重点：

- 支持收藏常用村庄。
- 支持玩家标记节点。
- 支持玩家在已知节点间手动创建商路图层。
- 支持玩家表达多节点、分支、闭环、重复停靠点等跑商理解。
- 不提供利润排序或最优路线推荐。

### 终局收束期

图册帮助玩家回到自己的高价值路线节点，例如工坊、染坊、冰工、驿站和粮仓。终局材料是否出现仍由贸易站、事件和合约决定。

终局收束期的设计重点：

- 不显示哪里有末影珍珠或烈焰棒。
- 不显示终局合约是否可用。
- 只帮助玩家管理已知村庄节点和目标路线。

## 待细化问题

- 图册 GUI 的具体视觉稿、图标、按钮文案和键鼠操作细节。
- 风格定位递增成本的具体阶梯、公式和货币面额配置。
- level 层级已定位村庄结构表的存档版本与兼容方案。
- 可跳过已定位结构的搜索封装或原版结构搜索调整方案。
- 商路绘制系统的数据模型和编辑交互。
- 玩家标记、收藏、短名等路线管理功能是否进入首版。
- 主动分享节点或路线是否作为后续扩展功能。
