## 问题

“规则驱动市场事件池”已经确定，但首批 `market_event_rules` 具体应该如何分层、选什么目标、怎样避免事件彼此打架，还没有形成统一设计。

如果没有这一步，后续很容易出现三个问题：

1. 规则虽然可写，但每条都像孤立样本，缺少整体结构
2. 同周期可能抽到互相抵消的镜像规则，传闻会变得无意义
3. 事件目标会滑向 `candidate/*` 这类生成标签，混淆系统语义层和内容生成层

## 目标

首批市场事件规则库需要同时满足：

1. **大多数周期都能给玩家明确的移动线索**
2. **情报文本一眼能懂，不需要玩家理解内部标签体系**
3. **村庄主题身份能通过事件继续被强化**
4. **早期常见资源优先进入市场循环，高阶资源保持少量稀有感**
5. **尽量在不增加新字段的前提下，先用现有 `target_ref + scopes + delta + weight` 跑通内容层**

## 硬规则

### 1. 事件目标优先使用系统池

首批事件规则的 `target_ref` 以以下两类为主：

- `#ruralroutes:pool/*` 系统池
- 少量精确物品 ID

不直接使用 `#ruralroutes:candidate/*` 作为市场事件目标。

原因：

- `candidate/*` 负责“本地更容易出现什么”
- `pool/*` 负责“这类资源在市场语义上是什么”

市场事件属于后者。

### 2. 首批不做简单镜像对

首批规则库不采用“同一目标 + 同一作用域 + 一条上涨 / 一条下跌”的机械镜像设计。

例如不建议同时放入：

- `GLOBAL + #pool/food +0.15`
- `GLOBAL + #pool/food -0.15`

因为当前生成器没有“互斥组”机制，镜像规则可能在同一周期同时抽中，最后只得到一条无意义的抵消结果。

首批应优先采用“带原因的单向事件”。

### 3. 一条规则只表达一个市场信号

单条规则只表达一个明确变化：

- 什么东西
- 在哪里
- 涨还是跌

不要让一条规则同时承担“丰收 + 运输中断 + 特产宣传”这类复合语义。

### 4. 高阶池后置

以下池不进入首批常见层：

- `#ruralroutes:pool/nether_goods`
- `#ruralroutes:pool/ocean_goods`
- `#ruralroutes:pool/end_goods`
- `#ruralroutes:pool/precious`

它们只进入低权重稀有层。

## 首批规则库分层

建议首批规则库采用四层：

| 层级 | 主要作用域 | 目标类型 | 建议权重 | 建议幅度 | 作用 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 常见全局层 | `GLOBAL` | 常见系统池 | 80-100 | 0.10-0.15 | 给每周期提供基础市场风向 |
| 群系区域层 | `BIOME` | 群系相关系统池 | 50-70 | 0.12-0.18 | 让不同区域开始分化 |
| 主题签名层 | `THEME` | 主题主产 / 主缺池，或少量精确物品 | 30-45 | 0.18-0.25 | 强化村庄身份，制造明确跑商点 |
| 稀有故事层 | `GLOBAL / BIOME / THEME` | 高阶池或记忆点物品 | 5-15 | 0.25-0.40 | 提供少量惊喜和长线目标 |

## 首批数量建议

在当前 `market.rule_pick_count = 3` 的前提下，建议首批规则总量控制在 **40-50 条**。

推荐配比：

| 层级 | 建议条数 |
| :--- | :--- |
| 常见全局层 | 8-10 |
| 群系区域层 | 10 |
| 主题签名层 | 15 |
| 稀有故事层 | 6-8 |

这样做的目的不是让每层平均出现，而是让加权抽取自然形成：

- 大多数周期至少出现 1 条常见事件
- 相当一部分周期出现 1 条群系或主题事件
- 只有少数周期会冒出高阶稀有事件

## 首批常见全局层

这层负责给所有周期提供“基础商路天气”。

### 适合进入首批常见层的系统池

- `#ruralroutes:pool/food`
- `#ruralroutes:pool/crop`
- `#ruralroutes:pool/wood`
- `#ruralroutes:pool/stone`
- `#ruralroutes:pool/mineral`
- `#ruralroutes:pool/leather_fiber`
- `#ruralroutes:pool/dye_decor`

### 建议事件原型

| 事件原型 | `target_ref` | 方向 | 建议 `delta` |
| :--- | :--- | :--- | :--- |
| 粮食吃紧 | `#ruralroutes:pool/food` | 上涨 | `+0.15` |
| 农货丰盈 | `#ruralroutes:pool/crop` | 下跌 | `-0.12` |
| 木料走俏 | `#ruralroutes:pool/wood` | 上涨 | `+0.12` |
| 石料滞销 | `#ruralroutes:pool/stone` | 下跌 | `-0.10` |
| 矿料抢手 | `#ruralroutes:pool/mineral` | 上涨 | `+0.15` |
| 皮货冷清 | `#ruralroutes:pool/leather_fiber` | 下跌 | `-0.10` |
| 染料热销 | `#ruralroutes:pool/dye_decor` | 上涨 | `+0.14` |

这层不追求“主题正确”，而追求“周期里总有方向感”。

## 首批群系区域层

这层负责把“市场风向”从全局，进一步压缩到玩家可前往的区域。

建议每个有村庄的群系至少有 2 条规则。

### 平原

| 原型 | 目标 | 方向 | 说明 |
| :--- | :--- | :--- | :--- |
| 丰收年 | `#ruralroutes:pool/crop` | 下跌 | 平原农产过剩 |
| 粮仓备货 | `#ruralroutes:pool/mineral` | 上涨 | 工具与农具材料紧俏 |

### 沙漠

| :--- | :--- | :--- | :--- |
| 商队延误 | `#ruralroutes:pool/food` | 上涨 | 补给到货慢 |
| 建材走热 | `#ruralroutes:pool/stone` | 上涨 | 沙漠建筑石料被追捧 |

| 陶瓦出货 | `#ruralroutes:pool/dye_decor` | 下跌 | 成品一时充足 |

## 稀有故事层

这层负责制造“偶尔会让人专程跑一趟”的长线动机。

### 适合放入稀有层的目标

- `#ruralroutes:pool/nether_goods`
- `#ruralroutes:pool/ocean_goods`
- `#ruralroutes:pool/end_goods`
- `#ruralroutes:pool/precious`
- 主题特产对应的精确物品

## 命名规范建议

建议规则 ID 按层级与主题组织，便于后续维护：

| 层级 | 建议 ID 格式 |
| :--- | :--- |
| 常见全局层 | `ruralroutes:common/<event_id>` |
| 群系区域层 | `ruralroutes:biome/<biome>/<event_id>` |
| 主题签名层 | `ruralroutes:theme/<theme>/<event_id>` |
| 稀有故事层 | `ruralroutes:rare/<event_id>` |

示例：

- `ruralroutes:common/food_shortage`
- `ruralroutes:biome/plains/harvest_year`
- `ruralroutes:theme/plains_workshop/ore_shortage`
- `ruralroutes:rare/precious_surge`

## 与情报系统的配合

当前情报系统只显示：

- 目标
- 作用域
- 方向

因此首批规则库应优先选择“光看这三个信息也能让人理解”的事件。

适合首批的表达：

- `食物 + 雪原 + 上涨`
- `木料 + 针叶林 + 下跌`
- `矿物 + 平原匠坊 + 上涨`

不适合首批的表达：

- 目标过碎但没有独特记忆点的冷门单物品
- 玩家无法从文本中推断商路意义的内部材料

## 推荐先后顺序

建议按以下顺序补首批数据：

1. 先补常见全局层
2. 再补 5 个群系区域层
3. 然后补 15 个主题签名层
4. 最后补少量稀有故事层

原因是这样最容易边做边测：

- 先验证价格变化是否可感知
- 再验证玩家是否会因为区域差异移动
- 最后再引入高记忆点和高波动内容

## 待定内容

- 首批是否需要引入“精确物品下跌型”稀有事件
- 是否要为未来生成器预留“互斥组”字段，彻底解决镜像冲突
- `market.rule_pick_count = 3` 是否在规则库扩到 50 条后仍然合适
- 稀有故事层是否需要绑定原版进度阶段
