# 贸易周期

全局统一的库存刷新节律。

## 全局统一

所有村庄使用同一刷新周期。

## 配置项

| 配置项 | 字段名 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 时间模式 | `time_mode` | 枚举 | `server_time` | 可选：`server_time`（服务器运行时间）/ `game_time`（游戏内昼夜循环） |
| 周期长度 | `cycle_days` | 整数 | 3 | 建议范围 1-30，单位为游戏日 |
| 刷新时刻 | `refresh_time` | 枚举 | `sunrise` | 可选：`sunrise` / `noon` / `sunset` / `midnight` |

### 时间模式说明

| 模式 | 说明 |
| :--- | :--- |
| `server_time` | 基于服务器累计运行时间（`gameTime`），服务器运行时间推进则周期推进 |
| `game_time` | 基于游戏内昼夜循环（`dayTime`），与游戏内实际经历的昼夜一致 |

周期刷新时，会按主题候选重建当期买卖清单，并将库存恢复为该期初始状态。

该期初始库存先由主题模板给出基准，再叠加当前周期市场事件的库存修正。

玩家在上一周期的买卖行为不跨周期保留。

## 首次刷新（固定逻辑）

节点首次创建时，所有库存先应用主题基准，再叠加当前周期的市场库存修正。

第一个周期自此刻开始计时。

此逻辑不可配置。

## 流转动力

库存为 0 时，不等刷新就出发——这是玩家继续上路的最自然理由。
