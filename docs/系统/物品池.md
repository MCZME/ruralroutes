# 物品池

物品池是《乡野商路》对可交换物品进行逻辑分组的工具，采用数据包标签实现。

## 定义

物品池是对可交换物品进行逻辑分组的工具。它并非游戏内可见的概念，而是驱动多个系统模块的底层组织结构。

## 实现形式

- **数据包标签**：使用 Minecraft 标签系统
- **命名规范**：本模组自定义池建议使用 `#ruralroutes:pool/<池名>` 格式，但这只是内部规范，非强制要求
- **候选标签**：生成层偏向内容使用独立标签，群系候选建议采用 `#ruralroutes:candidate/biome/<群系>/<类别>`，主题候选建议采用 `#ruralroutes:candidate/theme/<主题>/<类别>` 格式
- **灵活引用**：可直接使用原版标签（如 `#minecraft:planks`）或其他模组标签，无需强制转换格式
- **实现偏好**：优先复用原版与 NeoForge 现有标签；仅在标签缺位，或物品本身具有特殊稀有语义时直接列出物品
- 模组内置默认数据包提供基础池定义，玩家可通过数据包覆盖或扩展

## 核心规则

### 系统池与候选标签分工

- **系统池 `pool/*`**：服务系统层分类，回答“这是什么类资源”
- **候选标签 `candidate/*`**：服务生成层偏向，回答“这里更容易出现什么具体物品”

系统池用于价值表、主题价格修正、市场波动与传闻文本。

候选标签用于主题模板生成买卖清单时的本地内容偏向，不直接承担价值分类与市场语义。

### 多池归属

一个物品可被多个池标签包含，即属于多个池。

允许部分重叠，但每个池的语义必须准确。重叠应来自不同语义维度，而不是边界模糊。

例如：
- 绯红木、诡异木可同时属于 `wood` 与 `nether_goods`
- 龙息、鞘翅可同时属于 `end_goods` 与 `precious`

### 无主池概念

不设主池标签，不做主池判定。

### 价格修正延后

多池物品的修正系数如何选取，由主题模板设计时决定，不在物品池模块处理。

### 群系偏向不写入系统池

群系或主题的“偏向内容”不通过新增系统池表达。

例如针叶林偏向云杉木，应通过 `candidate/biome/taiga/wood` 这类候选标签解决，而不是把“云杉木”做成新的系统池。

### 第一批候选标签范围

第一批同时引入：

- **群系级候选标签** `candidate/biome/*`
- **主题级候选标签** `candidate/theme/*`

其中：

- `candidate/biome/*` 负责同群系多个主题共享的本地底色
- `candidate/theme/*` 负责在同群系内部对群系候选做主题化补充，表达工具货、石作货、补给品等更细语义

## 查询方式

直接使用原生 API，无需额外管理器：

| 查询方向 | 方法 |
| :--- | :--- |
| 物品→池 | `itemStack.getTags()` 过滤所需标签 |
| 池→物品 | `BuiltInRegistries.ITEM.getOrCreateTag(tagKey)` |

支持 `/reload` 热重载，标签数据自动更新。

## 在模组中的角色

| 角色 | 说明 |
| :--- | :--- |
| 价值表组织 | 同一池内物品遵循相近的基础价值计算逻辑 |
| 主题价格修正 | 主题通过物品池批量定义修正系数 |
| 市场波动选取 | 每次刷新以物品池为单位随机选取影响目标 |
| 情报文本生成 | 传闻板根据受影响的池生成模糊价格情报 |
| 候选内容归类 | 群系/主题通过候选标签从系统池对应的具体物品中抽样 |

## 第一批系统池边界

### `stone`

- 语义：主世界石土类地质建材池
- 纳入：普通石材、圆石、深板岩圆石、砂子、红沙、砂砾、普通砂岩、红砂岩
- 排除：下界岩、末地石、黑曜石、苔石圆石、虫蚀圆石

### `wood`

- 语义：木质资源及其一阶加工产物池
- 纳入：原木、菌柄、去皮原木/菌柄、木板、木棍、木炭
- 说明：下界木系属于木材语义，因此可与 `nether_goods` 重叠
- 排除：船、门、活板门、栅栏、箱子、木桶、木制工具、营火

### `crop`

- 语义：农业原料与田地产出池
- 纳入：小麦、胡萝卜、马铃薯、甜菜根、南瓜、西瓜、甘蔗、仙人掌
- 排除：面包、肉类、浆果、下界疣、可可豆

### `food`

- 语义：直接食用或明显作为食品流通的物品池
- 纳入：面包、常规生熟肉、鳕鱼、熟鳕鱼、鲑鱼、熟鲑鱼、甜浆果、发光浆果、金胡萝卜
- 排除：小麦、胡萝卜、马铃薯、甜菜根等仍以农作物身份为主的原料

### `mineral`

- 语义：主世界矿物成品与中间形态池
- 纳入：煤炭、粗铁、粗铜、粗金、铁锭、铜锭、金锭、红石、青金石、钻石、绿宝石
- 说明：粗矿属于 `mineral` 内部的阶段差异，不单独开新池
- 排除：矿石方块、储存块、工具装备、下界石英、荧石粉、下界合金

### `dye_decor`

- 语义：染料与装饰成品池
- 纳入：染料、普通陶瓦、带釉陶瓦、玻璃块、遮光玻璃
- 排除：骨粉、黏土球、沙子、玻璃板等生产原料或过细形态

### `leather_fiber`

- 语义：皮革与纤维类软性畜产 / 纺织材料池
- 纳入：羊毛、皮革、兔子皮、线、羽毛
- 排除：骨头、骨块、肉类、蛋、地毯、旗帜

### `ice_snow`

- 语义：天然冰雪资源池
- 纳入：雪球、雪块、冰、浮冰、蓝冰
- 排除：细雪桶、营火及其他仅具寒地风味但并非冰雪本体的物品

### `nether_goods`

- 语义：下界来源资源覆盖池
- 纳入：下界岩、下界砖、石英、荧石粉、下界疣、烈焰棒、灵魂沙、灵魂土、黑石、绯红/诡异菌柄、绯红/诡异木板、岩浆膏、恶魂泪、下界合金锭
- 说明：该池按来源组织，因此允许与 `wood`、`precious` 重叠

### `ocean_goods`

- 语义：海洋来源资源覆盖池
- 纳入：海晶碎片、海晶砂粒、墨囊、发光墨囊、鹦鹉螺壳、海洋之心、海带
- 排除：干海带

### `end_goods`

- 语义：末地系 / 末影系资源覆盖池
- 纳入：末影珍珠、紫颂果、爆裂紫颂果、潜影壳、末地石、龙息、鞘翅
- 说明：该池允许与 `precious` 重叠

### `precious`

- 语义：跨来源的高阶珍材覆盖池
- 纳入：下界合金锭、下界之星、龙息、附魔金苹果、鞘翅
- 说明：保持窄而准，不扩张成“高价值杂项池”
- 排除：钻石、绿宝石、石英、烈焰棒、潜影壳、海洋之心等仍以其他系统语义为主的物品

## 特产工具标签

| 标签 ID | 语义 |
| :--- | :--- |
| `#ruralroutes:pool/specialty` | 全局随机特产抽取用工具标签，不视为第一批常规系统池 |

### `specialty`

- 语义：全局随机特产抽取池。以“能增加村庄记忆点”为主，也允许少量终局前关键稀有物进入，用于形成全图共享的稀有发现感
- 纳入：金胡萝卜、錾制砂岩、青色染料、黄色带釉陶瓦、兔子皮、云杉船、发光浆果、营火、蓝冰、末影珍珠、烈焰棒
- 排除：面包、常规矿物、普通木板等过于常规的流通物
- 排除：鞍、铁砧等更接近功能稀有物或重型制成品的物品
- 排除：下界之星、龙息、鞘翅等已经应由 `precious` 或来源池承载的高阶稀缺物
- 说明：进入 `specialty` 不影响其原有来源语义。末影珍珠仍属于 `end_goods`，烈焰棒仍属于 `nether_goods`

## 候选标签方向

- 候选标签按“**群系 / 资源类**”组织，而不是按主题名组织
- 一个候选标签应主要对应一个系统池，避免重新发明一套平行分类
- 候选标签用于表达“本地更容易产出什么”，不用于表达“本地缺什么、想买什么”
- 同群系多个主题优先复用同一批 `candidate/biome/*`，由主题通过不同组合和抽样数量体现差异
- `candidate/theme/*` 用于在共享群系底色之上补充主题独有的语义子类，不按 `sell` / `buy` 直接切分命名
- 不是每个主题都必须创建 `candidate/theme/*`；若物品种类很少，直接在主题数据中写精确物品即可
- `candidate/theme/*` 的主要价值是为后续主题逻辑、价格修正和内容区分提供明确语义，而不是单纯为了简化主题数据书写

### 第一批候选标签组织

- `candidate/biome/*` 提供群系公共底色
  例如：`plains/wood`、`desert/stone`、`savanna/dye_decor`、`taiga/food`、`snowy/ice_snow`
- `candidate/theme/*` 提供主题级细化
  例如：`plains_workshop/tool_goods`、`desert_quarry/stonework`、`desert_dyeworks/finished_goods`、`snowy_waystation/supplies`
- 主题模板采用“核心精确物品 + biome candidate + 按需的 theme candidate + 精确特产”的组合，而不是完全依赖某一个单独标签
