## 定义

物品池是对可交换物品进行逻辑分组的工具。它并非游戏内可见的概念，而是驱动多个系统模块的底层组织结构。

## 实现形式

- **数据包标签**：使用 Minecraft 标签系统
- **命名规范**：本模组自定义池建议使用 `#ruralroutes:pool/<池名>` 格式，但这只是内部规范，非强制要求
- **灵活引用**：可直接使用原版标签（如 `#minecraft:planks`）或其他模组标签，无需强制转换格式
- 模组内置默认数据包提供基础池定义，玩家可通过数据包覆盖或扩展

## 核心规则

### 多池归属

一个物品可被多个池标签包含，即属于多个池。

### 无主池概念

不设主池标签，不做主池判定。

### 价格修正延后

多池物品的修正系数如何选取，由主题模板设计时决定，不在物品池模块处理。

## 查询方式

直接使用原生 API，无需额外管理器：

| 查询方向 | 方法 |
| :--- | :--- |
| 物品→池 | `itemStack.getTags()` 过滤所需标签 |
| 池→物品 | `BuiltInRegistries.ITEM.getOrCreateTag(tagKey)` |

支持 `/reload` 热重载，标签数据自动更新。

## 在模组中的角色

| 角色 | 说明 |
| :--- | :--- |
| 价值表组织 | 同一池内物品遵循相近的基础价值计算逻辑 |
| 主题价格修正 | 主题通过物品池批量定义修正系数 |
| 市场波动选取 | 每次刷新以物品池为单位随机选取影响目标 |
| 情报文本生成 | 传闻板根据受影响的池生成模糊价格情报 |
