# 村庄库存规则

单个村庄的库存上限、消耗方式、买卖库存关系。

## 库存上限

由主题模板定义库存基准值。

设计主题时，为每种物品直接指定该主题下的库存基准值；市场事件可在节点创建和周期刷新时，对本期库存基线做临时修正。

## 共享库存池

村庄对每种物品有一个**净持有量**，买卖共享同一库存池。

| 行为 | 库存变化 |
| :--- | :--- |
| 玩家购买 | 库存减少 |
| 玩家出售 | 库存增加（不超过上限） |

- 库存为 0 时，该物品不可购买。
- 库存达到上限时，该物品不可再向村庄出售。

### 货币例外

货币库存遵循同一共享库存池，但在“玩家向村庄支付货币”的入库场景中有一个定向例外：

- 货币条目允许 `current` 暂时超过 `max`
- 目的是保留玩家实际支付的货币，避免村庄货币库存已满时出现“交易成功但货币被虚空吞掉”
- 这只影响货币入库，不影响普通物品的收购上限，也不放宽村庄支付货币时的库存校验

## 收购限制

由共享池上限自然限制。

收购上限 = 库存上限 - 当前库存。

无须额外配额系统。

## 衍生规则

### 出售物品的归属

玩家卖给村庄的物品直接进入该物品的库存池：

- 若物品不在村庄出售清单上，玩家卖入后库存不为零，但无法买回。
- 若物品在出售清单上，玩家可卖完再买回（价格差可能不划算）。

### 刷货币防护

同一村庄对同一物品：**收购修正系数 < 出售修正系数**（即收购价 < 出售价）。

这是设计原则，技术上不强制校验，允许特别情况出现。主题模板设计时应遵循此原则。

### 刷新行为

全局刷新时，村庄会重建当期买卖清单，并按该期清单重新初始化库存。

- 出售物品以该期初始库存开局，初始值 = 主题库存基准 × 当期市场出售库存因子。
- 仅收购物品以空库存开局，但保留该期的收购空间；收购空间 = 主题库存基准 × 当期市场收购库存因子。
- 若同一物品同时出现在买卖两侧，则共享同一库存池：出售侧提供初始库存，收购侧额外提供可回收空间。

周期内的买卖行为不跨周期保留。

### 与价格的关系

库存不会反向参与报价计算，但可被市场状态影响。

也就是说：

- 价格仍只由基础价值、主题修正和市场波动因子决定
- 库存只决定“还能不能换、还能收多少”
- 市场事件可以同时影响价格和库存，但两条链路字段分离

## 规则总览

| 规则项 | 决策 |
| :--- | :--- |
| 库存上限 | 主题模板提供基准，市场状态可对当期基线做临时修正 |
| 库存池 | 买卖共享，单一净持有量 |
| 收购限制 | 由共享池上限自然限制 |
| 出售限制 | 库存为零时不可购买 |
| 货币例外 | 收取货币时允许超过当期 `max`，避免吞币；刷新后仍恢复为当期初始库存 |
| 刷货币防护 | 同村同物品收购价 < 出售价 |
| 刷新行为 | 全局刷新重建当期清单并恢复为该期初始库存，周期内行为不跨周期保留 |
