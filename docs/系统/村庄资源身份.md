# 村庄资源身份

每个村庄的资源身份由三层架构生成：**群系 → 主题 → 村庄**。

## 三层架构

### 群系层

群系定义该地貌下的资源底色，关联对应的物品池组合。

详见 [物品池](物品池.md)。

### 主题层

主题是我们手动设计的模板，介于群系与村庄之间。每个群系预设 3-5 个主题。

主题从系统池与候选标签中选取子集，定义具体的买卖倾向和辨识度。

当群系公共底色不足以表达主题时，可额外定义少量主题候选标签作为补充；若物品种类本就很少，则直接在主题数据中写精确物品即可。

示例（沙漠群系）：
- **砂岩开采村** - 专营石材类物品池
- **绿洲农场** - 专营农作物类物品池
- **染料工坊** - 专营染料与装饰类物品池

### 村庄层

村庄生成时，从所属群系的 3-5 个主题中随机选取一个。

同一群系的不同村庄因主题不同，买卖清单有本质差异。

## 主题模板结构

一个主题包含以下字段：

| 字段 | 说明 |
| :--- | :--- |
| 名称 | 如"砂岩开采村""绿洲农场" |
| 所属群系 | 该主题绑定到哪个群系 |
| 出售候选 | 该主题主要出售的候选物品池/物品，生成节点时先展开再按规则抽样 |
| 收购候选 | 该主题主要收购的候选物品池/物品，生成节点时先展开再按规则抽样 |
| 价格修正系数 | 对各物品池的出售/收购修正系数 |
| 代表性卖品 | 可通过少量高辨识度卖品形成主题记忆点，但不单独占用技术字段 |
| 特殊以物易物 | 可选，主题专属固定交换规则（玩法层填充） |
| 库存基准值 | 各物品池的库存上限 |

**价格修正系数规则**：主题修正不叠加。精确物品规则优先；若命中多个标签，则按 `price_modifiers` 的声明顺序取第一条命中规则。详见 [物品池](物品池.md)。

## 全局特产池

`#ruralroutes:pool/specialty` 仍然保留，用于描述“全局稀有记忆点商品”这一内容层分类。

当前规则：

- 它不再作为主题字段参与节点生成
- 它不再由村庄创建时单独抽取注入出售清单
- 展示柜也不再优先展示这类物品

是否能在某个村庄看到这些商品，取决于它们是否本来就进入了该主题的出售候选、固定合约或其他玩法入口。

## 清单生成流程

1. 村庄确定群系
2. 从该群系的 3-5 个主题中随机选取一个
3. 加载主题的出售候选和收购候选
4. 展开标签候选，并按 `pick` 等规则抽样生成当期买卖清单
5. 确定库存基准和价格修正，生成最终节点数据

展示柜随后从当期 `sellItems` 中随机选择展示物，不再依赖独立的特产注入链。
