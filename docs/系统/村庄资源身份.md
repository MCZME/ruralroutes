# 村庄资源身份

每个村庄的资源身份由三层架构生成：**群系 → 主题 → 村庄**。

## 三层架构

### 群系层

群系定义该地貌下的产出倾向，关联对应的物品池组合。

详见 [物品池](物品池.md)。

### 主题层

主题是我们手动设计的模板，介于群系与村庄之间。每个群系预设 3-5 个主题。

主题从系统池与候选标签中选取子集，定义具体的买卖倾向和辨识度。

当群系公共底色不足以表达主题时，可额外定义少量主题候选标签作为补充；若物品种类本就很少，则直接在主题数据中写精确物品即可。

示例（沙漠群系）：
- **砂岩开采村** - 专营石材类物品池
- **绿洲农场** - 专营农作物类物品池
- **染料工坊** - 专营染料与装饰类物品池

### 村庄层

村庄生成时，从所属群系的 3-5 个主题中随机选取一个。

同一群系的不同村庄因主题不同，买卖清单有本质差异。

## 主题模板结构

一个主题包含以下字段：

| 字段 | 说明 |
| :--- | :--- |
| 名称 | 如"砂岩开采村""绿洲农场" |
| 所属群系 | 该主题绑定到哪个群系 |
| 出售候选 | 该主题主要出售的候选物品池/物品，生成节点时先展开再按规则抽样 |
| 收购候选 | 该主题主要收购的候选物品池/物品，生成节点时先展开再按规则抽样 |
| 价格修正系数 | 对各物品池的出售/收购修正系数 |
| 主题特产 | 1-2 种该主题独有或高概率出现的特产 |
| 特殊以物易物 | 可选，主题专属固定交换规则（玩法层填充） |
| 库存基准值 | 各物品池的库存上限 |

**价格修正系数规则**：主题修正不叠加。精确物品规则优先；若命中多个标签，则按 `price_modifiers` 的声明顺序取第一条命中规则。详见 [物品池](物品池.md)。

## 特产规则

### 主题特产

由主题定义，1-2 种。同一主题的所有村庄都提供，保证辨识度。

### 随机特产

村庄生成时从全局特产池随机抽取，至少 1 种，最多 3 种。

### 约束

最终特产总种类数 ≥ 1（主题特产已保证此约束自动满足；随机特产至少 1 种）。

村庄特产数量有自然浮动：可能是 2 种（1 主题特产 + 1 随机特产），也可能是 4-5 种（1-2 主题特产 + 1-3 随机特产）。

## 清单生成流程

1. 村庄确定群系
2. 从该群系的 3-5 个主题中随机选取一个
3. 加载主题的出售候选、收购候选、主题特产
4. 展开标签候选，并按 `pick` 等规则抽样生成当期买卖清单
5. 从全局特产池随机抽取额外特产，补入出售清单
6. 确定库存基准和价格修正，生成最终节点数据
