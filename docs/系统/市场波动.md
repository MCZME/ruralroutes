# 市场波动

规则驱动的周期市场事件系统，在每次全局刷新时生成新的 `MarketState`，创造持续变化的价格与货量差异窗口。

## 定义

市场波动不是固定数量的预留槽位，而是一个**规则驱动市场事件池**：

- 内容层通过 `market_event_rules` 定义“可能发生的市场事件”
- 系统层在每个贸易周期刷新时从规则池中抽取规则
- 被抽中的规则会按其作用域展开为当前周期实际生效的一个或多个 `MarketEvent`
- 全部事件共同组成该周期的 `MarketState`

## 触发时机

与全局贸易周期刷新同步进行。

周期刷新时：

1. 上一周期的市场状态整体失效
2. 按当前规则池生成新的市场状态
3. 新状态在整个周期内保持不变

## 生成流程

每次全局刷新按以下流程生成市场波动：

1. 读取数据包中的市场事件规则
2. 按 `weight` 从规则池中抽取若干条规则
3. 每条被抽中的规则会根据 `scopes` 展开为一个或多个 `MarketEvent`
4. 将所有事件组成当前周期的 `MarketState`

**注意**：

- `scopes` 决定规则会展开到哪些村庄；`GLOBAL` 生成 1 条全局事件，`BIOME` / `THEME` 可按 `ALL` 或 `LIST` 展开为多条事件
- 同一 `ruleId + targetRef + scopeType + scopeTarget` 的事件会去重
- 系统不保证“每周期固定有全局事件、区域事件各一个”

## 规则结构

每条市场事件规则至少包含以下字段：

| 字段 | 说明 |
| :--- | :--- |
| `target_ref` | 目标引用。详见 [目标引用](../玩法/数据包制作/目标引用.md)。 |
| `scopes` | 事件作用域展开规则 |
| `delta` | 价格涨跌幅，正数上涨，负数下跌 |
| `stock` | 可选库存修正。`sell` 影响出售现货量，`buy` 影响收购容量 |
| `weight` | 被抽中的相对概率 |
| `name_key` | 本地化名称 key，用于调试或后续展示 |
| `rumor_family` | 可选，指定这条规则在传闻层属于“缺货 / 热销 / 放货 / 过剩”哪一类 |
| `rumor_target_key` | 可选，指定传闻层使用的玩家视角目标名称 |

## 目标单位

规则可直接作用于：

- **单个物品**
- **标签**
- **来源键**

写法和玩法文档里的 [目标引用](../玩法/数据包制作/目标引用.md) 一致。

设计上仍推荐优先使用**标签**作为主要目标单位，这样更容易形成玩家可感知的“某类货物行情变化”；但系统不强制，精确到单物品或来源键的特殊事件同样允许存在。

## 作用域类型

市场事件规则支持三种作用域类型：

| 作用域 | 说明 | 匹配方式 |
| :--- | :--- | :--- |
| `GLOBAL` | 全局生效 | 影响所有村庄 |
| `BIOME` | 指定群系生效 | 仅影响匹配群系的村庄 |
| `THEME` | 指定主题生效 | 仅影响匹配主题的村庄 |

其中：

- `GLOBAL` 不需要目标列表，直接生成全局事件
- `BIOME` / `THEME` 可通过 `ALL` 或显式列表展开
- `THEME` 作用域用于表达“某种村庄主题独有的行情变化”

## 物品受影响规则

基于“目标命中 + 作用域命中”双重判断：

- 物品命中 `target_ref`，且所在村庄命中事件作用域 → 受影响
- 只命中其一 → 不受该事件影响
- 同一物品命中多个事件 → 叠加

**叠加公式**：

> 最终市场因子 = 1 + Σ(所有命中事件的 delta)

**叠加上限**：

总涨跌幅受 `market.max_delta` 限制，默认上限为 ±50%。

## 库存联动规则

命中市场事件时，规则可额外影响库存生成基线：

- `stock.sell`：修正该物品本期卖给玩家的初始现货量
- `stock.buy`：修正该物品本期向玩家收货的容量

库存联动只在以下时机生效：

- 节点首次创建
- 贸易周期刷新后重建库存

周期内玩家买卖只继续消耗或填充库存，不会反向改变当期市场规则本身。

`stock.sell` 与 `stock.buy` 分别按加法叠加，并受 `market.max_stock_delta` 限制。

## 状态重置

- 每次周期刷新时，上一周期全部市场事件整体撤销
- 主题价格修正不被永久改写
- 新周期重新从规则池抽取并生成 `MarketState`
- 市场波动不跨周期累积

## 配置项

| 配置项 | 说明 | 默认值 |
| :--- | :--- | :--- |
| `market.enabled` | 是否启用市场系统 | true |
| `market.rule_pick_count` | 每周期抽取的规则数量 | 3 |
| `market.max_delta` | 市场事件叠加后的最大价格调整百分比 | 0.50 |
| `market.max_stock_delta` | 市场事件叠加后的最大库存调整百分比 | 0.75 |

单条事件的幅度、目标和作用域由规则内容自身定义，不再通过 `volatility_min` / `volatility_max` 这类统一公共参数生成。

## 与传闻板联动

传闻板不再预设“全局情报一条 + 区域情报一条”的固定展示结构，而是直接基于当前周期的 `MarketState` 生成情报：

- 每个市场事件对应一条传闻
- 按 `rumor_family × scope` 选择不同文本模板
- 可通过 `rumor_target_key` 用玩家视角名称覆盖技术标签名
- 若当前周期没有事件，则显示闲谈文本

情报使用模糊语言，不暴露具体数值。
