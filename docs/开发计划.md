## ✅ 预备阶段：技术选型与环境搭建（已完成）

- **确定平台与版本**：NeoForge 21.1.228 + Minecraft 1.21.1 + Java 21（已确定）
- **配置数据驱动框架**：建立 JSON 配置文件加载系统，为物品池、价值表、主题模板、市场事件规则、情报文本库等提供支持
- **集成载具模组 API**（可选）：与"浪漫的载具模组"开发者沟通，确认是否存在 API 可用于检测玩家旅行状态，若无则本模组保持完全独立

## ✅ 第一阶段：核心数据与静态村庄系统（已完成）

**目标**：让世界生成带有贸易站建筑的村庄，实现静态买卖清单。

**系统实现**
- **物品池管理器**：加载物品池配置文件，为每个物品建立多池归属索引 ✅
- **价值表加载与计算**：加载基础价值配置文件，实现 `基础价值 × 主题修正系数` 的静态价格计算 ✅
- **主题模板与村庄生成**：
  - 定义主题数据结构（出售/收购物品子集、主题特产、库存基准、价格修正倾向） ✅
  - 实现 `群系→主题→村庄` 的生成逻辑，每个村庄生成一个中型贸易站建筑 ✅
  - 村庄初始库存按基准值填满 ✅
- **贸易站方块与 GUI**：实现分标签页的常规买卖界面，遵循买卖共享库存、收购价<出售价等规则 ✅
- **展示柜方块**：显示主题特产与预留的特殊以物易物接口（功能暂不实现） ✅
- **货币物品**：铜板、铁币、金币三种面额 ✅
- **贸易周期管理器**：全局统一刷新节律，惰性刷新机制 ✅

**此阶段完成的设计内容**
- 最终确定物品池划分与物品归属列表
- 填入基础价值表的初始数值
- 设计首轮群系主题模板（每个群系 2-3 个），写出第一批主题 JSON

## ✅ 第二阶段：动态经济与情报系统（已完成）

**目标**：让世界"活"起来，实现全局刷新、市场波动与传闻板。

详见第二、三阶段开发计划（已归档）。

**系统实现**
- **贸易周期管理器**：实现全局统一刷新节律，刷新时重建当期清单并恢复所有村庄库存至当期初始状态 ✅
- **市场波动引擎**：
  - 刷新时按权重从规则池抽取市场事件并展开作用域 ✅
  - 将市场事件因子注入价格计算器，实现临时价格调整 ✅
  - 严格遵守多事件叠加规则与上限保护 ✅
- **传闻板方块与 GUI**：
  - 右键界面显示当前周期的市场情报 ✅
  - 方块表面视觉效果提示（延后至统一方块模型制作）
- **情报生成器**：
  - 从市场事件数据中提取目标、方向、作用域 ✅
  - 调用可配置文本库拼装情报 ✅
  - 闲谈文本在无波动时展示 ✅

**此阶段完成的设计内容**
- 确认市场事件规则结构与默认配置
- 填写情报文本库的初始内容

## ✅ 第三阶段：以物易物契约系统（已完成）

**目标**：抽象底层以物易物契约接口，统一动态货币篮交换、固定交换与铸币兑换的数据结构，实现展示柜特产展示功能。

详见第三阶段开发计划（已归档）。

**系统实现**
- **以物易物契约数据结构**：定义 `TradeContract` 接口，统一动态货币篮、固定交换、铸币兑换三种契约类型 ✅
- **契约执行引擎**：实现 `TradeContractExecutor` 统一执行器，支持单一契约和混合交易两种模式 ✅
- **货币篮组合器**：实现 `CurrencyBasketComposer`，支持 `smallest_only`、`largest_first`、`single` 三种组合策略 ✅
- **展示柜基础功能**：实现特产展示 GUI，支持物品图标、名称和 tooltip 显示 ✅
- **规则与主题关联**：在主题模板中定义交易契约，支持多态序列化 ✅
- **现有系统重构**：将价格交易体系重构为契约架构，保留 `TradePricingService` 定价逻辑 ✅

**此阶段完成的设计内容**
- 完成契约系统完整数据结构定义
- 实现 `plains_granary` 主题契约示例数据

## 第四阶段：资源完善与发布准备

**目标**：完成模组资源、玩法验证、平衡性调整，准备首次发布。

**资源完善**
- **方块模型与材质**：贸易站、展示柜、传闻板的正式材质与模型
- **其他风格贸易站**：沙漠、热带草原、针叶林、积雪风格的结构模板
- **货币材质**：铜板、铁币、金币的最终材质（已确定）
- **语言文件**：完善中英文翻译

**玩法验证**
- **生存流程测试**：纯生存模式验证仅通过交换能否通关至末影龙
- **经济平衡测试**：检查刷货币漏洞、价格异常、套利路径
- **多人测试**：验证全局刷新、库存同步、市场波动的多人兼容性

**平衡性调整**
- 根据测试结果调整基础价值表、主题修正系数、波动参数
- 调整特殊以物易物规则

**发布准备**
- **配置文件整理**：将所有可配置项整理为清晰注释的 TOML 文件
- **性能优化**：检查大量村庄生成、全局刷新对服务器 TPS 的影响
- **文档编写**：玩法说明、服务器管理员配置指南
