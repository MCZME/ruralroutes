## ✅ 预备阶段：技术选型与环境搭建（已完成）

- **确定平台与版本**：NeoForge 21.1.228 + Minecraft 1.21.1 + Java 21（已确定）
- **配置数据驱动框架**：建立 JSON 配置文件加载系统，为物品池、价值表、主题模板、市场波动参数、情报文本库等提供支持

## ✅ 第一阶段：核心数据与静态村庄系统（已完成）

**目标**：让世界生成带有贸易站建筑的村庄，实现静态买卖清单。

**系统实现**
- **物品池管理器**：加载物品池配置文件，为每个物品建立多池归属索引 ✅
- **价值表加载与计算**：加载基础价值配置文件，实现 `基础价值 × 主题修正系数` 的静态价格计算 ✅
- **主题模板与村庄生成**：
  - 定义主题数据结构（出售/收购物品子集、主题特产、库存基准、价格修正倾向） ✅
  - 实现 `群系→主题→村庄` 的生成逻辑，每个村庄生成一个中型贸易站建筑 ✅
  - 村庄初始库存按基准值填满 ✅
- **贸易站方块与 GUI**：实现分标签页的常规买卖界面，遵循买卖共享库存、收购价<出售价等规则 ✅
- **展示柜方块**：显示主题特产与预留的特殊以物易物接口（功能暂不实现） ✅
- **货币物品**：铜板、铁币、金币三种面额 ✅
- **贸易周期管理器**：全局统一刷新节律，惰性刷新机制 ✅

**此阶段完成的设计内容**
- 最终确定物品池划分与物品归属列表
- 填入基础价值表的初始数值
- 设计首轮群系主题模板（每个群系 2-3 个），写出第一批主题 JSON

## 第二阶段：动态经济与情报系统

**目标**：让世界"活"起来，实现全局刷新、市场波动与传闻板。

**系统实现**
- **贸易周期管理器**：实现全局统一刷新节律，刷新时恢复所有村庄库存至基准值
- **市场波动引擎**：
  - 刷新时随机选取全局/区域波动对象与幅度
  - 将波动因子注入价格计算器，实现临时价格调整
  - 严格遵守多池叠加规则与上限保护
- **传闻板方块与 GUI**：
  - 方块表面按物品池显示简略图标（有新消息时高亮）
  - 右键界面显示当前周期的全局情报与区域情报
- **情报生成器**：
  - 从市场波动数据中提取物品池、方向、群系
  - 调用可配置文本库拼装情报
  - 闲谈文本在无波动时展示

**此阶段完成的设计内容**
- 调整并确认市场波动默认参数
- 填写情报文本库的初始内容

## 第三阶段：玩法强化与平衡性调整

**目标**：填充特殊以物易物实例，进行游戏性打磨与经济平衡。

**系统实现**
- **特殊以物易物模块**：为展示柜添加 `item-for-item` 类型固定交换规则支持，在统一价值框架外生效
- **经济平衡遍历测试**：
  - 编写调试命令/小工具，模拟玩家跑商，检查刷货币漏洞、价格异常等问题
  - 纯生存流程测试，验证仅通过交换能否舒适通关至末影龙

**此阶段完成的设计内容**
- 设计首批特殊以物易物配方
- 根据测试结果大规模调整基础价值表、主题修正系数和波动幅度

## 第四阶段：配置优化、多人测试与发布准备

**目标**：提供友好的服务器配置体验，确保稳定，准备首次发布。

**主要工作**
- **配置文件集中整理**：将所有可配置项整理为清晰注释的 JSON/TOML 文件
- **多人服务器兼容**：验证全局刷新、库存共享、市场波动在多人下的同步机制，防止刷物品
- **性能优化**：检查大量村庄生成、全局刷新遍历对服务器 TPS 的影响，进行必要的异步处理或分批刷新
- **集成文档**：编写本模组与载具模组配合的推荐玩法说明，以及服务器管理员配置指南
